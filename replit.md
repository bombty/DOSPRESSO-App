# DOSPRESSO Franchise Management Platform

## Overview
DOSPRESSO is a comprehensive franchise management platform designed to streamline operations across various business functions, including HR, factory management, training, finance, and CRM. It aims to provide a unified system for managing franchise-specific processes, ensuring operational efficiency, quality control, and data-driven decision-making. The platform supports a multi-role system, accommodating various hierarchical levels from HQ administration to branch-level staff and factory floor operators. Key capabilities include task management, attendance tracking, AI-powered training academy, quality audits, and integrated communication tools. The platform is designed for scalability and robust data handling, crucial for a growing franchise network.

## System Architecture
The platform utilizes a modern web stack with React 18, TypeScript, and Vite for the frontend, leveraging Shadcn/ui, Tailwind CSS, and CVA for a consistent UI/UX. State management is handled by TanStack Query v5. The backend is built with Node.js and Express.js, also in TypeScript. PostgreSQL (Neon serverless) is used as the primary database with Drizzle ORM for data interaction, and `pgvector` for vector embeddings. Authentication is managed via Passport.js with local and kiosk PIN strategies, using session-based authentication.

**Key Architectural Decisions:**
- **Soft Deletion:** All business-critical data employs soft deletion (`isActive: false` + `deletedAt` timestamp) instead of hard deletes.
- **Data Locking:** Records are subject to time/status-based locking (HTTP 423), with a change request workflow for locked data.
- **Internationalization:** The UI supports multiple languages including Turkish, English, Arabic, and German, with a strong emphasis on proper Turkish character usage for all user-facing strings and error messages.
- **Role-Based Access Control (RBAC):** A granular role system with 27 distinct roles (System, Executive, HQ, Legacy HQ, Branch, Factory Floor, Kiosk) dictates access and permissions across modules and features.
- **Kiosk System:** Dedicated PIN-based authentication for factory and branch kiosks. Sessions are PostgreSQL-backed (`kiosk_sessions` table) with a 30-second in-memory cache layer for performance. All session functions (`createKioskSession`, `validateKioskSession`, `updateKioskStation`, `deleteKioskSession`) are async. Sessions persist across server restarts with 8-hour TTL. Expired sessions are cleaned up on startup and hourly. Middleware: `isKioskAuthenticated`. Factory kiosk uses username + PIN login flow (no personnel list visible on screen). **Factory kiosk token stored in `localStorage` as `factory-kiosk-token`** and sent via `x-kiosk-token` header using `kioskFetch()` helper. `isAuthenticated` middleware also accepts kiosk tokens as fallback (loads full user from DB). Stale shift sessions (>12h) auto-cleaned on startup and hourly.
- **Module Feature Flags:** A dynamic system (`module_flags`) allows for global, branch-level, and role-specific toggling of features and modules. This includes different behaviors like `fully_hidden`, `ui_hidden_data_continues`, and `always_on`, ensuring flexible deployment and configuration. Sub-modules inherit parent module states.
- **Mobile Compactness:** UI components like `CompactKPIStrip` and `MobileFilterCollapse` are designed for an optimized mobile experience, collapsing filters and using horizontal scrollable strips.
- **AI Integration:** AI capabilities (GPT-4o, GPT-4o-mini, Vision, Embeddings) are integrated into various modules, notably the Academy V3 for content generation and assistant features.
- **Branch Task Board:** Implements recurring task management with scheduling, instance generation, override capabilities, and a scoring system integrated into branch health metrics.
- **Academy HQ:** Refactored into a modular 7-tab structure for managing training modules, quizzes, webinars, analytics, exam requests, and certificate designs, with AI-powered content creation tools.
- **Training Optimizer Skill (Enhanced):** Mr. Dobody's `training_optimizer` skill expanded from 3 to 11 insight types — covering quiz performance trends, branch comparison, personal training recommendations, onboarding tracking, usage/engagement reports, certification pipeline, and quiz gap detection. Runs weekly, targets trainer/coach/ceo/cgo/admin roles. Test endpoint: `GET /api/agent/test-skill/training_optimizer` (admin only).
- **Employee Summary Service:** Aggregates per-employee performance metrics (training completion, quiz scores, tasks, branch tasks, attendance) into weighted composite scores. For factory workers, includes factory-specific metrics (production, waste, quality, attendance, break scores) with 50% weight on factory score. Endpoints: `GET /api/employee-summary/:userId`, `GET /api/employee-summary/branch/:branchId`, `GET /api/employee-summary/branch/:branchId/quick`. Role-based access: HQ roles see all, branch managers see own branch only.
- **Factory Worker Scoring Service (`server/services/factory-scoring-service.ts`):** Calculates daily/weekly performance scores for factory workers using 5 weighted components: production efficiency (35%, based on `factory_stations.target_hourly_output`), waste rate (10%), quality check pass rate (15%), attendance/shift coverage (25%), and break discipline (15%). Uses `factory_worker_scores` table with `periodDate` + `periodType` (daily/weekly/monthly). Default neutral scores (70-90) when data is insufficient. Scheduler: daily@02:00, weekly@Monday 03:00. API: `POST /api/factory/scoring/calculate` (admin/fabrika_mudur), `POST /api/factory/scoring/backfill` (admin only), `GET /api/factory/scoring/worker/:userId`. Orphaned break logs (>8h without end) auto-closed on startup and before daily scoring.
- **CareerTab Certificates:** Students can view and print their issued certificates from the Kariyer tab in Academy V3. Uses CertificateRenderer component with template-based rendering and PDF print via `printCertificate()`.
- **Stub Endpoints (`server/routes/stub-endpoints.ts`):** 30+ placeholder API endpoints for modules not yet fully implemented (payroll parameters, salary, academy achievements/streaks/study-groups, admin roles, knowledge-base, coaching, franchise performance, etc.). Registered in `routes.ts` and returns sensible empty defaults to prevent frontend 404/500 errors.
- **Agent Dedup (Global Title-Based):** Agent notification dedup changed from per-user 24h to global title-based 7-day window. Prevents spam like 245 copies of the same stock alert. All skill catch blocks now log errors instead of silently swallowing.
- **Permission Service:** `resolvePermissionScope` and `applyScopeFilter` exported from `server/permission-service.ts` for scope-based data filtering in payroll and HR routes.
- **Self-Service Password Change:** Users can change their own password via `POST /api/me/change-password` (requires current password). UI: Profile page → Ayarlar tab → "Şifre Değiştir" card. Admin bulk reset: `POST /api/admin/bulk-reset-passwords` (admin/muhasebe_ik only). Admin reset sets `mustChangePassword: true` flag, forcing password change on next login via `ForcePasswordChangeDialog` in App.tsx.
- **Notification Preferences:** Stored as JSONB on users table (`notification_preferences`). Categories: task_overdue, agent_guidance, sla_breach, shift_reminder, checklist_reminder, system_announcements. API: `GET /api/notification-preferences`, `PATCH /api/notification-preferences`. UI: Profile page → Ayarlar tab → toggle switches.
- **Shift AI Planner (Enhanced):** `ShiftScheduler` service enhanced with break auto-calculation (`calculateBreakTime`: 3.5-5h after shift start, 60min duration, 10min gap between breaks), break validation (`validateBreakSchedule`: max simultaneous breaks, min staff on floor), and weekly plan generation (`generateWeeklyPlan`: FT 45h/6d, PT 25h/3d, busy day optimization Cu-Ct-Pz, OFF rotation). API: `POST /api/shifts/ai-generate` (preview only), `POST /api/shifts/ai-apply` (save with soft-delete of old shifts, confirmation required), `POST /api/shifts/validate-breaks` (day validation). PDKS compliance: `GET /api/shifts/compliance` (planned vs actual PDKS comparison, on_time/slightly_off/significantly_off/no_pdks status). UI: `vardiya-planlama.tsx` has 3 tabs — Haftalık Görünüm, Takas Talepleri, Uyumluluk (new). Launch week shifts (105) backfilled with staggered break times.
- **Mr. Dobody Guidance System (Autonomous):** Proactive gap detection and user guidance system. `server/services/system-completeness-service.ts` scans 15 gap categories across all branches: personnel (manager/staff/kiosk), checklists (opening/closing), recurring tasks, shifts (global + per-branch weekly), SLA, certificates, feedback, training assignments, bordro parameters, PDKS records, satınalma (suppliers/orders), fabrika (onboarding/targets/shifts), and kalite kontrol (HACCP/audits). Returns `CompletenessItem[]` with severity levels (critical/high/medium/low), Turkish descriptions, deepLinks, and role-based targeting. API: `GET /api/agent/guidance` (role-filtered), `POST /api/agent/guidance/:id/dismiss` (with `guidance_dismissals` table for per-user persistence). Widget `client/src/components/widgets/guidance-widget.tsx` renders at layout level in App.tsx for 13 management roles. Daily scheduler runs gap detection and sends notifications for critical items. Items auto-resolve when gaps are fixed (dynamic, not static).

## External Dependencies
- **OpenAI API**: For AI vision, chat, embeddings, and summarization capabilities.
- **Replit Auth**: Used for user authentication via OpenID Connect.
- **AWS S3**: Employed for cloud storage of user uploads and system backups.
- **Neon Database**: Provides a serverless PostgreSQL database solution.
- **IONOS SMTP**: Utilized for sending email notifications.
