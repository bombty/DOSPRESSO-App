# DOSPRESSO Cheat Sheet — Destek (HQ)

**Hedef Kullanıcı**: HQ Destek — şube/fabrika kullanıcılarının operasyonel sorunlarına ilk hat çözüm
**Cihaz**: Bilgisayar (asıl) + telefon + chat
**Erişim**: Mr. Dobody chat görünümü, tüm şube/fabrika destek talepleri, knowledge base

---

## 1. Destek Nedir?

Sen DOSPRESSO operasyon ekibinin **ilk hat yardım masası**sın. Kullanıcılar Mr. Dobody'e mesaj attığında ve AI çözemediğinde sana yönlendirilir. Müdür/teknik/IT'den önce sen bakarsın.

Pilot süresince: **destek hattı protokolü** (`docs/pilot/destek-hatti-prosedur.md`) sahibi sen olacaksın.

## 3. Ana Ekran (Destek Paneli)

| Widget | İçerik |
|--------|--------|
| **Aktif Talepler** | Bekleyen destek talepleri (öncelik renkli) |

| **Sık Sorulan** | Bu hafta tekrar eden 5 sorun |

| **Eskalasyon Listesi** | Senin çözemediğin → teknik/coach |

---

## 4. Günlük İş Akışı

### Sabah
1. Aktif talepler → kritik öncelik
2. Gece gelen Mr. Dobody eskalasyonu
3. SLA dışında (>1 saat) bekleyen var mı?

### Gün İçi
1. Talep çözme (ortalama 15 dk hedef)
2. Knowledge base güncelle (yeni çözümler)
3. Eskalasyon (gerekirse teknik / coach / müdür)

### Akşam
1. Bugün çözülen vs çözülemeyen
2. Çözülemeyen → yarın için eskalasyon

### Haftalık
1. Sık tekrarlanan 5 sorun raporu
2. Knowledge base genişlet
3. Mr. Dobody'e öğretici yanıt önerileri

---

## 5. Sık Kullanılan İşler

### Destek Talebi Çözme
1. Talep aç → kullanıcı + sorun + ekran görüntüsü
2. Knowledge base ara (benzer sorun)
3. Çözüm uygula veya kullanıcıya talimat
4. Çözüldü → "Kapalı" + çözüm notu (knowledge base güncelle)
5. Çözülmedi → eskalasyon (sebep yaz)

### Mr. Dobody'e Cevap Önerme
1. Mr. Dobody eskalasyonu → AI niye çözemedi?
2. Sen çöz + Mr. Dobody'e "Bu yanıtı öğret" işaretle
3. Sonraki sefer Mr. Dobody benzer soruda cevabı verir

### Eskalasyon (Teknik/Coach)
1. Çözemediğin talep → "Eskalasyon" butonu
2. Kim çözmeli (teknik / coach / müdür / IT)
3. Sebep yaz + bağlam aktar
4. Eskalasyon edilenden takip al
5. Kullanıcıya bilgi ver (beklemiyor şüphede kalmasın)

### Sık Sorulan Sorun Tespiti
1. Sidebar → "Raporlar" → "Sık Sorular"
2. Bu hafta 3+ tekrar eden sorunlar
3. Trainer'a "Eğitim modülü ekle" öneri
4. Mr. Dobody'e "Otomatik cevap ekle" öneri

## 6. Acil Durumlar

| Durum | Aksiyon |
|---|---|
| 1 saat+ bekleyen kritik talep | Anında çözüm + eskalasyon paralel |
| Sistem geneli sorun (10+ kullanıcı aynı sorun) | IT/teknik anında, tüm şubelere bilgi |
| Müşteri sağlık şikayeti çağrısı | Kalite kontrol + müdür anında |
| Mali sapma şikayeti (kasa açığı, ödeme sorunu) | Muhasebe + müdür eskalasyon |
| Personel disiplin şikayeti | İK + müdür eskalasyon (sen müdahale etme) |
| **Sistem çökmesi (genel)** | IT + Mr. Dobody üzerinden tüm kullanıcılara bilgi |
