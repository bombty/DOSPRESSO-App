## FIX 1 — /api/factory/cost-dashboard-stats (CRITICAL — 500 error)

**File:** `server/routes/factory.ts` around line 1156

**Problem:** `db.execute()` returns a NeonQueryResult object `{rows: [...]}`, not a destructurable array.
Code does `const [productCount] = await db.execute(sql\`...\`)` which fails.

**Fix pattern:** Replace all `db.execute()` destructuring with `.rows` access:

```typescript
// WRONG:
const [productCount] = await db.execute(sql`SELECT COUNT(*) FROM ...`);

// RIGHT:
const result = await db.execute(sql`SELECT COUNT(*) FROM ...`);
const productCount = result.rows[0];
```

Find all `db.execute()` calls in the cost-dashboard-stats handler and apply this fix.
Also check: is there a Drizzle builder alternative? If the queries are simple counts/sums,
prefer Drizzle builder over raw SQL:

```typescript
// Preferred (Drizzle builder):
const [productResult] = await db.select({ count: count() }).from(factoryProducts);
const productCount = productResult?.count ?? 0;
```

## FIX 3 — /api/factory/waste-dashboard-stats (CRITICAL — 500 error)

**File:** `server/routes/factory.ts` around line 1312

**Problem:** Same `db.execute()` destructuring issue as Fix 1.
`trendData.map is not a function` because `trendData` is a NeonQueryResult, not an array.

**Fix:**
```typescript
// WRONG:
const trendData = await db.execute(sql`SELECT ...`);
(trendData as any[]).map(...)

// RIGHT:
const trendResult = await db.execute(sql`SELECT ...`);
const trendData = trendResult.rows;
trendData.map(...)
```

Apply this fix to ALL `db.execute()` calls in the waste-dashboard-stats handler.

## FIX 5 — Orphaned Shift Sessions (HIGH)

**Problem:** 4 shift sessions stuck in 'active' state (oldest: 73 days).
These corrupt active worker counts and station occupancy displays.

**Fix:** Close them with a DB migration:

```sql
-- Close all shift sessions that have been active for more than 24 hours
UPDATE factory_shift_sessions 
SET 
  status = 'completed',
  ended_at = NOW(),
  updated_at = NOW()
WHERE 
  status = 'active' 
  AND started_at < NOW() - INTERVAL '24 hours';
```

Run this SQL using `db.execute()` in a one-time migration, or run directly.

**Also add auto-cleanup:** In the scheduler (server/services/agent-scheduler.ts or similar),
add a daily check that closes shift sessions older than 12 hours:

```typescript
// Add to daily scheduler:
export async function closeOrphanedShiftSessions() {
  try {
    await db.execute(sql`
      UPDATE factory_shift_sessions 
      SET status = 'completed', ended_at = NOW(), updated_at = NOW()
      WHERE status = 'active' 
      AND started_at < NOW() - INTERVAL '12 hours'
    `);
    console.log('[Factory] Closed orphaned shift sessions');
  } catch (err) {
    console.error('[Factory] Failed to close orphaned sessions:', err);
  }
}
```

Register with schedulerManager: run once on startup (after 1 min delay), then daily.
