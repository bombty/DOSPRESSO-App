4) Replit’e yapıştırmalık tek prompt (sonraki iş)

Aşağıdakini Replit → Plan/Agent prompt alanına yapıştır:

Implement an AI Policy Console for Mr. Dobody (LLM assistant) with admin-manageable policies.

Goal:
Prevent sensitive data leakage with DB-backed, auditable policies. No security rules hardcoded in prompts only.

Requirements:
1) Define Data Domains (e.g., pii, personnel_performance, factory_costs, procurement_prices, supplier_contracts, recipes, branch_health, equipment_faults, academy_progress, crm_metrics, inventory, production_batches).
2) Introduce RoleGroup abstraction (branch_floor, branch_mgmt, hq_ops, hq_finance, factory, executive) and map all roles to a RoleGroup.
3) Policy table: (role_or_rolegroup, domain, decision=ALLOW|AGG|DENY, scope=self|branch|hq|factory|global, redaction=none|no_names|initials_only|numeric_only, isActive).
4) Policy engine:
   - Detect domain(s) from user question via deterministic keyword/intents.
   - Lookup policy by exact role first, then fallback to roleGroup.
   - Enforce decision:
     - DENY: return refusal + suggest allowed pages.
     - AGG: only aggregated outputs, apply redaction mode strictly.
     - ALLOW: allow within scope, still redact PII by default.
   - Enforce scope clamp (branch_mgmt sees only own branch; factory sees only factory scope, etc).
5) Admin UI page: /admin/ai-politikalari with 3 tabs:
   - Veri Alanlari (CRUD domains)
   - Politika Matrisi (grid roleGroup x domain with decision/scope/redaction)
   - AI Loglari (filterable audit logs)
6) Audit logging:
   - Every assistant request logs to ai_agent_logs (or dedicated table) with role, roleGroup, policy decisions, scope/redaction, input_summary/output_summary (no PII).
7) Add smoke tests:
   - Branch user requesting factory_costs/procurement -> DENY
   - Any user requesting PII -> DENY
   - Executive requesting factory_costs -> AGG(numeric_only) (if policy says so)
8) Keep backward compatibility: if no policy found, default DENY (secure by default).

Deliverables:
- DB migrations (drizzle) for domains + policies
- server/services policy engine module
- integrate into /api/ai/chat endpoint
- admin UI + routes
- QA report with PASS/FAIL evidence
