Başka yapay zekaya DOSPRESSO Webapp’i tanıtmak için kopyala-yapıştır metin

Aşağıdaki metni Gemini/Replit/başka bir AI’a direkt verebilirsin:

⸻

DOSPRESSO HQ Webapp — Sistem Tanıtımı (Özet + Çalışma Prensibi)

Amaç:
DOSPRESSO’nun HQ + şube operasyonlarını tek panelden yöneten, role-based (RBAC), denetlenebilir (auditlenebilir) ve deterministik çalışan bir “işletim sistemi” kurmak: İK + Operasyon + Eğitim (Academy) + Denetim/Checklist + Şikayet + Ekipman + Raporlama.

Temel prensip (kritik):
	•	Deterministik çekirdek (skorlar, yetki, scoping, kayıtlar) LLM’siz çalışır.
	•	LLM sadece read-only yorum/özet katmanında kullanılabilir.
	•	Write path (DB’ye yazan işlemler / yetki değişimi / finansal kayıt) LLM’e açılmaz.
	•	LLM çıktıları katı JSON şeması ile doğrulanır; şema dışıysa fallback deterministik çıktıya dönülür.

Mevcut Modüller / Bileşenler
	1.	RBAC / Rol Scoping

	•	HQ rolleri tüm şubeleri görür
	•	Şube rolleri sadece kendi şubesini görür
	•	Diğer roller erişirse 403

	2.	Branch Health Score Dashboard

	•	Deterministik 5 bileşenli skor motoru (ağırlıklı): Denetim, Şikayet, Ekipman, Eğitim, Operasyonel Hijyen
	•	API: GET /api/reports/branch-health?range=7d|30d|90d
	•	UI sayfaları: /raporlar/sube-saglik ve /sube-saglik-skoru
	•	Trend, risk bayrakları, “yetersiz veri” mantığı mevcut

	3.	Academy (Eğitim)

	•	Academy V2 route sistemi mevcut (academy-v2 routes yapısı)
	•	Eğitim içerikleri / quiz / SOP üretimi için uygun bir “write-safe” alan

	4.	AI log & izlenebilirlik

	•	ai_agent_logs tablosu mevcut (AI çağrıları/çıktıları loglanabilir)
	•	Ama PII loglanmamalı (masking/redaction)

Teknik hedef: “AI Orchestration Layer” (MVP)

Yeni bir framework kurmadan, mevcut Express route mimarisine hafif bir endpoint eklenir:
	•	Örn: server/routes/ai-nba.ts veya server/routes/ai-ops-summary.ts
	•	Bu endpoint deterministik API’lardan veriyi alır → LLM’e gönderir → JSON schema validate eder → UI’a döner
	•	LLM başarısızsa / JSON bozuksa / timeout olursa → circuit breaker ile deterministik fallback döner

Güvenlik / Kalite şartları
	•	LLM sadece read-only summary üretir
	•	Her response: schemaVersion, generatedAt, sourceDeepLinks[] gibi alanlar içerir
	•	Rate limit + timeout + retries kontrollü
	•	“Tool hallucination” riskine karşı: yalnızca sistemin verdiği deepLink/aksiyonları kullanır (uydurmaz)

İstenen çıktı (AI’dan beklenen)
	•	Teknik plan (max 6 task)
	•	Implementasyon (route + prompt + schema + validation + fallback)
	•	QA raporu (PASS/FAIL/SKIP, evidence table, commands)
