## A1: tasks tablosuna yeni kolonlar (⚠ DANIŞMA)

```sql
-- Görev atama akışı için yeni kolonlar
ALTER TABLE tasks ADD COLUMN IF NOT EXISTS task_scope VARCHAR(20) DEFAULT 'branch';
  -- branch: şube görevi, factory: fabrika görevi, hq: merkez ofis görevi, global: tüm lokasyonlar

ALTER TABLE tasks ADD COLUMN IF NOT EXISTS target_department VARCHAR(50);
  -- Teknik, Muhasebe, Operasyon, Eğitim, İK, Kalite, Fabrika, Genel

ALTER TABLE tasks ADD COLUMN IF NOT EXISTS target_branch_ids INTEGER[];
  -- Birden fazla şubeye atanabilir: {5, 6, 7}

ALTER TABLE tasks ADD COLUMN IF NOT EXISTS is_group_task BOOLEAN DEFAULT false;
  -- Grup görevi mi? Sohbet paneli açılır

ALTER TABLE tasks ADD COLUMN IF NOT EXISTS acceptance_required BOOLEAN DEFAULT false;
  -- Atanan kişinin kabul etmesi gerekiyor mu?

ALTER TABLE tasks ADD COLUMN IF NOT EXISTS allow_extension BOOLEAN DEFAULT true;
  -- Süre uzatma talebi yapılabilir mi?

ALTER TABLE tasks ADD COLUMN IF NOT EXISTS parent_task_id INTEGER REFERENCES tasks(id);
  -- Alt görev (sub-task) ilişkisi

ALTER TABLE tasks ADD COLUMN IF NOT EXISTS source VARCHAR(30) DEFAULT 'manual';
  -- manual, crm_ticket, system, dobody, quick_action, kiosk

ALTER TABLE tasks ADD COLUMN IF NOT EXISTS source_id VARCHAR(50);
  -- İlgili ticket/arıza/plan ID'si (kaynak takibi)
```

## A2: task_assignees tablosuna yeni kolonlar

```sql
-- Atanan kişinin yanıt durumu
ALTER TABLE task_assignees ADD COLUMN IF NOT EXISTS acceptance_status VARCHAR(20) DEFAULT 'pending';
  -- pending, accepted, rejected, extension_requested

ALTER TABLE task_assignees ADD COLUMN IF NOT EXISTS accepted_at TIMESTAMP;
ALTER TABLE task_assignees ADD COLUMN IF NOT EXISTS rejected_at TIMESTAMP;
ALTER TABLE task_assignees ADD COLUMN IF NOT EXISTS rejection_reason TEXT;

ALTER TABLE task_assignees ADD COLUMN IF NOT EXISTS extension_requested_at TIMESTAMP;
ALTER TABLE task_assignees ADD COLUMN IF NOT EXISTS extension_reason TEXT;
ALTER TABLE task_assignees ADD COLUMN IF NOT EXISTS extension_days INTEGER;
ALTER TABLE task_assignees ADD COLUMN IF NOT EXISTS extension_approved BOOLEAN;

ALTER TABLE task_assignees ADD COLUMN IF NOT EXISTS completion_rate INTEGER DEFAULT 0;
  -- Kişi bazlı tamamlama yüzdesi (0-100)

ALTER TABLE task_assignees ADD COLUMN IF NOT EXISTS notes TEXT;
  -- Kişinin göreve eklediği not/detay talebi
```

## A3: Görev sohbet/yorum tablosu (varsa genişlet, yoksa oluştur)

```sql
-- Görev üzerinde sohbet (CRM ticket_comments benzeri)
CREATE TABLE IF NOT EXISTS task_comments (
  id SERIAL PRIMARY KEY,
  task_id INTEGER NOT NULL REFERENCES tasks(id),
  user_id VARCHAR(36) NOT NULL REFERENCES users(id),
  message TEXT NOT NULL,
  comment_type VARCHAR(20) DEFAULT 'message',
    -- message: normal yorum
    -- status_change: durum değişikliği notu
    -- extension_request: süre uzatma talebi
    -- question: soru/detay talebi
    -- system: sistem mesajı (otomatik)
  attachment_url TEXT,
  is_internal BOOLEAN DEFAULT false,  -- sadece yöneticiler görür
  created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX IF NOT EXISTS idx_task_comments ON task_comments(task_id, created_at);
```

## B3: Toplu görev atama

Şube/rol bazlı toplu atama:
```
"Tüm şubelerin supervisor'larına haftalık stok sayımı ata"
→ 20 şube × 1 supervisor = 20 görev (otomatik oluşturulur)
→ Her görev ilgili şubenin supervisor'ına atanır
→ İlerleme: 20 görevden 15 tamamlandı (%75)
```

Backend:
```
POST /api/tasks/bulk
{
  title: "Haftalık stok sayımı",
  targetScope: "branch",
  targetBranchIds: [5,6,7,8,9,...], // veya "all"
  targetRole: "supervisor",  // bu roldeki kişilere ata
  dueDate: "2026-03-30",
  isGroupTask: false,
  acceptanceRequired: true
}
→ Her şubedeki supervisor için ayrı task oluşturur
→ Toplu ilerleme takibi: /api/tasks/bulk/:bulkId/progress
```

## E2: Bildirim entegrasyonu

Görev ile ilgili tüm bildirimler:
- Görev atandığında → atanan kişiye bildirim
- Kabul/red → oluşturana bildirim
- Süre uzatma talebi → oluşturana bildirim
- Sohbet mesajı → görevdeki herkese bildirim (kendisi hariç)
- Görev tamamlandığında → oluşturana bildirim
- Bitiş tarihi yaklaştığında (1 gün kala) → atanan kişiye hatırlatma
- Geciken görev → oluşturana + atanana uyarı

**Bildirim seviye filtresi (Sprint 9'dan):**
- Bireysel görev bildirimi → operational (supervisor seviyesinde kalır)
- Toplu görev gecikme özeti → tactical (müdür + coach görür)
- Genel görev KPI uyarısı → strategic (CEO/CGO görür)
