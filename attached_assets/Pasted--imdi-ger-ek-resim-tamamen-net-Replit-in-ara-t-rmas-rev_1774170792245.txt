**"Araştırman mükemmel — şimdi dashboard Mission Control Aşama 1'e başlıyoruz. Senin revize planını takip ediyoruz.**

## AŞAMA 1: Mission Control Temel Altyapı

### Hedef
Klasik ↔ Mission Control toggle sistemi. Mevcut dashboard'lara dokunmadan, yanlarına yeni bir görünüm seçeneği ekliyoruz. Bu aşamada Mission Control sayfalarının kendisi YOK — sadece altyapı + toggle + Mr. Dobody panel.

### 1. users tablosuna dashboard_preferences kolonu

Senin önerdiğin gibi JSONB:
```sql
ALTER TABLE users ADD COLUMN IF NOT EXISTS dashboard_preferences JSONB DEFAULT '{"mode": "classic"}';
```

Schema dosyasına da ekle (hangi schema-XX.ts dosyasında users tanımlıysa oraya).

### 2. useGuidanceData hook'u

Mevcut guidance-widget.tsx'teki veri çekme mantığını bir hook'a çıkar:
- `useGuidanceData()` → `{ totalGaps, criticalCount, items, grouped, isLoading }`
- Mevcut guidance-widget.tsx bu hook'u kullanacak şekilde güncelle (kendi fetch'i yerine hook çağırsın)
- Böylece hem mevcut widget hem de yeni Mr. Dobody paneli aynı veriyi kullanır
- API değişikliği YOK — aynı `/api/agent/guidance` endpoint'i

### 3. useDashboardMode hook'u

```typescript
// Tercih okuma: GET /api/me/dashboard-preferences
// Tercih yazma: PATCH /api/me/dashboard-preferences
// Return: { mode, setMode, isLoading }
```

Backend endpoint'leri:
- `GET /api/me/dashboard-preferences` → kullanıcının dashboard_preferences JSONB'sini dön
- `PATCH /api/me/dashboard-preferences` → güncelle (merge, overwrite değil)
- Admin default: `ai_system_config` tablosunda `default_dashboard_mode = 'classic'` kaydı

### 4. Toggle bileşeni

Basit bir switch/toggle — Klasik ↔ Mission Control. Mevcut shadcn/ui Switch veya kendi toggle'ın. Dashboard sayfalarının header'ında sağ üstte görünecek.

### 5. Dashboard routing entegrasyonu

Senin araştırmanda belirttiğin 5 adımlı routing'e toggle eklenmesi:
- dashboard.tsx: Her yönlendirme noktasında `mode === 'mission-control'` kontrolü
- hq-ozet sayfası: Aynı kontrol (CEO/CGO/Admin buraya redirect oluyor)
- mode === 'mission-control' olduğunda şimdilik placeholder göster: "Mission Control — Aşama 2'de gelecek"
- mode === 'classic' → mevcut davranış AYNEN devam (hiçbir değişiklik yok)

### 6. Mr. Dobody sol panel (DobodyPanel)

Masaüstü davranış:
- Varsayılan: 44px rail (avatar + badge + severity dots)
- Tıklayınca: 240px panel (header + severity pills + scrollable feed + arama)
- localStorage'da durum sakla: `dobody-panel-expanded`
- Sadece Mission Control modunda görünür
- Veri: useGuidanceData hook'undan

Mobil davranış:
- Sol panel YOK — bunun yerine sayfanın üstünde kompakt kart
- Tıklayınca genişleyen özet

Feed items:
- Mevcut guidance-widget'taki gruplama ve severity renklendirmesini kullan
- Her item: severity badge + title + description + "Düzelt →" deepLink
- Sıralama: critical → high → medium → low

### 7. Admin ayarı

Admin settings sayfasına (zaten varsa) bir kart ekle:
- "Varsayılan Dashboard Görünümü" — Klasik / Mission Control radio
- Bu ai_system_config'deki default_dashboard_mode'u günceller
- Not: "Her kullanıcı kendi tercihini değiştirebilir"
