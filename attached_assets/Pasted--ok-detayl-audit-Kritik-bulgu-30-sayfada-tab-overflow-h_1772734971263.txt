### T002 — Dashboard Responsive Grid Fix (~2-3 saat)

**Problem:** 5-6 dashboard sayfasında `grid-cols-3` veya `grid-cols-4` breakpoint olmadan kullanılmış. Mobilde/tablette kartlar sıkışıyor.

**Yapılacak:**

Her dosyada sabit grid'i responsive yap:

```
manager-dashboard.tsx:43 → grid-cols-3 → grid-cols-1 sm:grid-cols-2 lg:grid-cols-3
barista-dashboard.tsx:46 → grid-cols-3 → grid-cols-1 sm:grid-cols-2 lg:grid-cols-3
teknik-dashboard.tsx:49 → grid-cols-3 → grid-cols-1 sm:grid-cols-2 lg:grid-cols-3
teknik-dashboard.tsx:89 → grid-cols-4 → grid-cols-2 sm:grid-cols-3 lg:grid-cols-4
muhasebe-dashboard.tsx:106 → grid-cols-3 → grid-cols-1 sm:grid-cols-2 lg:grid-cols-3
analytics-card.tsx:39 → grid-cols-3 → grid-cols-1 sm:grid-cols-2 lg:grid-cols-3
```

Ayrıca grep ile başka sabit grid'leri de tara:
```bash
grep -rn "grid-cols-[3-6]" client/src/pages/ --include="*.tsx" | grep -v "sm:\|md:\|lg:" | head -20
```

Bulunan her sabit grid'i responsive breakpoint ile düzelt.

**Doğrulama:**
- 768px genişlikte (iPad portrait) kartlar 2 sütun
- 375px genişlikte (mobil) kartlar 1 sütun
- 1024px+ genişlikte (iPad landscape / desktop) orijinal 3-4 sütun

### T003 — Safe Area Desteği (~1 saat)

**Problem:** viewport-fit=cover meta var ama env(safe-area-inset-bottom) kullanılmıyor. iPhone notch ve home indicator ile bottom nav çakışabilir.

**Yapılacak:**

1. Bottom nav component'ine safe area padding ekle:
```css
padding-bottom: env(safe-area-inset-bottom, 0px);
```

2. Ana layout'a safe area desteği:
```css
/* globals.css veya tailwind config'e */
.pb-safe {
  padding-bottom: env(safe-area-inset-bottom, 0px);
}
```

3. Bottom nav'ın bottom pozisyonunu safe area'ya göre ayarla:
```
bottom: max(12px, env(safe-area-inset-bottom, 12px))
```

**Doğrulama:**
- iPhone X+ modelde bottom nav home indicator'ın üzerinde kalıyor
- iPad'de değişiklik yok (safe area 0)

### T004 — Küçük Touch Hedefi Düzeltmeleri (~1-2 saat)

**Problem:** ~15 buton 24-28px yükseklikte (minimum 44px önerilir). Mobilde tıklaması zor.

**Yapılacak:**

1. h-6 (24px) icon butonlarını bul ve minimum h-9 (36px) veya h-10 (40px) yap:
```bash
grep -rn "h-6.*button\|h-6.*click\|h-6.*Button\|h-7.*button\|h-7.*click" client/src/ --include="*.tsx" | head -15
```

2. Her küçük butonu düzelt — butonun kendisi küçük kalabilir ama tıklanabilir alan (padding) büyütülmeli:
```tsx
// ÖNCE (kötü)
<button className="h-6 w-6">
  <Icon size={14} />
</button>

// SONRA (iyi) — görsel boyut aynı, dokunma alanı büyük
<button className="h-6 w-6 p-2 -m-2"> 
  <Icon size={14} />
</button>
// veya
<button className="min-h-[44px] min-w-[44px] flex items-center justify-center">
  <Icon size={14} />
</button>
```

3. text-xs butonlarda da minimum padding ekle

**Doğrulama:**
- Tüm tıklanabilir öğeler en az 36x36px dokunma alanına sahip

### T005 — Lazy Loading Genişletme (~2-3 saat)

**Problem:** 228 sayfa dosyasından sadece 11'i lazy-loaded. Geri kalanı App.tsx'te direkt import — ilk yükleme süresini artırıyor.

**Yapılacak:**

1. App.tsx'teki route tanımlarını kontrol et:
```bash
grep -rn "import.*from.*pages/" client/src/App.tsx | wc -l
head -80 client/src/App.tsx
```

2. En az kullanılan / en büyük sayfaları lazy-load'a çevir. Strateji:
   - Dashboard ve login: lazy YAPMA (ilk yüklenen sayfalar)
   - Diğer tüm sayfalar: React.lazy ile lazy load

3. Her lazy-loaded sayfa için Suspense wrapper + loading skeleton:
```tsx
const IKPage = React.lazy(() => import('./pages/ik'));

// Route'da:
<Suspense fallback={<PageSkeleton />}>
  <IKPage />
</Suspense>
```

4. PageSkeleton component oluştur (generic loading placeholder):
- Header skeleton
- Kart grid skeleton
- Tablo skeleton
- Sayfa tipine göre farklı skeleton'lar (opsiyonel)

5. Image lazy loading:
```bash
grep -rn "<img " client/src/ --include="*.tsx" | grep -v 'loading="lazy"' | head -20
```
Bulunan tüm img tag'lerine `loading="lazy"` ekle.

**Doğrulama:**
- Network tab'da ilk yükleme boyutu azalmış
- Sayfa geçişlerinde kısa loading skeleton görünüyor
- Fonksiyonalite aynı (lazy loading kırıklık yapmamış)
