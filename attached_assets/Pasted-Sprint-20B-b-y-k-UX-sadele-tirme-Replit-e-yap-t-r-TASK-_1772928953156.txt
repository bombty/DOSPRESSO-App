### T005 — Role-Based Tab Filtering for Mega Modules (~2 hours)

**Problem:** 13 pages have 8+ tabs with NO role filtering. Everyone sees every tab.

**Fix the top 5 worst offenders:**

**A) module-detail.tsx (23 tabs):**
```bash
grep -rn "TabsTrigger" client/src/pages/module-detail.tsx | head -25
```
- Barista/stajyer: Show only content + quiz tabs (max 4)
- Trainer: Show all content + quiz + stats tabs
- Admin: Show all tabs

**B) personel-detay.tsx (19 tabs):**
- Supervisor: profil, performans, egitim, vardiya (4 tabs)
- Muhasebe: profil, maas, izin (3 tabs)
- Admin/HR: all tabs

**C) receteler.tsx (19 tabs):**
- Barista: sadece reçete görüntüleme (1-2 tab)
- Trainer/Chef: edit + manage tabs
- Admin: all tabs

**D) yeni-sube-detay.tsx (19 tabs):**
- Only admin/ceo/cgo should see this page at all
- Other roles: redirect away or show limited view

**E) muhasebe.tsx (13 tabs):**
- Muhasebe: all tabs
- CEO/CGO: only summary/overview tab
- Others: no access

**Implementation pattern for each page:**

```typescript
const userRole = useAuth().user?.role;
const visibleTabs = ALL_TABS.filter(tab => {
  if (userRole === 'admin') return true;
  if (tab.roles && !tab.roles.includes(userRole)) return false;
  return true;
});
```

Add a `roles?: string[]` property to each tab definition. If roles is undefined, tab is visible to all. If roles is defined, only those roles see it.
