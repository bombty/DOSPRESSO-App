Replit’ten gelen değerlendirme doğru çizgide: DOSPRESSO’nun “deterministik çekirdeği” korunmalı; agent katmanı üstte orchestration (öneri/özet/üretim) olarak durmalı. Ben de aynı sonuca varıyorum — sadece 2 noktayı daha “ürün standardı” seviyesinde netleştiririm.

1) Lead + Subagent eklenmeli mi?

Evet. Ama 3 şartla:
	1.	Write-path LLM’e kapalı (RBAC, finans, stok, yetki, kayıt üretme yok).
	2.	Agent output’u daima şemalı JSON (serbest metin sadece UI için “render”).
	3.	Fallback deterministik (LLM yoksa NBA/Score zaten çalışmaya devam eder).

Bu 3’ü yapınca risk düşük, değer yüksek.

2) DOSPRESSO’da “LangGraph/CrewAI şart mı?”

Hayır. Replit agent’in dediği gibi:
Express + (Gemini/OpenAI) + Zod/Pydantic tarzı JSON doğrulama + log/limit ile MVP çıkıyor.

LangGraph gibi framework’ler ancak şunlarda anlamlı olur:
	•	uzun süren job’lar (dakikalar)
	•	checkpoint/resume ihtiyacı
	•	çok adımlı HITL akışları
	•	paralel worker/queue yapısı

MVP için “hafif orchestration layer” yeter.

4) Academy Content Factory için “ürün standardında” tasarım (önerim)

Kullanıcı akışı
	•	HQ kullanıcı → Academy > Content Factory
	•	Form:
	•	Hedef rol (Barista/Supervisor/Müdür/Cleaning/MESEM…)
	•	Konu (örn. “Espresso extraction”, “Hijyen”, “Kasa prosedürü”)
	•	Seviye (1–5)
	•	Dil (TR/EN)
	•	Çıktı tipi: SOP / Micro-lesson / Quiz / Checklist
	•	Çıktı:
	•	JSON (canonical)
	•	UI render (markdown)
	•	“Kaydet” → taslak olarak DB’ye (draft)
	•	“Yayınla” → sadece HITL onayı ile (admin)

Minimum endpoint set
	•	POST /api/academy/content-factory/generate  → üretim (read-only)
	•	POST /api/academy/content-factory/drafts → taslak kaydet
	•	GET /api/academy/content-factory/drafts → liste
	•	PATCH /api/academy/content-factory/drafts/:id → düzenle
	•	POST /api/academy/content-factory/drafts/:id/publish → onaylı yayın

JSON sözleşmeleri (örnek)
	•	SOP:
	•	title, audienceRole, objectives[], steps[{id, text, critical, evidenceHint}], commonMistakes[], checklist[], durationMin
	•	Quiz:
	•	title, questions[{q, choices[], answerIndex, rationale, difficulty, tags[]}]
	•	Micro-lesson:
	•	sections[{heading, bullets[]}] + recap + “do this today”

Guardrail’ler
	•	Model’e kesin kural: “DB’ye yazma / rol-yetki önerme / finansal talimat üretme yok”
	•	Max token + tool-call limit
	•	Her üretimde traceId, promptVersion, model, costEstimate log

6) Başka yapay zekaya DOSPRESSO’yu tanıtmak için “tek parça briefing”

Bunu diğer AI’ya olduğu gibi yapıştırabilirsin (kısa ama tam):

You are onboarding to DOSPRESSO HQ WebApp (coffee/donut franchise ops OS).

GOAL
A single role-based HQ + branch operating system: HR, Academy training, Ops audits/checklists, Logistics/production, Finance/COGS, CRM/loyalty, and reporting. Core principle: deterministic business logic + auditable RBAC.

CURRENT CORE (DETERMINISTIC)
- Role-based access (HQ roles see all branches; branch roles see only own branch; others blocked).
- NBA V2: rule-based Next Best Actions per user/branch.
- Branch Health Score Dashboard:
  - API: GET /api/reports/branch-health?range=7d|30d|90d
  - Deterministic 5-component scoring: Inspections 25%, Complaints 25%, Equipment 20%, Training 15%, Ops Hygiene 15%
  - Outputs: per-branch totalScore, components, trend, risk flags, deepLinks
  - UI routes: /raporlar/sube-saglik (primary) and /sube-saglik-skoru (backward compat)

IMPORTANT ENTITIES
Users (role, branchId, employeeTypeId), Branches, Inspections/Checklists, Complaints, Equipment logs, Training progress/Academy modules.

NON-NEGOTIABLE CONSTRAINTS
- No LLM in core enforcement: RBAC, scoring, finance, inventory, write operations must remain deterministic.
- Any AI feature must be “suggest → human approve → apply via deterministic modules”.
- AI outputs must be schema-validated JSON (no free-form actions).

NEXT PLANNED AI LAYER (ORCHESTRATION ONLY)
- Academy Content Factory: generate SOPs/micro-lessons/quizzes/checklists safely (draft → approval → publish).
- Ops Copilot: read-only Turkish action summaries on top of NBA V2 + branch health, with deepLinks.
