## PART 9: DEDUP FIX — Add Status Filter
## ═══════════════════════════════════════

The agent dedup has a side effect: if a problem is resolved and recurs, no alert for 7 days.

```bash
grep -n "dedup\|duplicate" server/agent/skills/skill-notifications.ts | head -10
```

Add status filter to dedup query:
```typescript
// Current: blocks ALL same-title alerts for 7 days
// Fix: only block if there's a PENDING or APPROVED alert
const existing = await db.select().from(agentPendingActions)
  .where(and(
    eq(agentPendingActions.title, action.title),
    inArray(agentPendingActions.status, ['pending', 'approved']),  // ADD THIS
    gte(agentPendingActions.createdAt, sevenDaysAgo)
  ));
```

This way: if the alert was completed/resolved/expired, a new occurrence will trigger a fresh alert.
