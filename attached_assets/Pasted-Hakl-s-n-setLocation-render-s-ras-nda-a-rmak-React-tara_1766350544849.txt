Haklısın. setLocation()’ı render sırasında çağırmak React tarafında uyarı (ve bazı durumlarda double-render/loop) üretebilir. Wouter’da da en güvenlisi, yönlendirmeyi useEffect içinde yapmak.

Aşağıda aynı çözümün “React uyumlu” ve minimal değişiklikli, useEffect’li hali var.

1) AuthCatchAllToLogin’ı useEffect ile düzelt

import { useEffect } from "react";
import { useLocation } from "wouter";

const PUBLIC_PATH_PREFIXES = ["/login", "/register", "/feedback"];

function isPublicPath(path: string) {
  return PUBLIC_PATH_PREFIXES.some((p) => path === p || path.startsWith(p + "/"));
}

export function AuthCatchAllToLogin() {
  const [location, setLocation] = useLocation();

  useEffect(() => {
    // Public sayfadaysak redirect yok (loop önler)
    if (isPublicPath(location)) return;

    // Hedefi sakla
    sessionStorage.setItem("postLoginRedirect", location);

    const next = encodeURIComponent(location);
    setLocation(`/login?next=${next}`);
  }, [location, setLocation]);

  return null;
}

2) Router’da kullanımı (aynı kalıyor)

import { Switch, Route } from "wouter";
import { useAuth } from "./auth/useAuth";
import { AuthCatchAllToLogin } from "./AuthCatchAllToLogin";

export function Router() {
  const { isAuthenticated, isLoading } = useAuth();

  if (isLoading) return <div>Yükleniyor...</div>;

  return (
    <Switch>
      <Route path="/login" component={Login} />
      <Route path="/register" component={Register} />
      <Route path="/feedback" component={MusteriFeedbackPublic} />

      {!isAuthenticated && <Route path="/:rest*" component={AuthCatchAllToLogin} />}

      {isAuthenticated && (
        <>
          <Route path="/" component={Dashboard} />
          <Route path="/gorevler" component={Tasks} />
          <Route path="/ekipman" component={Equipment} />
          {/* ... 100+ protected route */}
        </>
      )}

      <Route component={NotFound} />
    </Switch>
  );
}

3) Login sonrası geri dönüş (aynı konsept, küçük güvenlik filtresiyle)

import { useLocation } from "wouter";

function getSafeNextFromUrl(): string | null {
  const params = new URLSearchParams(window.location.search);
  const raw = params.get("next");
  if (!raw) return null;

  const decoded = decodeURIComponent(raw);
  return decoded.startsWith("/") ? decoded : null; // open-redirect önlemi
}

export function Login() {
  const [, setLocation] = useLocation();

  async function onLoginSuccess() {
    const nextFromUrl = getSafeNextFromUrl();
    const nextFromStorage = sessionStorage.getItem("postLoginRedirect");
    sessionStorage.removeItem("postLoginRedirect");

    const target =
      nextFromUrl ||
      (nextFromStorage && nextFromStorage.startsWith("/") ? nextFromStorage : null) ||
      "/";

    setLocation(target);
  }

  return (
    <div>
      {/* submit başarılı → onLoginSuccess() */}
    </div>
  );
}

Bu revizyonla:
	•	React uyarısı kalkar (render içinde state update yok).
	•	100+ route’u sarmadan “auth yoksa her protected path → /login” davranışı sağlanır.
	•	Login sonrası geldiği sayfaya dönüş korunur.

İstersen bir sonraki adım olarak, “/api” çağrıları da auth yokken 401/403 dönüyor mu ve frontend bunları otomatik login’e atıyor mu (session expire senaryosu) onu da aynı mantıkla tek noktadan ele alacak şekilde tasarlayabiliriz.
