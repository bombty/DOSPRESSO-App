### 1A: employee_documents table

```sql
CREATE TABLE IF NOT EXISTS employee_documents (
  id SERIAL PRIMARY KEY,
  user_id VARCHAR(36) NOT NULL REFERENCES users(id),
  document_type VARCHAR(50) NOT NULL,
  title VARCHAR(200) NOT NULL,
  description TEXT,
  file_url TEXT,
  file_type VARCHAR(20),
  file_size INTEGER,
  uploaded_by VARCHAR(36) REFERENCES users(id),
  is_required BOOLEAN DEFAULT false,
  expiry_date DATE,
  status VARCHAR(20) DEFAULT 'active',
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);
CREATE INDEX IF NOT EXISTS idx_emp_docs_user ON employee_documents(user_id, document_type);
```

Document types: kimlik, adli_sicil, ikametgah, diploma, saglik_raporu, is_sozlesmesi, sgk_bildirge, maas_bordrosu, izin_formu, tutanak, performans, egitim_sertifika, diger

### 1B: disciplinary_records table

```sql
CREATE TABLE IF NOT EXISTS disciplinary_records (
  id SERIAL PRIMARY KEY,
  user_id VARCHAR(36) NOT NULL REFERENCES users(id),
  record_type VARCHAR(30) NOT NULL,
  title VARCHAR(200) NOT NULL,
  description TEXT NOT NULL,
  incident_date DATE NOT NULL,
  witnesses TEXT,
  evidence_urls TEXT,
  response TEXT,
  response_date DATE,
  action_taken TEXT,
  created_by VARCHAR(36) NOT NULL REFERENCES users(id),
  approved_by VARCHAR(36) REFERENCES users(id),
  status VARCHAR(20) DEFAULT 'active',
  severity VARCHAR(20) DEFAULT 'medium',
  affects_score BOOLEAN DEFAULT true,
  score_impact INTEGER DEFAULT 0,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);
CREATE INDEX IF NOT EXISTS idx_disciplinary_user ON disciplinary_records(user_id, status);
```

Record types: sozlu_uyari, yazili_uyari, ihtar, savunma_talebi, kinama, ucret_kesintisi, fesih, odullendirme, tesekkur

    hr.ts                → employee_documents, disciplinary_records, leaves

### CRITICAL — barrel re-export pattern:
```typescript
// shared/schema.ts becomes:
export * from './schema/users';
export * from './schema/branches';
export * from './schema/factory';
export * from './schema/shifts';
export * from './schema/pdks';
export * from './schema/payroll';
export * from './schema/tasks';
export * from './schema/training';
export * from './schema/messaging';
export * from './schema/crm';
export * from './schema/agent';
export * from './schema/settings';
export * from './schema/hr';
// ... etc

// Also keep shared types/enums that are used across modules:
export * from './schema/shared-types';
```

This way ALL existing imports still work:
```typescript
// This still works — no changes needed anywhere else:
import { users, branches, shifts } from '@shared/schema';
```

### Process:
1. Create shared/schema/ directory
2. Move ONE group at a time (start with smallest: settings)
3. Add re-export to schema.ts
4. Verify: `npx tsc --noEmit` or app starts
5. Move next group
6. Repeat until schema.ts is just re-exports (~50 lines)

### 4C: Replace `catch (error: any)` → `catch (error: unknown)`

```bash
grep -rn "catch.*: any\|catch.*:any" server/ --include="*.ts" | grep -v node_modules | head -20
```

Replace pattern:
```typescript
// BEFORE:
catch (error: any) { console.error(error.message); }
// AFTER:
catch (error: unknown) {
  const message = error instanceof Error ? error.message : 'Unknown error';
  console.error(message);
}
```

### 4D: Remove unnecessary `as any` from Drizzle queries

```bash
grep -rn "as any" server/ --include="*.ts" | grep -v node_modules | grep "db\.\|select\|insert\|update\|delete" | head -15
```

Drizzle infers types — most `as any` on query results are unnecessary. Remove them.
