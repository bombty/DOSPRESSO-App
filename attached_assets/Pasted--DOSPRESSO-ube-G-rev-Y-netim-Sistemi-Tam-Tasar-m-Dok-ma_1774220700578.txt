## 1. VİZYON

Coach veya Trainer, her şube için günlük/haftalık/aylık tekrarlayan görevler tanımlar (cam temizliği, sayım, ekipman bakımı vb.). Bu görevler zamanı gelince otomatik oluşur. Mr. Dobody şubeye hatırlatma yapar. Görev ya birine atanır ya da "açık havuz"da kalır — bir çalışan inisiyatif alıp sahiplenir. İnisiyatif alan çalışan performans skorunda ödüllendirilir. Tamamlama, gecikme ve inisiyatif verileri raporlanır.

## 3. SİSTEM MİMARİSİ

### 3.1 Veri Modeli

#### Mevcut tablolar (DOKUNULMAYACAK)

- `tasks` — Ana görev tablosu (1.183 kayıt)
- `task_templates` — Recurring task şablonları
- `task_assignments` — Görev atamaları
- `checklist_assignments` — Checklist atamaları (163 kayıt)

#### Yeni/Güncellenecek alanlar

**tasks tablosuna eklenecek alanlar:**

```
initiative_claimed_by  VARCHAR(36)  — İnisiyatif alan kişinin user_id'si
initiative_claimed_at  TIMESTAMP    — İnisiyatif alınma zamanı
is_open_pool           BOOLEAN      — Açık havuz görevi mi (atanmamış, herkes alabilir)
initiative_bonus       INTEGER      — Bu görev için inisiyatif bonusu (varsayılan: 5)
completed_at           TIMESTAMP    — Tamamlanma zamanı (süre hesabı için)
completion_quality     VARCHAR(20)  — Tamamlama kalitesi: excellent/good/acceptable/poor
```

**Yeni tablo: branch_task_schedules (şube bazlı takvim)**

```sql
CREATE TABLE IF NOT EXISTS branch_task_schedules (
  id SERIAL PRIMARY KEY,
  template_id INTEGER NOT NULL REFERENCES task_templates(id),
  branch_id INTEGER NOT NULL REFERENCES branches(id),
  is_active BOOLEAN DEFAULT true,
  custom_frequency VARCHAR(20),     -- Şubeye özel sıklık (override)
  custom_day_of_week INTEGER,       -- Şubeye özel gün (override)
  custom_time VARCHAR(5),           -- Şubeye özel saat (override)
  notes TEXT,                       -- Şubeye özel not
  created_by VARCHAR(36) REFERENCES users(id),
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW(),
  UNIQUE(template_id, branch_id)
);
```

**Yeni tablo: initiative_scores (inisiyatif skor kaydı)**

```sql
CREATE TABLE IF NOT EXISTS initiative_scores (
  id SERIAL PRIMARY KEY,
  user_id VARCHAR(36) NOT NULL REFERENCES users(id),
  task_id INTEGER NOT NULL REFERENCES tasks(id),
  branch_id INTEGER NOT NULL REFERENCES branches(id),
  score_type VARCHAR(20) NOT NULL,  -- 'initiative_claim', 'early_completion', 'quality_bonus'
  points INTEGER NOT NULL,          -- Pozitif değer
  period_week INTEGER NOT NULL,     -- Yılın kaçıncı haftası
  period_year INTEGER NOT NULL,
  created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_initiative_user_period 
  ON initiative_scores(user_id, period_year, period_week);
```

### 3.2 Görev Yaşam Döngüsü

```
OLUŞTURMA (Coach/Trainer/Müdür)
    │
    ├── Recurring template → Otomatik oluşur (TaskTrigger)
    │   └── Her Pazartesi 07:00 → "Cam temizliği" task oluşur
    │
    └── Manuel oluşturma → Anlık görev
    
    ↓

ATAMA KARARI
    │
    ├── Doğrudan atama → Belirli kişiye atanır
    │   └── Bildirim: "Size yeni görev atandı: Cam temizliği"
    │
    └── Açık havuz → is_open_pool = true, kimseye atanmamış
        └── Mr. Dobody: "Bugün cam temizliği var — kim alacak?"
    
    ↓

İNİSİYATİF ALMA (sadece açık havuz görevleri)
    │
    └── Çalışan "Ben Alıyorum" butonuna tıklar
        ├── initiative_claimed_by = user_id
        ├── initiative_claimed_at = NOW()
        ├── initiative_scores → +5 puan (initiative_claim)
        └── Bildirim: Coach'a "Ahmet cam temizliğini üstlendi"
    
    ↓

TAMAMLAMA
    │
    ├── Zamanında → Normal tamamlama
    │   └── Erken tamamlama bonusu: deadline'dan 2+ saat önce → +3 puan
    │
    ├── Gecikmeli → Gecikme kaydı
    │   └── Mr. Dobody: Müdüre "Cam temizliği 3 saat gecikti"
    │
    └── Yapılmadı → Escalation
        └── Mr. Dobody: Coach'a "Işıklar şubesi cam temizliğini yapmadı"
    
    ↓

RAPORLAMA
    ├── Haftalık: Şube bazlı tamamlama oranı
    ├── Aylık: Çalışan bazlı inisiyatif skoru sıralaması
    └── Dashboard: Açık görev sayısı, tamamlama trendi
```

### 3.3 İnisiyatif Skor Sistemi

Fabrikadaki 5 bileşenli skor sisteminden esinlenerek, şube çalışanları için **görev performans skoru**:

| Bileşen | Ağırlık | Açıklama |
|---------|---------|----------|
| Görev tamamlama | %40 | Atanan görevlerin tamamlanma oranı |
| İnisiyatif | %25 | Açık havuzdan alınan görev sayısı |
| Zamanında tamamlama | %20 | Deadline'a uyum oranı |
| Kalite | %15 | Tamamlama kalitesi değerlendirmesi |

**İnisiyatif puan tablosu:**

| Aksiyon | Puan | Koşul |
|---------|------|-------|
| Açık görev alma (claim) | +5 | is_open_pool task'ı sahiplenme |
| Erken tamamlama | +3 | Deadline'dan 2+ saat önce |
| Zamanında tamamlama | +1 | Deadline'a kadar |
| Gecikmeli tamamlama | 0 | Deadline sonrası |
| Hiç tamamlamama | -3 | 24 saat geçtikten sonra |
| Kalite bonusu | +2 | completion_quality = 'excellent' |

**Hesaplama:** Haftalık otomatik (Pazartesi 04:00, fabrika skorundan sonra)

## 7. API ENDPOİNT TASARIMI

### 7.1 Yeni Endpoint'ler

```
# Şube takvimi yönetimi
GET    /api/branch-tasks/schedules              — Tüm şube takvimleri

# İnisiyatif mekanizması
GET    /api/branch-tasks/open-pool/:branchId    — Açık havuz görevleri
POST   /api/branch-tasks/:taskId/claim          — İnisiyatif al
POST   /api/branch-tasks/:taskId/complete       — Görevi tamamla
POST   /api/branch-tasks/:taskId/approve        — Tamamlamayı onayla

# Skor ve raporlama
GET    /api/branch-tasks/scores/:branchId       — Şube inisiyatif skorları
GET    /api/branch-tasks/scores/user/:userId     — Kişi bazlı skor detayı
GET    /api/branch-tasks/report/weekly/:branchId — Haftalık tamamlama raporu
GET    /api/branch-tasks/report/comparison       — Şubeler arası karşılaştırma
```

## 10. SKOR → PERFORMANS ETKİSİ

### 10.1 Mevcut performans sistemi ile entegrasyon

Şu an şube çalışanlarının `employee_summary_scores` tablosunda skoru var. İnisiyatif skoru buraya ek bileşen olarak eklenir:

```
Mevcut skor bileşenleri (şube çalışanı):
  - Devam/PDKS       → %30
  - Eğitim ilerleme   → %25  
  - Checklist          → %20
  - Değerlendirme      → %15
  - [YENİ] İnisiyatif  → %10  ← EKLENİYOR
```

### 10.2 İnisiyatif bonusunun etkisi

Bir çalışanın haftada 3 inisiyatif alması (15 puan) vs. hiç almaması:
- İnisiyatif alan: toplam skora +8-10 puan katkı
- Almayan: 0 katkı

Bu fark zamanla birikerek terfi kararlarını, Mr. Dobody önerilerini ve coach değerlendirmesini etkiler.

### 11.2 TaskTrigger Service ile Entegrasyon

1. Cron zamanı gelince → template'e bak
2. Template'in branch_id'si varsa → o şube için task oluştur
3. Branch_id NULL ise → tüm şubeler için oluştur (HQ fix sonrası)

Yeni davranış:
1. `branch_task_schedules` tablosuna bak
2. Template × Branch eşleşmesi varsa → sadece o şubeler için oluştur
3. Yoksa → eski davranış (tüm şubeler veya belirli şube)
4. is_active = false ise → o şube için atla

### 11.3 Module Flag

`sube_gorevleri` flag'ı şu an `is_enabled = false`. Aşama 1'de `true` yapılacak. Frontend sayfası mevcut — menüde görünecek.
