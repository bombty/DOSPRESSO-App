## FIX 2 — app-sidebar.tsx Temizliği

**Problem:** app-sidebar.tsx (425 lines) is dead UI code kept only because 
`yonetim/menu.tsx` imports `invalidateMenuCache` from it.

**Steps:**
1. Open `client/src/components/app-sidebar.tsx`
2. Find the `invalidateMenuCache` function (or whatever is imported by yonetim/menu.tsx)
3. Create a new small utility file:

**Create:** `client/src/lib/menu-utils.ts`
```typescript
// Extracted from app-sidebar.tsx
// Menu cache invalidation utility

import { queryClient } from './query-client'; // adjust import path

export function invalidateMenuCache() {
  queryClient.invalidateQueries({ queryKey: ['/api/menu'] }); // adjust query key
}
```

4. Check the actual implementation of `invalidateMenuCache` in app-sidebar.tsx and copy it exactly to menu-utils.ts. Use the correct import paths.

5. Update `yonetim/menu.tsx` to import from the new location:
```typescript
// Before:
import { invalidateMenuCache } from '@/components/app-sidebar';

// After:
import { invalidateMenuCache } from '@/lib/menu-utils';
```

6. Verify no other file imports from app-sidebar.tsx:
```bash
grep -rn "app-sidebar" client/src --include="*.tsx" --include="*.ts"
```

7. If app-sidebar.tsx has NO remaining imports, delete it.
   If any other file still imports from it, do NOT delete — report instead.
