## TASK: Sprint 16B — Sevkiyat Detay + Şube Stok Transferi

### ÖZET

Sprint 16A'da sevkiyat temel altyapısı kuruldu (factory_shipments + factory_shipment_items, atomik stok düşme, row lock). Ama şube tarafı eksik — fabrikadan ürün çıkıyor ama şube stokuna girmiyor. Şubeler sipariş talebi oluşturamıyor. Bu sprint tam sevkiyat döngüsünü tamamlıyor.

**İş akışı hedefi:**
```
Şube sipariş talebi → Fabrika hazırlık → Sevk → Yolda → Teslim → Şube stokuna giriş
```

**KRİTİK KURALLAR:**
- Işıklar Şubesi (Bombtea'ya ait) = iç transfer (maliyet aktarımı)
- 19 franchise şube = satış (fatura kesilir)
- SKT geçmiş lot SEVKİYATA ÇIKAMAZ
- Kalite onayı olmayan ürün sevkiyata çıkamaz

### T001 — Şube Sipariş Talebi Sistemi (~2-3 saat)

**Problem:** Şubeler fabrikadan ürün talep edemiyor. Şu an sevkiyat sadece fabrika tarafından oluşturuluyor.

**A) Yeni tablo (yoksa):**
```sql
CREATE TABLE IF NOT EXISTS branch_order_requests (
  id SERIAL PRIMARY KEY,
  branch_id INTEGER REFERENCES branches(id) NOT NULL,
  requested_by VARCHAR(255) REFERENCES users(id) NOT NULL,
  status VARCHAR(20) DEFAULT 'pending', -- pending, approved, preparing, shipped, delivered, cancelled
  priority VARCHAR(20) DEFAULT 'normal', -- urgent, normal, low
  requested_delivery_date DATE,
  notes TEXT,
  approved_by VARCHAR(255) REFERENCES users(id),
  approved_at TIMESTAMP,
  shipment_id INTEGER REFERENCES factory_shipments(id), -- Sevkiyata bağlandığında
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE IF NOT EXISTS branch_order_items (
  id SERIAL PRIMARY KEY,
  order_id INTEGER REFERENCES branch_order_requests(id) NOT NULL,
  product_id INTEGER REFERENCES factory_products(id) NOT NULL,
  requested_quantity INTEGER NOT NULL,
  approved_quantity INTEGER, -- Fabrika onayladığında (stok yetersizse azaltılabilir)
  unit VARCHAR(20) NOT NULL,
  notes TEXT,
  created_at TIMESTAMP DEFAULT NOW()
);
```

**B) Sipariş endpoint'leri:**
```
POST /api/branch-orders — Yeni sipariş talebi (supervisor/müdür)
GET /api/branch-orders — Sipariş listesi (şube bazlı filtre)
GET /api/branch-orders/:id — Sipariş detay
PATCH /api/branch-orders/:id/approve — Fabrika onay (stok kontrolü + miktar ayarı)
PATCH /api/branch-orders/:id/cancel — İptal
```

**C) Sipariş akışı:**
```
1. Supervisor/Müdür → Sipariş oluştur (ürün + miktar listesi)
2. Fabrika müdürü → Onay (stok yeterliyse approved_quantity = requested_quantity)
   → Stok yetersizse: approved_quantity < requested_quantity + not
3. Onaylanan sipariş → Sevkiyat hazırlığına düşer
4. Sevkiyat oluşturulduğunda → shipment_id bağlanır
5. Teslimde → sipariş status='delivered'
```

**D) Stok yeterlilik kontrolü:**
- Onay sırasında her ürün için factory_inventory kontrol edilir
- Yetersizse: "Stokta 300 adet var, 500 talep edildi — 300 olarak onaylansın mı?"
- Tamamen yoksa: "Stokta yok — üretim planına eklensin mi?"

### T002 — Sevkiyat → Şube Stok Girişi (~2-3 saat)

**Problem:** Fabrikadan çıkan ürün şube stokuna girmiyor. Sevkiyat "teslim edildi" olduğunda şube stoku güncellenmiyor.

**A) Şube stok tablosunu kontrol et:**
```bash
# Şube stok tablosu var mı?
grep -rn "branch.*stock\|branch.*inventory\|sube.*stok\|branch_inventory" server/ --include="*.ts" | head -10
```

```sql
-- Şube stok tablosu
SELECT column_name, data_type FROM information_schema.columns 
WHERE table_name ILIKE '%branch_stock%' OR table_name ILIKE '%branch_inventory%'
ORDER BY ordinal_position;
```

Yoksa oluştur:
```sql
CREATE TABLE IF NOT EXISTS branch_inventory (
  id SERIAL PRIMARY KEY,
  branch_id INTEGER REFERENCES branches(id) NOT NULL,
  product_id INTEGER REFERENCES factory_products(id) NOT NULL,
  current_stock DECIMAL(10,2) DEFAULT 0,
  minimum_stock DECIMAL(10,2) DEFAULT 5,
  unit VARCHAR(20),
  last_received_at TIMESTAMP,
  last_counted_at TIMESTAMP,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW(),
  UNIQUE(branch_id, product_id)
);

CREATE TABLE IF NOT EXISTS branch_stock_movements (
  id SERIAL PRIMARY KEY,
  branch_id INTEGER REFERENCES branches(id) NOT NULL,
  product_id INTEGER REFERENCES factory_products(id) NOT NULL,
  movement_type VARCHAR(30) NOT NULL, -- 'sevkiyat_giris', 'satis', 'zayiat', 'sayim_duzeltme', 'iade'
  quantity DECIMAL(10,2) NOT NULL,
  previous_stock DECIMAL(10,2),
  new_stock DECIMAL(10,2),
  reference_type VARCHAR(50), -- 'shipment', 'daily_sales', 'waste', 'count'
  reference_id INTEGER,
  lot_number VARCHAR(50), -- Hangi lot'tan geldi (izlenebilirlik)
  expiry_date TIMESTAMP, -- O lot'un SKT'si
  notes TEXT,
  created_by VARCHAR(255) REFERENCES users(id),
  created_at TIMESTAMP DEFAULT NOW()
);
```

**B) Teslim akışı:**

Sevkiyat "teslim edildi" olarak işaretlendiğinde (mevcut endpoint güncelle):

```
1. factory_shipments.status = 'delivered'
2. Her shipment_item için:
   → branch_inventory'de o ürün var mı? Yoksa oluştur.
   → branch_inventory.current_stock += quantity
   → branch_stock_movements'a kayıt (movement_type: 'sevkiyat_giris')
   → lot_number ve expiry_date bilgisi shipment_item'dan alınır
3. İlgili branch_order_request varsa status='delivered' yap
4. Şube supervisor'a bildirim: "Sevkiyat teslim alındı: X ürün, Y adet"
```

**C) Tüm işlem tek transaction içinde:**
```typescript
await db.transaction(async (tx) => {
  // shipment status güncelle
  // her item için branch_inventory güncelle
  // branch_stock_movements kayıt
  // sipariş status güncelle
  // bildirim gönder
});
```

### T006 — SKT Sevkiyat Kontrolü (~30 dakika)

**Mevcut sevkiyat oluşturma endpoint'inde SKT kontrolü ekle:**

```typescript
// Sevkiyat oluşturulurken her item için:
for (const item of items) {
  // LOT'un SKT'sini kontrol et
  const lot = await tx.select().from(productionLots)
    .where(and(
      eq(productionLots.productId, item.productId),
      eq(productionLots.status, 'in_stock'),
      gt(productionLots.expiryDate, new Date()) // SKT geçmemiş olmalı
    ));
  
  if (!lot.length) {
    throw new Error(`${item.productName} için geçerli SKT'li stok bulunamadı`);
  }
}

// Kalite onayı kontrolü
// quality_status = 'approved' olmayanlar sevkiyata çıkamaz
```

**Ayrıca:** Sevkiyat item'larına otomatik lot atama:
- FIFO (First In First Out): En eski SKT'li lot'tan başla
- Lot bilgisi shipment_item'a kaydedilsin (lot_number, expiry_date)
- Şube tesliminde aynı lot bilgisi branch_stock_movements'a geçsin

### T007 — Bombtea vs Franchise Ayrımı (~1 saat)

**Problem:** Işıklar şubesi (Bombtea'ya ait) ile franchise şubeleri farklı muhasebe işlemi gerektirir.

**Yapılacak:**

1. branches tablosunda franchise_type alanı kontrol et:
```bash
grep -rn "franchise\|ownership\|branch_type\|is_franchise" shared/schema.ts | head -10
```
```sql
SELECT column_name, data_type FROM information_schema.columns 
WHERE table_name = 'branches' AND (column_name ILIKE '%type%' OR column_name ILIKE '%franchise%');
```

Yoksa ekle:
```sql
ALTER TABLE branches ADD COLUMN IF NOT EXISTS ownership_type VARCHAR(20) DEFAULT 'franchise';
-- 'bombtea' = Işıklar (kendi şubemiz)
-- 'franchise' = Bayi şubeleri
```

2. Işıklar şubesini 'bombtea' olarak işaretle:
```sql
UPDATE branches SET ownership_type = 'bombtea' WHERE name ILIKE '%ışıklar%' OR name ILIKE '%isiklar%';
```

3. Sevkiyat oluşturulduğunda:
   - Franchise şubeye → transfer_type: 'sale' (satış fiyatından)
   - Bombtea şubeye → transfer_type: 'internal' (maliyet fiyatından)

4. factory_shipments tablosuna ekle:
```sql
ALTER TABLE factory_shipments ADD COLUMN IF NOT EXISTS transfer_type VARCHAR(20) DEFAULT 'sale';
ALTER TABLE factory_shipments ADD COLUMN IF NOT EXISTS total_cost DECIMAL(12,2);
ALTER TABLE factory_shipments ADD COLUMN IF NOT EXISTS total_sale_price DECIMAL(12,2);
```

5. Raporlarda ayrım:
   - "Franchise Satış" raporu: franchise şubelere gönderilen ürünlerin toplam tutarı
   - "İç Transfer" raporu: Işıklar'a gönderilen ürünlerin maliyeti
