========================
1) DATA MODEL (DB)
========================
A) Add tables (drizzle schema + migration):
1) waste_reason_categories
- id, code (demand_ops_logistics_factory_customer), name_tr, name_en, sortOrder, isActive default true

2) waste_reasons
- id, categoryId FK, code, name_tr, name_en, isActive default true

3) waste_events
- id
- branchId (required)
- createdByUserId (required)
- occurredAt timestamp (required)
- productSku or productId (whichever exists in system; if neither, use freeTextProductName)
- quantity number (required) + unit (adet/kg)
- unitCost (optional now) + currency (default TRY)
- lotNumber (optional but strongly encouraged)
- productionDate (optional)
- expiryDate (optional)
- deliveryDate (optional)
- storageType enum (-18,+4,room) optional
- primaryReasonId FK (required)
- secondaryReasonId FK optional
- attribution enum (demand, operations, logistics, factory) optional
- note text (min 10 chars on UI)
- status enum (open, reviewed, actioned, closed) default open

4) waste_attachments
- id, wasteEventId FK, uploadedByUserId FK
- type enum (photo, doc)
- objectStorageKey/url (use existing object storage)
- createdAt

5) corrective_actions
- id
- scope enum (branch, hq, factory)
- branchId nullable
- ownerRole enum/string (coach/trainer/qa/factory_manager/supervisor)
- ownerUserId nullable
- title, description
- dueAt, status enum (open, in_progress, done, canceled)
- relatedWasteEventIds jsonb (or junction table if preferred minimal)
- createdByUserId, createdAt, closedAt

B) Seed data (minimal):
- Seed 4 categories (Demand/Merch, Branch Ops, Logistics Cold Chain, Factory Quality) TR+EN
- Seed ~20 reasons total (5 per category) TR+EN (keep extensible)

2) RULES (Deterministic, NO AI)
========================
Create minimal rules service for data quality:
- Waste event cannot be submitted without:
  - primaryReasonId
  - quantity + unit
  - note length >= 10
  - at least 1 photo attachment (required)
- If lotNumber missing and category=Factory/Logistics: WARN but allow submit.

Return: {severity: warn/block, message, fields[]}

3) API ROUTES
========================
Add routes under /api/waste:
- POST /events (create draft)
- POST /events/:id/submit (runs rules; blocks if missing required)
- GET /events?branchId=&from=&to=&status= (RBAC scoped)
- GET /events/:id (RBAC scoped)
- POST /events/:id/attachments (upload using existing objectStorage; returns attachment)
- POST /corrective-actions (create)
- PATCH /corrective-actions/:id (status updates)
- GET /insights?scope=branch|hq|factory&from=&to=
  Insights v0 (deterministic):
  - Top products by waste count
  - Top reasons
  - Branch ranking (HQ only)
  - Lot clusters (same lotNumber across branches) (HQ/QA only)

Audit log events:
- waste.event_created, waste.event_submitted, waste.attachment_added
- waste.corrective_action_created, waste.corrective_action_updated, waste.corrective_action_closed

5) LEAD AGENT v0 (No AI decisions)
========================
Implement “Signals → Tasks” deterministic generator:
- Signal rules (daily at 09:00 TR + on dashboard load):
  S1: If branch waste count > threshold (config) in last 7 days → create task for Coach
  S2: If same lotNumber appears in >=2 branches → create task for QA/Factory manager
  S3: If corrective action overdue → escalate to CGO (task)
- These create tasks in existing tasks/action-cards system (use existing task creation APIs).
- No auto-close. Only assigns + tracks.

Audit log:
- agent.signal_detected, agent.task_created, agent.escalated
