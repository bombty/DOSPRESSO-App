## TASK: Sprint 21A — Mr. Dobody Flow Mode

Mr. Dobody currently sends suggestions that users can dismiss. This sprint transforms Mr. Dobody from a "notification bot" into a "work flow guide" that proactively leads users through their daily tasks step by step.

### CONCEPT

When a user logs in, instead of just showing a static dashboard, Mr. Dobody greets them and presents their prioritized task list. The user taps "Start" and Mr. Dobody guides them through each task sequentially. When one task is done, Mr. Dobody presents the next one. When all tasks are complete, Mr. Dobody congratulates them.

This is NOT a chatbot. This is a guided workflow overlay on existing landing pages.

### T001 — Flow Mode Component (~4 hours)

Create a new reusable component: `client/src/components/dobody-flow-mode.tsx`

**Props:**
```typescript
interface FlowModeProps {
  userId: string;
  userRole: string;
  userName: string;
  branchId?: string;
}
```

**Component behavior:**

1. On mount, fetch today's task list from a new API endpoint (T002)
2. Display a greeting card at the TOP of the landing page (above existing content):

```
┌─────────────────────────────────────────┐
│ 🤖 İyi Günler, Cihan!                   │
│                                          │
│ Bugün 3 işin var:                        │
│                                          │
│ 1. ☐ Açılış Checklist         ~5 dk     │
│ 2. ☐ Cold Brew Eğitimi        ~5 dk     │
│ 3. ☐ Stok Sayım Bildirimi     ~2 dk     │
│                                          │
│        [ İlk İşe Başla → ]              │
│                                          │
│ ─── veya ───                             │
│ Kendi başıma devam etmek istiyorum →     │
└─────────────────────────────────────────┘
```

3. When user taps "İlk İşe Başla":
   - Navigate to the relevant page for task #1 (e.g., /checklists for checklist task)
   - Show a floating mini-bar at the bottom of the screen (above bottom nav):
   ```
   ┌──────────────────────────────────────┐
   │ 🤖 Görev 1/3: Açılış Checklist       │
   │                     [ Tamamlandı ✓ ] │
   └──────────────────────────────────────┘
   ```

4. When user taps "Tamamlandı" or the task is auto-detected as complete:
   - Mini-bar updates: "Harika! Sıradaki: Cold Brew Eğitimi"
   - After 2 seconds, navigate to next task's page
   - If user is already on the correct page, just update the mini-bar

5. When ALL tasks are complete:
   - Show completion card:
   ```
   ┌─────────────────────────────────────────┐
   │ 🎉 Bugünkü görevlerini tamamladın!      │
   │                                          │
   │ ⭐ 8 gün streak!                        │
   │ 📊 Skor: 78/100                         │
   │                                          │
   │ Başka bir şey yapmak ister misin?        │
   │ [ Mr. Dobody'ye Sor ]  [ Kapat ]        │
   └─────────────────────────────────────────┘
   ```

6. "Kendi başıma devam etmek istiyorum" tapped:
   - Dismiss the flow mode card
   - Save preference to localStorage: `dobody_flow_dismissed_${date}`
   - Show normal landing page without flow overlay
   - Next day, flow mode appears again

**Visual design:**
- Card has subtle gradient background (brand colors)
- Mr. Dobody avatar (small robot icon) on the left
- Tasks listed with checkbox style
- Estimated time shown for each task
- Smooth animations for transitions
- Dark mode compatible
- Mobile-first (works at 375px)

### T002 — Flow Mode API Endpoint (~3 hours)

Create: `GET /api/dobody/flow-tasks`

**Response:**
```json
{
  "greeting": "İyi Günler, Cihan!",
  "tasks": [
    {
      "id": "task_1",
      "title": "Açılış Checklist",
      "description": "Günlük açılış checklistini tamamla",
      "estimatedMinutes": 5,
      "navigateTo": "/checklists",
      "type": "checklist",
      "isCompleted": false,
      "autoDetect": true
    },
    {
      "id": "task_2", 
      "title": "Cold Brew Eğitimi",
      "description": "Bugünün önerilen eğitim modülü",
      "estimatedMinutes": 5,
      "navigateTo": "/egitim",
      "type": "training",
      "isCompleted": false,
      "autoDetect": true
    }
  ],
  "completedToday": 0,
  "streak": 8,
  "score": 78
}
```

**Task generation logic per role:**

**Barista / Bar Buddy / Stajyer:**
1. Uncompleted daily checklist → "Açılış/Kapanış Checklist"
2. AI recommended training module (daily-recommendation endpoint) → "Bugünün Eğitimi"
3. Pending tasks assigned to user → "Atanan Görevler"
4. If streak milestone approaching → "Streak'ini koru — 1 modül daha!"

**Supervisor / Supervisor Buddy:**
1. Uncompleted daily checklist → "Açılış Checklist"
2. Pending task approvals → "X onay bekliyor"
3. Staff with low attendance → "Devamsız personel kontrol"
4. Low stock items → "Stok uyarılarını incele"
5. Unanswered customer complaints → "X müşteri yanıt bekliyor"

**Müdür:**
1. Same as supervisor +
2. Weekly report review → "Haftalık raporu incele"
3. Staff evaluations due → "Performans değerlendirmesi"

**CEO / CGO:**
1. Branch alerts (kritik/dikkat) → "X şube dikkat gerektiriyor"
2. SLA breaches → "X SLA ihlali"
3. Weekly performance review → "Haftalık performans özeti"
4. Mr. Dobody priority suggestions → top 2 suggestions as tasks

**Coach:**
1. Overdue branch visits → "X şubeyi ziyaret et"
2. Staff with critical scores → "Y personel dikkat gerektiriyor"
3. Training gaps → "Z şubede eğitim gecikmiş"

**Trainer:**
1. Pending module approvals → "X modül onay bekliyor"
2. Low quiz pass rates → "Y modülde başarı oranı düşük"
3. Content review needed → "Z içerik gözden geçirilmeli"

**Fabrika Müdür:**
1. Daily production target → "Bugünkü üretim hedefi: X kg"
2. Pending quality checks → "Y kalite kontrol bekliyor"
3. Staff attendance → "Z personel gelmedi"
4. Low stock warnings → "Hammadde stok uyarısı"

**Muhasebe:**
1. Pending payroll calculations → "X şubenin bordrosu hesaplanmadı"
2. Missing PDKS entries → "Y eksik PDKS kaydı"
3. Upcoming deadlines → "Bordro teslim tarihi Z gün kaldı"

**For other roles (kalite_kontrol, gida_muhendisi, satinalma, marketing, etc.):**
Use the Dobody suggestions already created in Sprint 20B and convert top 3 suggestions to flow tasks.

**Auto-detection of task completion:**
- Checklist type: Check if user completed today's checklist via API
- Training type: Check if user completed a module today via training_completions
- Task type: Check if assigned task status changed to completed
- Approval type: Check if pending count decreased

Poll every 30 seconds OR use the existing page navigation to re-check.

### T004 — Flow Mode Mini-Bar (Floating Task Indicator) (~2 hours)

When user navigates away from landing page to complete a task, show a floating mini-bar:

Create: `client/src/components/dobody-mini-bar.tsx`

**Behavior:**
- Fixed position at bottom of screen, above bottom nav (z-index above content, below modals)
- Shows current task: "🤖 Görev 1/3: Açılış Checklist [Tamamlandı ✓]"
- Tapping "Tamamlandı" marks task complete and shows next task
- Tapping the bar text navigates back to landing page
- Auto-hides after all tasks complete (with celebration animation)
- Dismissible with swipe-down gesture

**State management:**
- Use React Context: `DobodyFlowContext`
- Provider wraps the main App layout
- Stores: currentTasks, activeTaskIndex, isFlowActive, isMinimized

```tsx
// In main layout wrapper
<DobodyFlowProvider>
  <AppLayout>
    {children}
  </AppLayout>
  <DobodyMiniBar />  {/* renders above bottom nav when flow is active */}
</DobodyFlowProvider>
```

**Mini-bar states:**
1. Active task: Blue/brand color, shows task title + "Tamamlandı ✓" button
2. Transitioning: Green flash, "Harika! Sıradaki: [next task]", auto-navigate after 2s
3. All complete: Green, "🎉 Bugünlük tamamladın!", auto-dismiss after 5s
4. Minimized: Small floating dot with task count badge (user can tap to expand)
