## TASK: Sprint 14 — Offline Resilience & WiFi Kesinti Koruma

### ÖZET

Şubelerde WiFi bazen kesiliyor. Barista görev tamamlıyor, quiz çözüyor, form dolduruyor — tam gönder'e basarken internet kesilirse veri kayboluyor. Bu sprint tüm kritik akışlara retry mekanizması, offline algılama ve veri kurtarma ekliyor.

### T002 — API İstek Retry Mekanizması (~2-3 saat)

**Problem:** Tek bir API isteği başarısız olunca (network error, timeout) kullanıcıya hata gösteriliyor ve veri kayboluyor.

**Yapılacak:**

1. Global fetch wrapper / React Query retry konfigürasyonu:

```bash
# Mevcut React Query konfigürasyonunu bul
grep -rn "QueryClient\|queryClient\|defaultOptions\|retry" client/src/ --include="*.tsx" --include="*.ts" | head -15
```

2. React Query retry ayarları:
```typescript
const queryClient = new QueryClient({
  defaultOptions: {
    queries: {
      retry: 3,                    // 3 kez dene
      retryDelay: (attempt) => Math.min(1000 * 2 ** attempt, 10000), // exponential backoff
      networkMode: 'offlineFirst', // offline'da cache'ten göster
      staleTime: 5 * 60 * 1000,   // 5 dk cache
    },
    mutations: {
      retry: 3,
      retryDelay: (attempt) => Math.min(1000 * 2 ** attempt, 10000),
      networkMode: 'offlineFirst',
    },
  },
});
```

3. Mutation (POST/PUT/DELETE) için özel retry:
- Network error → otomatik 3 kez dene (exponential backoff: 1s, 2s, 4s)
- 500 error → 2 kez dene
- 400/401/403 → retry YAPMA (kullanıcı hatası, tekrar denemek anlamsız)
- Timeout (10s) → 2 kez dene

4. Retry sırasında kullanıcıya gösterge:
```
"Gönderiliyor..." → "Bağlantı sorunu, tekrar deneniyor (2/3)..." → Başarılı veya "Gönderim başarısız — tekrar dene"
```

### T003 — Offline Mutation Queue (Kritik Veri Kurtarma) (~2-3 saat)

**Problem:** Retry bile başarısız olursa (internet tamamen kesilmiş), kullanıcının girdiği veri tamamen kaybolur.

**Yapılacak:**

1. Başarısız mutation'ları localStorage kuyruğuna kaydet:

```typescript
// client/src/lib/offline-queue.ts

interface QueuedMutation {
  id: string;
  url: string;
  method: 'POST' | 'PUT' | 'PATCH' | 'DELETE';
  body: any;
  timestamp: number;
  retryCount: number;
  description: string; // "Görev tamamlama", "Quiz cevabı", "İzin talebi"
}

// Kuyruğa ekle
function addToQueue(mutation: QueuedMutation): void

// Kuyruktan gönder (internet gelince)
function processQueue(): Promise<void>

// Kuyruk durumu
function getQueueSize(): number
function getQueueItems(): QueuedMutation[]
function clearQueue(): void
```

2. İnternet geldiğinde otomatik kuyruk işleme:
```typescript
window.addEventListener('online', () => {
  processQueue(); // Bekleyen tüm mutation'ları sırayla gönder
});
```

3. Kuyrukta bekleyen öğe varsa bottom nav'da badge göster:
```
[📤 3] — 3 bekleyen gönderim
```

4. Kuyruk işleme sırasında progress göster:
```
"Bekleyen veriler gönderiliyor... (2/3)"
→ Başarılı: "Tüm veriler başarıyla gönderildi ✅"
→ Hâlâ hata: "1 gönderim hâlâ bekliyor — internet bağlantınızı kontrol edin"
```

5. 24 saatten eski kuyruk öğelerini temizle (artık geçerliliğini yitirmiş olabilir)

**KRİTİK:** Quiz cevapları kuyruğa eklenmemeli — anti-cheat sistemiyle çakışır. Quiz sırasında internet kesilirse sınav iptal edilmeli (mevcut davranış korunur).

### T004 — Kritik Akışlarda Offline Koruma (~2 saat)

Her kritik kullanıcı akışında offline durumu kontrol et ve uygun davran:

**A) Görev Tamamlama:**
- Gönder basılınca internet yoksa → kuyruğa ekle + "İnternet gelince gönderilecek" mesajı
- Görev kartında "Beklemede 📤" badge göster

**B) Checklist Tamamlama:**
- Aynı mantık — kuyruğa ekle
- Tamamlama durumunu local state'te işaretle, gönderilince server'a senkronize

**C) İzin Talebi:**
- Form dolduruldu, gönder basıldı, internet yok → kuyruğa ekle
- "Talebiniz internet gelince gönderilecek" mesajı

**D) Arıza Bildirimi:**
- Fotoğraf dahil kuyruğa ekle (localStorage boyut sınırına dikkat — fotoğrafları küçült veya IndexedDB kullan)
- "Arıza bildiriminiz internet gelince gönderilecek"

**E) Quiz/Sınav:**
- Quiz sırasında internet kesilirse: "İnternet bağlantısı kesildi. Sınavınız geçersiz sayılacak." uyarısı
- Quiz cevapları KUYRUKLANMAZ — anti-cheat prensipleriyle çakışır
- Sınav iptal edilir, kullanıcı daha sonra tekrar deneyebilir

**F) Misafir Geri Bildirim (Public Form):**
- Müşteri formu doldurup gönder'e bastıysa → kuyruğa ekle
- "Teşekkürler! Değerlendirmeniz internet gelince iletilecek." mesajı
- Bu çok önemli — müşteri feedback'ini kaybetmek kabul edilemez

**G) Mesaj Gönderme:**
- Mesaj kuyrukta → "Gönderiliyor..." göstergesi
- İnternet gelince otomatik gönderilir

**Her akış için:**
```bash
# İlgili mutation hook'ları bul
grep -rn "useMutation\|mutation" client/src/ --include="*.tsx" | grep -i "task\|gorev\|checklist\|leave\|izin\|fault\|ariza\|feedback\|message\|mesaj" | head -20
```

Her mutation hook'una offline fallback ekle:
```typescript
const mutation = useMutation({
  mutationFn: submitTask,
  onError: (error) => {
    if (isNetworkError(error)) {
      addToQueue({
        url: '/api/tasks/123/complete',
        method: 'POST',
        body: { ... },
        description: 'Görev tamamlama'
      });
      toast.info("İnternet gelince gönderilecek");
    } else {
      toast.error(error.message);
    }
  }
});
```

**Yapılacak:**

1. Mevcut Service Worker'ı kontrol et:
```bash
grep -rn "service.*worker\|serviceWorker\|sw.js\|workbox" client/src/ --include="*.ts" --include="*.tsx" --include="*.js" | head -10
cat client/public/sw.js 2>/dev/null | head -30
```

2. Service Worker stratejisi güncelle:
- HTML shell: cache-first (her zaman gösterilsin)
- API response'ları: network-first, cache fallback
- Static assets (JS, CSS, images): cache-first

3. Offline'da sayfa yenilendiğinde:
- App shell yüklensin (sidebar, bottom nav, header)
- Son cache'lenen veri gösterilsin
