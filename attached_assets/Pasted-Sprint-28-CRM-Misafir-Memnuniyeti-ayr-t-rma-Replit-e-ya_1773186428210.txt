## TASK: Sprint 28 — CRM & Guest Feedback Module Separation

The CRM module currently mixes 3 different concepts: guest satisfaction (QR ratings), business relationships (franchise investors), and support tickets (branch requests to HQ). These must be separated into distinct modules.

### CURRENT STATE (Broken Logic)

```
CRM Module (currently):
  ├── Dashboard → shows guest ratings + campaign metrics (MIXED)
  ├── Geri Bildirimler → guest QR feedback (WRONG PLACE)
  ├── Şikayetler → guest complaints + product complaints (MIXED)
  ├── Kampanyalar → marketing campaigns (CORRECT)
  ├── Analizler → mixed analytics (MIXED)
  └── SLA Takibi → feedback response SLA (WRONG PLACE)
```

### TARGET STATE (Clean Separation)

```
Module 1: Misafir Memnuniyeti (Guest Satisfaction) — SEPARATE
  ├── Dashboard → avg ratings, trend, branch comparison
  ├── Geri Bildirimler → QR/Google feedback list + responses
  ├── Misafir Şikayetleri → auto-created from low ratings
  ├── SLA Takibi → response time tracking
  ├── Form Ayarları → QR form config (colors, fields, languages)
  └── Analizler → satisfaction trends, category breakdown

Module 2: CRM (Business Relations) — CLEANED
  ├── Dashboard → campaign metrics, investor relations
  ├── Kampanyalar → campaign management
  ├── Ticket/Talepler → branch-to-HQ requests (renamed from Şikayetler)
  ├── Ürün Şikayetleri → product quality complaints from branches
  └── Analizler → campaign performance, business analytics

Module 3: /misafir-memnuniyeti page → already exists, becomes the main entry
```
