## A1: QR Geri Bildirim → Ticket Oluşturma

Mevcut QR geri bildirim formunu bul ve ticket oluşturacak şekilde güncelle:

**Akış:**
1. Müşteri QR okutup forma girer
2. Form: yıldız puanları (Hizmet, Temizlik, Ürün, Personel) + yorum + isteğe bağlı iletişim bilgisi
3. Submit → POST /api/crm/tickets (veya mevcut /api/iletisim/tickets)
4. Otomatik alanlar:
   ```
   channel: 'misafir'
   ticket_type: 'musteri_geri_bildirim'
   source: 'qr'
   priority: rating_avg <= 2 ? 'high' : rating_avg <= 3 ? 'medium' : 'low'
   ```
5. SLA otomatik atama:
   - Puan ≤2 → SLA: 4 saat (acil müdahale)
   - Puan 3 → SLA: 24 saat (takip)
   - Puan ≥4 → SLA: 72 saat (teşekkür mesajı)

**⚠ Eğer mevcut QR form customer_feedback tablosuna yazıyorsa → support_tickets'a yazacak şekilde değiştir.**

## A2: Misafir Kanalı Ticket Oluşturma Formu

CRM sayfasında Misafir kanalı seçiliyken "Yeni" butonuna basınca açılacak form:

- Şube seçimi (dropdown)
- Yıldız puanları: Hizmet (1-5), Temizlik (1-5), Ürün (1-5), Personel (1-5)
- Yorum/açıklama (textarea)
- Müşteri bilgileri (opsiyonel): Ad, telefon, email
- Kaynak: Manuel (HQ tarafından girilen)
- Submit → ticket oluştur, channel: 'misafir', source: 'manual'

## B1: Bildirim Arşivleme (⚠ DANIŞMA — 30K+ kayıt)

### B1a: Archive tablosu oluştur

```sql
-- notifications ile aynı yapıda
CREATE TABLE IF NOT EXISTS notifications_archive (
  LIKE notifications INCLUDING ALL
);
CREATE INDEX IF NOT EXISTS idx_notif_archive_user ON notifications_archive(user_id, created_at DESC);
```

### B1b: 30 günden eski bildirimleri arşive taşı (⚠ DANIŞMA)

```sql
-- Önce sayıyı kontrol et
SELECT count(*) FROM notifications WHERE created_at < NOW() - INTERVAL '30 days';

-- Batch olarak taşı (10K'lık gruplar halinde)
WITH moved AS (
  DELETE FROM notifications 
  WHERE id IN (
    SELECT id FROM notifications 
    WHERE created_at < NOW() - INTERVAL '30 days' 
    ORDER BY id LIMIT 10000
  )
  RETURNING *
)
INSERT INTO notifications_archive SELECT * FROM moved;

-- Tekrarla ta ki eski kayıt kalmayana kadar
```

### B1c: Otomatik arşivleme scheduler'ı

Mevcut scheduler sistemine ekle:
```typescript
// Her gece 03:00'te çalışır
// 30 günden eski okunmuş bildirimleri archive'a taşır
// Okunmamış bildirimleri 90 güne kadar tutar
```
