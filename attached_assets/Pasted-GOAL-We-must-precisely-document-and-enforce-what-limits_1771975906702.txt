GOAL
We must precisely document and enforce what limits/rules Mr. Dobody follows (role-based answers, strict data isolation between Branch vs Factory vs HQ, no leaking other employees’ private data, and no leaking HQ/Factory sensitive info to branches). Determine the current implementation and produce a concise “Dobody Security & Scope Spec” + minimal fixes if gaps exist.

NON-NEGOTIABLE SECURITY RULES
1) Backend is the source of truth. UI/prompt-only restrictions are not considered secure.
2) Branch users must never receive HQ/Factory sensitive data (e.g., purchase prices, supplier contracts, production batch details, recipes, inventory beyond their scope).
3) A user must never receive another person’s private/identifiable data unless explicitly allowed for their role (e.g., supervisor may see team aggregates but not personal identifiers unless needed).
4) If Dobody uses any LLM, it must only receive already-scoped, minimal data (least privilege) and must not include PII.
