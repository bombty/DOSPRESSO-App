## Module Feature Flag System — Sprint 2: Admin Toggle Panel UI

### Context
Sprint 1B+1C created the module_flags backend (31 flags, 4-level lookup, role-based, 3 behavior types). Now we need an admin UI to toggle flags without using the API directly.

Read these skills first:
- .agents/skills/dospresso-architecture/SKILL.md
- .agents/skills/dospresso-quality-gate/SKILL.md

### Design Requirements
- New page at /admin/module-flags (add to admin routes in App.tsx)
- Add to sidebar for admin role in menu-service.ts (under Sistem group)
- Follow existing DOSPRESSO admin page patterns — use Shadcn/ui Card, Switch, Badge, Select, Table components
- Turkish UI, dark mode compatible, no hardcoded colors
- Mobile responsive
- No emoji — use Lucide icons only

### Page Layout

#### Top Section: Scope Selector
- Dropdown (Shadcn Select): "Global Ayarlar" (default) + list of all branches (fetch from /api/branches)
- When "Global Ayarlar" selected → show global flags with toggle switches
- When a branch selected → show that branch's effective flags (global default + branch overrides)

#### Main Section: Flag Cards grouped by category

**Group 1: "Sistem (Kapatılamaz)"** — gray/muted section
- Show admin, dashboard, bordro, dobody, fabrika, satinalma
- Each shows a Lock icon (Lucide) + "Her zaman aktif" badge (green)
- Toggle switch is DISABLED (grayed out) — cannot be toggled
- Purpose: admin sees what exists but can't break core modules

**Group 2: "Veri Toplama Devam Eder"** — amber/warning styled header
- Show pdks, vardiya, fabrika.vardiya
- Toggle switch enabled
- When OFF: show amber badge "UI Gizli — Veri Toplanıyor"
- When ON: show green badge "Aktif"
- Info tooltip: "Bu modül kapatıldığında kullanıcılar görmez ama arka planda veri toplanmaya devam eder"

**Group 3: "Şube Modülleri"** — main section
- Show checklist, gorevler, akademi, crm, stok, ekipman, denetim, iletisim_merkezi, raporlar, finans, delegasyon, franchise
- Toggle switch enabled
- When OFF: show red badge "Kapalı"
- When ON: show green badge "Aktif"

**Group 4: "Fabrika Alt-Modülleri"** — nested under fabrika
- Show fabrika.sevkiyat, fabrika.sayim, fabrika.hammadde, fabrika.siparis, fabrika.kalite, fabrika.kavurma, fabrika.stok
- Each indented with a tree-line visual to show parent relationship
- Toggle switch enabled
- Note at top: "Ana fabrika modülü her zaman aktiftir. Alt-modüller bağımsız kapatılabilir."

**Group 5: "Mr. Dobody"** — nested under dobody
- Show dobody.chat, dobody.bildirim, dobody.flow
- Each indented with tree-line visual
- Toggle switch enabled
- Note: "AI arka plan analizleri her zaman çalışır. Burada sadece kullanıcıya görünen özellikler kapatılır."

#### Branch Override Section (only when a branch is selected)
- Show all flags with TWO columns:
  - "Global Durum" — the global default (read-only, shown as muted text/badge)
  - "Şube Override" — toggle switch
    - If no override exists: show "Global ayarı kullanıyor" with a dashed border toggle
    - If override exists: show active toggle (green/red)
    - "Override Kaldır" button (small, muted) — calls DELETE /api/module-flags/:id to remove branch override and fall back to global
- When admin toggles a branch override that doesn't exist yet → POST /api/module-flags to create it
- When admin toggles an existing override → PATCH /api/module-flags/:id

#### Role-Based Overrides Section (collapsible, at bottom)
- Title: "Rol Bazlı Özelleştirmeler"
- Collapsible accordion (default collapsed)
- Inside: mini-form to create role override:
  - Select module (dropdown of all toggleable modules)
  - Select role (dropdown of all 27 roles)
  - Select scope: Global / specific branch
  - Toggle: Enabled/Disabled
  - "Override Oluştur" button
- Below the form: table of existing role overrides with delete button
- This is an advanced feature — keep it simple, no inline editing needed

### Technical Implementation

#### Frontend: client/src/pages/admin/module-flags.tsx
- Use useQuery to fetch GET /api/module-flags (global view) or GET /api/module-flags/branch/:id (branch view)
- Use useMutation for PATCH, POST, DELETE operations
- After each mutation: invalidateQueries for module-flags
- Use Shadcn Switch component for toggles
- Use Shadcn Badge for status indicators
- Use Shadcn Card for each group
- Use Shadcn Select for branch selector and role selector
- Use Shadcn Accordion for role overrides section
- Loading state: skeleton cards while data loads
- Error state: "Modül bayrakları yüklenirken hata oluştu" with retry button

### Testing checklist:
1. App starts without errors
2. Navigate to /admin/module-flags as admin — page loads with all 31 flags grouped correctly
3. "Sistem" group shows 6 always_on modules with disabled switches
4. Toggle a module off (e.g. akademi) → verify PATCH works, badge changes to "Kapalı"
5. Toggle it back on → badge changes to "Aktif"
6. Select a branch from dropdown → view changes to branch mode with global defaults shown
7. Create a branch override by toggling → verify POST creates the override
8. Click "Override Kaldır" → verify DELETE removes it, falls back to global
9. Open "Rol Bazlı Özelleştirmeler" → create a role override → verify it appears in table
10. Check dark mode — all elements readable
11. Check mobile — layout doesn't break
