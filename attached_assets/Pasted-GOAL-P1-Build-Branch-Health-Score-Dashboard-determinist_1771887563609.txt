GOAL (P1): Build “Branch Health Score Dashboard” (deterministic) with API + UI.
We will NOT touch auth/login. Focus only on data + endpoint + page + nav + QA.

REQUIREMENTS
A) Scoring model (deterministic, no LLM):
- Output per branch: totalScore 0-100, components[], trend, riskFlags[], and drilldown links.
- Components (5):
  1) Inspections / checklists completion (weight 25)
  2) Complaints rate (weight 25)
  3) Equipment faults / downtime (weight 20)
  4) Training / academy performance (weight 15)
  5) Ops hygiene signals (inventory gaps / missed tasks) (weight 15)
- Each component returns 0-100, then weighted sum.
- Trend: compare last period vs previous equal period (e.g., 30d vs previous 30d) => up/down/flat.
- Risk flags: e.g., “High complaints”, “Low checklist compliance”, “Frequent faults”, “Low training score”.

B) API endpoint:
- Add GET /api/reports/branch-health?range=7d|30d|90d (default 30d)
- Auth required (reuse existing isAuthenticated / localAuth).
- Role scoping:
  - Branch roles see ONLY their own branch.
  - HQ/admin/exec roles can see all branches (optionally filter by branchId).
- Response shape:
{
  range: "30d",
  generatedAt: ISO,
  branches: [
    {
      branchId, branchName,
      totalScore,
      components: [{key,label,score,weight}],
      trend: {direction:"up"|"down"|"flat", delta:number},
      riskFlags: [{key,label,severity:"low"|"med"|"high"}],
      deepLinks: {details:"/raporlar/sube-saglik?branchId=..." }
    }
  ]
}

C) UI page:
- Route: /raporlar/sube-saglik
- Show:
  - Top summary cards (avg score, #high risk branches)
  - Table of branches (sortable by totalScore) with risk badges + trend arrow
  - Range selector (7/30/90)
  - If user is branch-scoped, show only their branch and hide “all branches” controls.

D) Navigation + visibility:
- Add nav item “Şube Sağlık Skoru” under Reports for HQ/admin; for mudur/supervisor allow if branch-scoped.
- Use existing role-visibility maps; do not introduce new roles.

E) QA (must do in Build mode end):
- 401 when unauth.
- 200 when auth.
- Verify branch-scoped returns 1 branch.
- Verify HQ returns multiple branches (if exists) and handles empty gracefully.
- LSP diagnostics: zero errors in changed files.
- Provide final QA report + manual curl commands.
