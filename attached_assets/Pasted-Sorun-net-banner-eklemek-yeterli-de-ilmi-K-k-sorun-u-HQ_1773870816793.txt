### FIX PLAN (apply based on investigation results)

#### Fix A: Force redirect for HQ roles on /akademi

In `client/src/pages/akademi-v3/index.tsx`, add an automatic redirect (not just a banner):

```typescript
import { useLocation } from "wouter";
import { useAuth } from "@/hooks/useAuth";

// At the TOP of the component, before any other rendering:
const { user } = useAuth();
const [, navigate] = useLocation();

const HQ_MANAGEMENT_ROLES = ['admin', 'ceo', 'cgo', 'coach', 'trainer', 'kalite_kontrol', 'gida_muhendisi', 'fabrika_mudur'];

// Auto-redirect HQ users to management panel
// Unless they explicitly navigated here with ?preview=true for content review
const searchParams = new URLSearchParams(window.location.search);
const isPreviewMode = searchParams.get('preview') === 'true';

useEffect(() => {
  if (user && HQ_MANAGEMENT_ROLES.includes(user.role) && !isPreviewMode) {
    navigate('/akademi-hq', { replace: true });
  }
}, [user, navigate, isPreviewMode]);

// If in preview mode, show the banner
// If redirecting, show loading spinner
if (user && HQ_MANAGEMENT_ROLES.includes(user.role) && !isPreviewMode) {
  return <div className="flex items-center justify-center h-64"><Loader2 className="h-8 w-8 animate-spin" /></div>;
}
```

This way:
- Coach clicks ANY link to /akademi → auto-redirected to /akademi-hq
- Coach can still preview student view via /akademi?preview=true
- Banner shows in preview mode with "Yönetim Paneline Dön" button
