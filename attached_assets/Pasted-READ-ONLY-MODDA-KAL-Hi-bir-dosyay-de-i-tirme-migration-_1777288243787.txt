READ-ONLY MODDA KAL. Hiçbir dosyayı değiştirme, migration çalıştırma, DB write yapma, package install yapma.

Amaç: Pilot test ekran görüntülerinde görülen P0/P1 hataların kök nedenini tespit etmek ve fix planı çıkarmak.

İncelenecek ana hatalar:

1) RGM / Fabrika Reçeteleri — Malzeme düzenleme hatası
- Fabrika > Reçeteler > Reçete Detayı ekranında mevcut hammadde düzenlenirken kullanıcı hammaddeyi başka hammaddeyle değiştiremiyor.
- Örnek: BEYAZ ÇİKOLATA satırı açıldığında modal sadece miktar/kategori/not gösteriyor.
- Modalda “Hammadde Seç” alanı olmalı.
- Mevcut hammadde ön seçili gelmeli.
- Kullanıcı başka hammadde seçebilmeli.
- Birim alanı boş geliyor; mevcut birim dolu gelmeli ve boş kaydedilmemeli.
- Kaydet sonrası reçete malzeme listesi yenilenmeli.
- Eski/yeni hammadde ve miktar audit log’a yazılmalı.

2) Onboarding sayfaları
- Birçok rolde onboarding’e tıklanınca boş ekran, loading, hata veya yetkisiz erişim görünüyor.
- Menüde görünen sayfa tıklanınca erişim engellenmemeli.
- Yetki yoksa menüde hiç görünmemeli.
- API null/empty dönerse frontend çökmemeli, empty state göstermeli.

3) Menü / Yetki tutarsızlıkları
- Menüde görünen ama tıklanınca “Erişim Engellendi” veren sayfaları tespit et.
- Backend permission, frontend route guard ve sidebar görünürlüğü hangi dosyalarda ayrışıyor göster.
- Özellikle supervisor, supervisor_buddy, yatirimci_branch, sube_kiosk, mudur, admin, cgo, coach rollerini kontrol et.

4) Kiosk rolü
- sube_kiosk insan personel değildir.
- Personel, bordro, izin, mesai, akademi, performans listelerinde normal çalışan gibi görünmemeli.
- Sadece kiosk/device/station bağlamında görünmeli.
- Nerelerde personel listesine dahil edildiğini tespit et.

5) Görülen runtime/frontend hataları
- “Can’t find variable: checklists”
- “shifts.filter is not a function”
- Onboarding boş/loading kalan ekranlar
- İK Raporları hata ekranı
- Alerjen & Besin sayfasında 403
- Canlı Takip ekranında ham template metni görünüyor:
  branchData[branchId] ? ... : 'Yükleniyor...'

İş kuralları:
- Müdür sadece kendi şubesindeki personel, maaş, puantaj, vardiya ve operasyon detaylarını görebilir.
- Yatırımcı Şube / Franchise alan sadece kendi şubesindeki personel detayları, maaş, puantaj ve performans özetini görebilir; başka şubeleri veya merkez/HQ personelini göremez.
- Supervisor HR tarafından izlenebilir ama personel silme yetkisi olmamalı.
- Supervisor Buddy normalde sınırlı yetkilidir; ancak bir şubede supervisor yoksa veya HQ merkez yetki devri yaparsa supervisor görevlerini üstlenebilmelidir.
- Yetkiler şube bazlı ve rol bazlı override/delegation mantığıyla esnek olmalıdır.

İstenen çıktı:
1. Her hata için:
   - Dosya adı
   - İlgili component/route/function adı
   - Tahmini kök neden
   - Severity: P0/P1/P2
   - Fix önerisi
2. Hiçbir değişiklik yapmadığını doğrula:
   - git status --short
   - git diff --stat
3. En sonunda “Uygulama için önerilen sırama” başlığı altında maksimum 6 maddelik fix sırası ver.
