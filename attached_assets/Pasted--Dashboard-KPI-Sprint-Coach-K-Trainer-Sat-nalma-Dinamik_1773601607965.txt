## IMPORTANT NOTES

1. **Check actual table names first** before writing any query.
   The grep commands in each task will show what's available.

2. **If a table doesn't exist yet**, return `null` or `0` for that metric.
   Frontend should show `'—'` when value is null, not crash.

3. **Don't create new tables** in this sprint — only read from existing ones.

4. **Minimum viable approach:** Even if only 1-2 real metrics are available per dashboard,
   replace the hardcoded numbers with those real values.
   It's better to show "—" for unknown metrics than to show wrong hardcoded numbers.

5. **Register new endpoints** in the main routes file if hq-dashboard-routes.ts
   is not already registered. Check: `grep -n "hq-dashboard" server/index.ts`
