### T004 — Concurrent Session Limit (~1 hour)

Max 2 sessions per user. On 3rd login, terminate oldest:

```bash
grep -rn "session\|passport\|login.*success\|deserialize" server/ --include="*.ts" | grep -v node_modules | head -15
```

In the login success handler:
```typescript
const activeSessions = await db.select({ count: sql`count(*)` }).from(sessions)
  .where(eq(sessions.userId, user.id));

if (activeSessions[0].count >= 2) {
  // Delete oldest session
  const oldest = await db.select().from(sessions)
    .where(eq(sessions.userId, user.id))
    .orderBy(asc(sessions.createdAt))
    .limit(1);
  if (oldest[0]) await db.delete(sessions).where(eq(sessions.sid, oldest[0].sid));
  
  await createAuditLog({
    action: 'auth.session_forced_logout',
    userId: user.id,
    details: { reason: 'Max 2 concurrent sessions' }
  });
}
```

### T007 — Suspicious Activity Detection Skill (~1 hour)

Create `server/agent/skills/security-monitor.ts`:

**Schedule:** Hourly
**Checks:**
1. 5+ failed logins from same user in 1 hour → alert admin
2. Login between 00:00-05:00 Turkey time → alert admin
3. 3+ bulk data operations in 1 hour → alert admin
4. Login from new IP for admin/ceo/cgo roles → alert admin

```typescript
async function detectSuspiciousActivity() {
  const alerts = [];
  const oneHourAgo = subHours(new Date(), 1);
  
  // 1. Brute force detection
  const failedLogins = await db.select({
    username: auditLogs.details,
    count: sql`count(*)`
  }).from(auditLogs)
    .where(and(eq(auditLogs.action, 'auth.login_failed'), gt(auditLogs.createdAt, oneHourAgo)))
    .groupBy(auditLogs.details);
  
  for (const fl of failedLogins) {
    if (Number(fl.count) >= 5) {
      alerts.push({ severity: 'high', message: `${fl.username}: ${fl.count} başarısız giriş (son 1 saat)` });
    }
  }
  
  // 2. Late night access
  const lateLogins = await db.select().from(auditLogs)
    .where(and(
      eq(auditLogs.action, 'auth.login_success'),
      gt(auditLogs.createdAt, oneHourAgo),
      sql`EXTRACT(HOUR FROM created_at AT TIME ZONE 'Europe/Istanbul') BETWEEN 0 AND 5`
    ));
  
  for (const ll of lateLogins) {
    alerts.push({ severity: 'medium', message: `Gece girişi: ${ll.userId}` });
  }
  
  // 3. Mass operations
  const massOps = await db.select({ count: sql`count(*)` }).from(auditLogs)
    .where(and(
      sql`action LIKE '%export%' OR action LIKE '%bulk%' OR action LIKE '%delete%'`,
      gt(auditLogs.createdAt, oneHourAgo)
    ));
  
  if (Number(massOps[0]?.count) > 3) {
    alerts.push({ severity: 'high', message: `${massOps[0].count} toplu işlem (son 1 saat)` });
  }
  
  for (const alert of alerts) {
    await createNotification({
      targetRole: 'admin',
      title: `🔒 Güvenlik: ${alert.message}`,
      type: 'security_alert',
      priority: alert.severity === 'high' ? 'high' : 'normal'
    });
  }
}
```

**Create middleware:**
```typescript
async function checkDataLock(tableName: string, recordCreatedAt: Date, recordStatus?: string) {
  const rule = await db.select().from(dataLockRules)
    .where(and(eq(dataLockRules.tableName, tableName), eq(dataLockRules.isActive, true)))
    .limit(1);
  
  if (!rule.length) return { locked: false };
  const r = rule[0];
  
  if (r.lockImmediately) return { locked: true, reason: r.description, canRequest: r.canRequestChange };
  if (r.lockOnStatus && recordStatus === r.lockOnStatus) return { locked: true, reason: r.description, canRequest: r.canRequestChange };
  if (r.lockAfterDays !== null) {
    const lockDate = new Date(recordCreatedAt.getTime() + r.lockAfterDays * 86400000);
    if (new Date() > lockDate) return { locked: true, reason: r.description, canRequest: r.canRequestChange };
  }
  return { locked: false };
}
```

Apply to ALL PATCH/PUT/DELETE endpoints for locked tables. Return 423 (Locked) with reason if locked:


**API endpoints:**
```
POST   /api/change-requests              — Create (any authenticated user)
GET    /api/change-requests              — List (own requests + admin sees all)
GET    /api/change-requests/:id          — Detail
PATCH  /api/admin/change-requests/:id    — Approve/reject (admin only)
```

**POST body:**
```json
{
  "tableName": "purchase_orders",
  "recordId": 1234,
  "fieldName": "unitPrice",
  "currentValue": "4500",
  "requestedValue": "4200",
  "reason": "Fatura kontrolünde hatalı giriş tespit edildi",
  "supportingDocumentUrl": "/uploads/fatura.jpg"
}
```

**On admin approval:**
1. Apply the change to the actual record
2. Save old value to record_revisions (T010)
3. Save to data_change_log (T005)
4. Update request status → 'approved'
5. Notify requester: "Düzeltme talebiniz onaylandı"

**On admin rejection:**
1. Update status → 'rejected' + review_note
2. Notify requester: "Talebiniz reddedildi. Sebep: [note]"

```typescript
async function createRevision(tableName, recordId, changes, userId, source, changeRequestId?) {
  const lastRevision = await db.select({ max: sql`MAX(revision_number)` })
    .from(recordRevisions)
    .where(and(eq(recordRevisions.tableName, tableName), eq(recordRevisions.recordId, recordId)));
  
  const revNum = (lastRevision[0]?.max || 0) + 1;
  
  await db.insert(recordRevisions).values({
    tableName, recordId, revisionNumber: revNum,
    fieldChanges: changes, changedBy: userId,
    changeSource: source, changeRequestId
  });
}
```
