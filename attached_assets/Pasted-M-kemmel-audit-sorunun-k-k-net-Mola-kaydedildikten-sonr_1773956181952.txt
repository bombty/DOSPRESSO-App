## CRITICAL FIX: Factory Kiosk — Break Flow Must Return to Working Screen

### Problem
When a factory kiosk user takes a break (mola), the system:
1. Logs the break correctly (DB record created ✓)
2. Then calls startAutoLogout() → resetWorker() → kicks user back to select-user screen ✗

User must re-enter PIN and re-select station after EVERY break. This is unacceptable for production workflow.

### Expected Flow
```
Working → STOP → Mola → log break → RETURN TO WORKING (same station, same user)
Working → STOP → Görev Bitir → log + production entry → auto-logout → select-user
Working → STOP → Vardiya Kapat → log + production entry + end shift → auto-logout → select-user
Working → STOP → Arıza Bildir → fault report → auto-logout → select-user
Working → STOP → Yardım İste → production entry → auto-logout → select-user
Working → STOP → Özel İhtiyaç → log break → RETURN TO WORKING (same as mola)
```

Only "Görev Bitir", "Vardiya Kapat", and "Arıza Bildir" should logout. "Mola" and "Özel İhtiyaç" should return to the working screen.

Change the mola/özel ihtiyaç flow to return to working screen instead of logging out:

```typescript
function handleBreakReasonSelect(reason: string) {
  if (reason === 'gorev_bitis' || reason === 'vardiya_kapat' || reason === 'yardim') {
    // These go to production-entry first, then eventually logout
    setBreakReason(reason);
    setStep('production-entry');
  } else if (reason === 'ariza_bildir') {
    // Arıza goes to fault report, then logout
    setBreakReason(reason);
    setStep('fault-report');
  } else {
    // Mola and Özel İhtiyaç — log break then RETURN TO WORKING
    setBreakReason(reason);
    logBreakMutationNoLogout.mutate({ reason });
  }
}
```

Create a separate mutation for breaks that DON'T logout:

```typescript
const logBreakMutationNoLogout = useMutation({
  mutationFn: async (data: { reason: string }) => {
    const res = await fetch("/api/factory/kiosk/log-break", {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({
        eventType: data.reason,
        stationId: selectedStation?.id,
        notes: `${data.reason} - moladan dönecek`,
      }),
    });
    if (!res.ok) throw new Error("Break log failed");
    return res.json();
  },
  onSuccess: () => {
    // Show a brief "Mola kaydedildi" toast/message
    setStep('on-break'); // Go to break screen, NOT auto-logout
  },
  onError: (error) => {
    console.error("[Kiosk] Break log error:", error);
    // Still go to break screen even if log fails
    setStep('on-break');
  },
});
```

### BreakTimer Component (simple elapsed timer)
```typescript
function BreakTimer() {
  const [seconds, setSeconds] = useState(0);
  
  useEffect(() => {
    const interval = setInterval(() => setSeconds(s => s + 1), 1000);
    return () => clearInterval(interval);
  }, []);
  
  const mins = Math.floor(seconds / 60);
  const secs = seconds % 60;
  
  return (
    <span className="text-4xl font-mono font-bold">
      {String(mins).padStart(2, '0')}:{String(secs).padStart(2, '0')}
    </span>
  );
}
```
