## FILE 3: .claude/skills/dospresso-debug-guide.md

---
name: dospresso-debug-guide  
description: DOSPRESSO debugging procedures. Use when investigating any bug or error in the DOSPRESSO project.
---

# 401 Unauthorized
- Check: router.use(isAuthenticated) applied?
- Check: credentials: 'include' on fetch calls
- Check: session expired (8hr TTL, max 2 sessions)
- Check: user inactive 60+ days (auto-deactivated)

# 403 Forbidden
- Check: user's actual role vs endpoint's allowed roles
- Check: branch user accessing HQ endpoint?

# Stale data after mutation
queryClient.invalidateQueries({ queryKey: ['/api/exact-endpoint'] });

# Query returns undefined
- Check: queryFn missing? (project uses global fetcher, no queryFn needed)
- Check: enabled condition (e.g. enabled: !!user?.branchId)

# Empty query results
- Check: forgot isNull(table.deletedAt) filter!
- All selects must filter soft deletes

# FK constraint error
- Check: FK to users must be VARCHAR not integer

# Radix UI dispatcher.useState crash
- DO NOT upgrade Radix UI packages
- Fix: npm dedupe, pin exact versions

# HTTP 423 on write
- Data Lock System active — user must submit change request
- 13 lock rules in server/services/data-lock.ts

# SLA wrong timezone
- Uses Europe/Istanbul via addBusinessHours()
- Check sla_business_hours table has 1 row
- Mon-Fri 08:00-18:00 default

# TypeScript: req.user
const userId = (req as any).user?.id;
const userRole = (req as any).user?.role;
