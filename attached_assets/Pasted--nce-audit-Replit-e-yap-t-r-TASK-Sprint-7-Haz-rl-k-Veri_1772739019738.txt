### BÖLÜM A: Şube İsimleri & Yapısı

```sql
-- 1. Tüm şubeler (aktif + pasif)
SELECT id, name, is_active, city, district, type, created_at 
FROM branches ORDER BY is_active DESC, name;

-- 2. Şüpheli isimler (test, örnek, yanlış yazım)
SELECT id, name FROM branches 
WHERE name ILIKE '%test%' OR name ILIKE '%örnek%' OR name ILIKE '%sample%' 
OR name ILIKE '%deneme%' OR name ILIKE '%branch%' OR name ILIKE '%temp%';

-- 3. Tutarsız isimlendirme (Mallof vs Mall of, eksik şehir adı)
SELECT id, name, city FROM branches WHERE is_active = true ORDER BY name;

-- 4. Fabrika ve HQ şube kayıtları
SELECT id, name, type FROM branches 
WHERE type ILIKE '%fabrika%' OR type ILIKE '%hq%' OR type ILIKE '%merkez%' 
OR name ILIKE '%fabrika%' OR name ILIKE '%merkez%' OR name ILIKE '%hq%';

-- 5. Şubeye bağlı kullanıcı sayıları
SELECT b.id, b.name, b.is_active, COUNT(u.id) as user_count 
FROM branches b 
LEFT JOIN users u ON u.branch_id = b.id AND u.is_active = true 
GROUP BY b.id, b.name, b.is_active 
ORDER BY user_count DESC;
```

- Toplam kaç şube? Aktif/pasif ayrımı

### BÖLÜM B: Kullanıcı Verisi Kalitesi

```sql
-- 1. Tüm aktif kullanıcılar, roller ve şubeleri
SELECT role, COUNT(*) as count FROM users WHERE is_active = true GROUP BY role ORDER BY count DESC;

-- 2. Test/seed kullanıcıları
SELECT id, username, full_name, role, email, branch_id, is_active, created_at 
FROM users 
WHERE username ILIKE '%test%' OR username ILIKE '%demo%' OR username ILIKE '%seed%' 
OR username ILIKE '%admin%' OR username ILIKE '%sample%' OR full_name ILIKE '%test%'
ORDER BY created_at;

-- 3. Şubesiz kullanıcılar
SELECT id, username, role, full_name FROM users 
WHERE branch_id IS NULL AND is_active = true;

-- 4. Eksik bilgiler (email, telefon, full_name boş)
SELECT id, username, role, 
  CASE WHEN full_name IS NULL OR full_name = '' THEN 'İsim yok' ELSE 'OK' END as name_status,
  CASE WHEN email IS NULL OR email = '' THEN 'Email yok' ELSE 'OK' END as email_status,
  CASE WHEN phone IS NULL OR phone = '' THEN 'Telefon yok' ELSE 'OK' END as phone_status
FROM users WHERE is_active = true
AND (full_name IS NULL OR full_name = '' OR email IS NULL OR email = '' OR phone IS NULL OR phone = '');

-- 5. Duplicate kullanıcılar (aynı isim/email)
SELECT full_name, COUNT(*) as count FROM users WHERE is_active = true 
GROUP BY full_name HAVING COUNT(*) > 1;

-- 6. Hire date ve probation kontrolü
SELECT id, username, role, hire_date, probation_end_date 
FROM users WHERE is_active = true AND hire_date IS NOT NULL 
ORDER BY hire_date DESC LIMIT 20;
```

Raporla:
- Toplam aktif kullanıcı sayısı ve rol dağılımı
- Test/seed kullanıcılar hangileri? Deaktif mi?
- Eksik bilgi olan kullanıcılar (isim, email, telefon)
- Şubesiz kullanıcılar
- Duplicate isimler

### BÖLÜM D: Orphan Kayıtlar & Referans Bütünlüğü

```sql
-- 1. Şubesi silinmiş ama kullanıcısı aktif olan
SELECT u.id, u.username, u.branch_id FROM users u 
LEFT JOIN branches b ON u.branch_id = b.id 
WHERE u.is_active = true AND (b.id IS NULL OR b.is_active = false);

-- 2. Kullanıcısı silinmiş ama görevi aktif olan
SELECT t.id, t.title, t.assigned_to_id FROM tasks t 
LEFT JOIN users u ON t.assigned_to_id = u.id 
WHERE t.status IN ('active', 'pending') AND (u.id IS NULL OR u.is_active = false)
LIMIT 20;

-- 3. Modülü silinmiş ama career_path_levels'da referans edilen
SELECT cpl.level, cpl.title, cpl.required_module_ids FROM career_path_levels cpl;
-- Sonra bu ID'lerin training_modules'da var olduğunu kontrol et

-- 4. Boş/kullanılmayan tablolar
SELECT schemaname, tablename, n_tup_ins, n_tup_upd, n_live_tup 
FROM pg_stat_user_tables 
WHERE n_live_tup = 0 
ORDER BY tablename;

-- 5. Çok büyük tablolar (potansiyel şişme)
SELECT schemaname, tablename, n_live_tup, pg_size_pretty(pg_total_relation_size(schemaname||'.'||tablename)) as size
FROM pg_stat_user_tables 
ORDER BY n_live_tup DESC 
LIMIT 20;
```

Raporla:
- Orphan kullanıcılar (şubesi yok/pasif)
- Orphan görevler (atanan kişi yok/pasif)
- Referans bütünlüğü ihlalleri
- Boş tablolar listesi
- En büyük tablolar (şişme riski)
