Each quick action button should navigate to the correct tab or open a dialog:
- "Onboarding Şablonu" → switch to Modüller tab + open Onboarding creator dialog
- "Eğitim Raporu" → switch to İstatistikler tab
- "Sertifika Ayarları" → switch to Sertifikalar tab

Implementation: The buttons need onClick handlers that change the active tab. Since the tabs are in the parent (index.tsx) and buttons might be in RoleDashboard, pass a callback:

```typescript
// In index.tsx:
const [activeTab, setActiveTab] = useState("moduller");

// Pass to RoleDashboard:
<RoleDashboard user={user} onNavigateTab={setActiveTab} />

// In RoleDashboard:
interface RoleDashboardProps {
  user: any;
  onNavigateTab?: (tab: string) => void;

// Button handlers:
<button onClick={() => onNavigateTab?.("moduller")}>AI Modül Üret</button>
<button onClick={() => onNavigateTab?.("istatistikler")}>Eğitim Raporu</button>
<button onClick={() => onNavigateTab?.("sertifikalar")}>Sertifika Ayarları</button>
<button onClick={() => onNavigateTab?.("moduller")}>Onboarding Şablonu</button>
```

### FIX 2: Runtime Error — Find and Fix Root Cause

```bash
# The error "Objects are not valid as a React child (found: object with keys {$$typeof, render})"
# means a component reference is used as a child instead of rendered as JSX

# Check ALL akademi-hq files for this pattern
grep -rn "{[A-Z][a-zA-Z]*}" client/src/pages/akademi-hq/ --include="*.tsx" | grep -v "import\|from\|className\|onClick\|onChange\|onSubmit\|return\|const\|let\|type\|interface\|useState\|useMemo\|useQuery\|<" | head -20

# Check for Lucide icons used incorrectly (most common cause)
grep -rn "}\s*[A-Z]" client/src/pages/akademi-hq/ --include="*.tsx" | grep -v "import\|from\|className" | head -15

# Check for forwardRef components ($$typeof + render keys = forwardRef)
grep -rn "forwardRef\|React.forwardRef" client/src/pages/akademi-hq/ --include="*.tsx" | head -5

# Check the Sertifika tab (likely source since it's the last tab visible)
grep -rn "{[A-Z][a-zA-Z]*Icon\|{Icon\|{Award\|{Trophy\|{Medal\|{Shield" client/src/pages/akademi-hq/SertifikaTab.tsx | head -10

# Check ALL tabs for the pattern
for f in client/src/pages/akademi-hq/*.tsx; do
  echo "=== $f ===";
  grep -n ">\s*{[A-Z]" "$f" | grep -v "import\|className\|onClick\|<" | head -5;
done

# Also check: is the error from akademi-hq or from a different page?
# The error screenshot shows a different URL — check which page causes it
```

The `{$$typeof, render}` pattern strongly suggests a forwardRef component or a Lucide icon being used as `{Icon}` instead of `<Icon />`. Find it and fix it.

Common fix patterns:
```typescript
// WRONG (Lucide icon as object):
<span>{GraduationCap} Başlık</span>
// CORRECT:
<span><GraduationCap className="h-4 w-4" /> Başlık</span>

// WRONG (forwardRef component as child):
<div>{SomeComponent}</div>
// CORRECT:
<div><SomeComponent /></div>
```
