ROL: Senior full-stack engineer + data pipeline designer.
PROJE: DOSPRESSO WebApp (Replit), HR/IK modülü.
HEDEF: HQ İK ekranından personelleri Excel (XLSX) ile:
  1) dışa aktar (filtreli + tam detay)
  2) içe aktar (toplu ekleme/güncelleme) 
güvenli, izlenebilir ve geri alınabilir şekilde yönetmek.

MUTLAK KURALLAR:
- Admin kullanıcılar/import ile ASLA değiştirilemez, ASLA pasife alınamaz, ASLA silinemez.
- Import her zaman önce DRY RUN (simülasyon) çalıştırmalı; kullanıcı onayından sonra APPLY.
- Import işlemleri audit_logs’a yazılmalı ve import_batch_id ile ilişkilendirilmeli.
- Import sırasında hiçbir kayıt hard delete edilmemeli. (Soft delete/pasifleme opsiyonel ve sadece “Sync Mode” ile.)

KAPSAM (ÖZELLİKLER):
A) Export (XLSX)
- UI: HQ/İK > Personeller sayfasına “Excel Dışa Aktar” butonu ekle.
- Filtreler: scope (HQ / Factory / Branch / All), branchId(ler), title/role, status, işe giriş tarih aralığı.
- 2 export tipi:
  1) “Liste Export” (tek sheet)
  2) “Tam Detay Export” (çok sheet: employees, leaves, payroll, discipline, terminations, documentsLinks)
- Excel oluşturma: Node tarafında xlsx (SheetJS) veya exceljs kullan.
- Büyük veri için streaming / chunk düşün (50 şube hedefi).

B) Import (XLSX)
- UI: “Excel İçe Aktar” butonu + dosya seçici.
- Import Mode seçimi (wizard):
  1) Append (sadece ekle)
  2) Upsert (ekle + güncelle)
  3) Sync (Excel kaynak, riskli) — varsayılan KAPALI, sadece HQ Admin + 2 onay.
- Eşleştirme anahtarı seçimi:
  - Öncelik: employeeId
  - fallback: email (unique)
- DRY RUN sonucu ekranda göster:
  - toCreate, toUpdate, toSkip, toError sayılarını ve satır bazında reason.
- APPLY onayı sonrası transaction’lı uygula.
- Her satır için import_results kaydı tut (import_batch_id, rowNumber, status, message, employeeId).
- Rollback: “Bu importu geri al” endpoint’i (7 gün içinde) — audit_logs before/after ile geri döndür.

C) Backend API
- POST /api/hr/employees/export (filters -> returns file)
- POST /api/hr/employees/import/dry-run
- POST /api/hr/employees/import/apply
- POST /api/hr/employees/import/:batchId/rollback
- GET  /api/hr/employees/import/batches (history)

D) DB / Schema
- import_batches: id, createdByUserId, mode, scope, createdAt, summaryJson
- import_results: id, batchId, rowNumber, status, employeeId, message, beforeJson, afterJson
- Indexler: batchId, employeeId

E) Güvenlik ve Veri Kuralları
- Admin protection: role=admin veya isSystemUser true ise SKIP.
- HQ/Factory personelleri branchId zorunluluğunu doğru ele al (mevcut yapınıza göre).
- Validasyon: tarih formatları, maaş numeric, email format, zorunlu alanlar.
- Hata mesajları kullanıcıya anlaşılır, server detail sızmasın.
