### Problem
On ALL pages (dashboards, kiosk, admin, etc.), the header/navbar scrolls OUT OF VIEW when the user scrolls down. On tablet/mobile, pulling down causes the header to disappear. This affects EVERY page in the app, not just the kiosk.

This is a GLOBAL layout issue — the fix must be in the shared layout component, NOT in individual pages.

### CRITICAL: CSS-only fix. Do NOT change any page logic, routes, or backend.

## STEP 2: Understand the current structure

The typical DOSPRESSO layout is:
```
<html>
  <body>
    <div id="root">

            </main>
```

## STEP 3: Apply the fix

The layout must be restructured so the header is FIXED and content scrolls independently:

```tsx
// Layout component should be:
<div className="h-screen flex flex-col overflow-hidden">
  
  {/* Header — FIXED at top, never scrolls */}
  <header className="flex-shrink-0 h-14 border-b border-border bg-background z-50">
    {/* logo, nav, user menu, hamburger */}
  </header>
  
  {/* Below header — sidebar + content */}
  <div className="flex-1 flex overflow-hidden">
    
    {/* Sidebar — desktop only, fixed height, scrolls internally */}
    <aside className="hidden md:flex flex-shrink-0 w-64 border-r border-border overflow-y-auto">
      {/* sidebar nav items */}
    </aside>
    
    {/* Main content — scrolls independently */}
    <main className="flex-1 overflow-y-auto">
      {/* Page content renders here */}
      <Outlet /> {/* or children or Route */}
    </main>
    
  </div>
</div>
```

### Key CSS rules:
- Outer: `h-screen flex flex-col overflow-hidden` — locks to viewport
- Header: `flex-shrink-0` — never compressed
- Content row: `flex-1 flex overflow-hidden` — fills remaining space
- Sidebar: `overflow-y-auto` — scrolls if menu is long
- Main: `flex-1 overflow-y-auto` — page content scrolls HERE, not at body level

### Also fix body/html:
```css
/* In index.css or global styles */
html, body, #root {
  height: 100%;
  overflow: hidden; /* Prevent body-level scroll */
}
```

### Remove conflicting styles:
```bash
# Find and remove any min-h-screen on the layout wrapper
grep -rn "min-h-screen" client/src/components/layout*.tsx client/src/components/Layout*.tsx client/src/components/app-layout*.tsx 2>/dev/null | head -10

# Find body scroll styles
grep -n "overflow.*scroll\|overflow.*auto" client/src/index.css | head -10
```

Remove `min-h-screen` from the layout — replace with `h-screen overflow-hidden`.
Remove `overflow: auto` or `overflow: scroll` from body/html if present.
