Kritik problem: Publish (Production) domain’de hiçbir kullanıcı login olamıyor (401). Preview/Dev’de login çalışıyor. Bu, Production DB’de users tablosunun boş olması veya auth env/secret farklı olması ihtimali.

Hedef: Production’da login’i garanti edecek şekilde düzelt.

İstediğim plan + uygulama (öncelik sırası):

A) PROD teşhis
1) Production logs’ta login denemesinde hata reason’u ayırt et: “user not found” mı “password mismatch” mı?
2) DB bağlantısı doğru mu: Production DATABASE_URL/DB provider ile bağlanıyor mu? (Sadece “connected” ve “users count” gibi güvenli bilgi; gizli bilgi/log sızdırma yok)

B) PROD bootstrap (idempotent, güvenli)
3) Server startup’ta çalışacak bootstrapAdminUser() ekle:
   - Eğer users tablosunda username='admin' yoksa oluştur.
   - Password hash, sistemin kullandığı aynı hash fonksiyonu ile üretilecek.
   - Şifre environment variable’dan okunacak: ADMIN_BOOTSTRAP_PASSWORD (yoksa startup’ta açık hata logla ve admin oluşturma).
   - Admin oluşturulduktan sonra accountStatus=approved ve isActive=true olacak (sistemin şemasına göre).
   - Eğer admin zaten varsa hiçbir şey yapma (idempotent).

C) Prod health/diagnostics (GÜVENLİ)
4) /api/health endpoint’ini production’da güvenli tut:
   - Sadece: status, timestamp, dbConnected(true/false), usersCount gibi minimum bilgi dönsün.
   - DATABASE_URL veya host/db name gibi detayları asla response’a yazma.

D) Deployment checklist (ben IT uzmanı değilim)
5) Replit Deployments/Production UI içinde benim yapacağım adımları yaz:
   - Hangi env var’ları ekleyeceğim (en az: ADMIN_BOOTSTRAP_PASSWORD; ayrıca gerekliyse DATABASE_URL, SESSION_SECRET/JWT_SECRET vs).
   - Nerede eklenecek: Deployments → Production → Environment Variables
   - Sonra Republish/Restart

E) Doğrulama
6) Publish URL’de admin ile login test adımları:
   - admin / (ADMIN_BOOTSTRAP_PASSWORD)
   - Login sonrası /api/messages gibi endpoint’lerin çalıştığını doğrula.

Kısıtlar:
- Production DB’de mevcut veriyi silme.
- Secrets/log sızdırma yok.
- Kod değişiklikleri net dosya yollarıyla yapılacak.
