## A1: Menü veri yapısını güncelle (menu-service.ts)

Mevcut düz yapı yerine hiyerarşik yapı oluştur. **Mevcut SIDEBAR_ALLOWED_ITEMS mantığını BOZMADAN** üstteki bölüm tanımlarını grupla:

### Yeni menü hiyerarşisi:

```
Ana Sayfa (grup değil, tek öğe)
  → Dashboard
  → Ajanda

CRM (grup — collapse/expand)
  → Franchise (kanal)
  → Misafir (kanal)
  → Task (kanal — YENİ, Bölüm B'de eklenecek)

Operasyon (grup)
  → Ekipman
  → Checklist
  → Kalite & Denetim
  → Gıda Güvenliği

  → Personel
  → Vardiyalar
  → Devam / PDKS
  → Bordro

  → Mali Yönetim
  → Satınalma
  → Stok

Yönetim (grup — sadece admin/ceo/cgo)

  → Şubeler
```

### menu-service.ts değişikliği:

Bölüm (section) tanımlarına `parentGroup` veya `groupId` alanı ekle. Veya mevcut section yapısını koru — frontend'de grupla.

**Eğer menu-service.ts'te section yapısı zaten var ve frontend bunu gruplayabiliyorsa:**
- Backend'e dokunma, sadece frontend sidebar bileşenini değiştir
- Section'ların sıralamasını ve isimlendirmesini güncelle

**Eğer backend değişikliği gerekiyorsa:**
- Section'lara `collapsible: true` alanı ekle
- Alt öğeleri olan section'lar varsayılan collapsed başlasın

### ⚠ DANIŞMA: Menü yapısı değişmeden ÖNCE tüm rollerin mevcut menü öğe sayısını kaydet. Değişiklik SONRASINDA aynı sayıyı gördüğünden emin ol. Hiçbir rol menü öğesi KAYBETMEMELİ.

## A2: Sidebar bileşenini yeniden yaz

Mevcut sidebar/nav bileşenini bul ve güncelle:

### Davranış:
1. **Varsayılan:** Grup ikon + grup adı görünür (collapse hali)
2. **Grup tıklanınca:** O grubun alt öğeleri açılır (diğerleri kapanır — accordion)
3. **Alt öğe tıklanınca:** O sayfaya navigate et
4. **Mobilde:** Sidebar üstten overlay olarak açılır, sayfa seçilince kapanır

- Collapse: 56px (ikon + kısa grup adı — VEYA sadece ikon)

```

```

## C3: Arşiv sistemi

### Todo arşiv:
- Tamamlanan todo'lar otomatik "Arşiv" bölümüne taşınır
- Ajanda'da "Arşiv" tab/görünüm eklenir: "Takvim | Todo | Notlar | Arşiv"
- Arşiv'de: tamamlanan todo'lar + iptal edilen event'ler + silinen notlar
- Toplu silme: checkbox ile seç → "Seçilenleri sil" butonu
- Tek tek silme: swipe veya sağ tık → sil

### DB değişikliği:
```sql
-- user_todos'a archive kolonu ekle (yoksa)
ALTER TABLE user_todos ADD COLUMN IF NOT EXISTS archived_at TIMESTAMP;
-- user_calendar_events'e archive kolonu ekle
ALTER TABLE user_calendar_events ADD COLUMN IF NOT EXISTS archived_at TIMESTAMP;
-- user_notes'a archive kolonu ekle
ALTER TABLE user_notes ADD COLUMN IF NOT EXISTS archived_at TIMESTAMP;
```

### API güncelleme:
```
PATCH /api/ajanda/todos/:id/archive → archived_at = now()
PATCH /api/ajanda/todos/:id/unarchive → archived_at = null
GET /api/ajanda/archive → tüm arşivlenmiş öğeler (todo + event + note)
DELETE /api/ajanda/archive/bulk → { ids: [...] } toplu silme
```
