Süreci daha güvenli + hızlı yapmak için nasıl optimize ederiz?

En iyi pratik: “Tek run = tek feature” kuralı

Replit Agent’a aynı anda 3 farklı prompt atmak yerine:
	•	1. Prompt = Build (implement)
	•	2. Prompt = QA automation
	•	3. Prompt = Deploy + Prod smoke checklist

Çünkü Replit’in akışı zaten: Plan → Start building → (gerekirse) Revise plan → Build → QA → Done.
Bir run içinde hem implement hem QA hem deploy talimatlarını karıştırırsan agent bazen gereksiz refactor’a kayabiliyor veya QA’yı “erken PASS” sayabiliyor.

“Revise plan” ne zaman?
	•	Agent plan çıkardı ama scope fazla geniş (refactor, framework ekleme, db schema genişletme vs.) görüyorsan → Revise plan.
	•	Plan doğruysa → Start building.
	•	Build bitti + QA PASS → yeni prompt ile Deploy/Smoke.

⸻

Kopyala-yapıştır “standart prompt paketim” (Replit için)

PROMPT-1 (Build)

Bunu yeni feature’larda kullan:
	•	“Yeni feature X’i implement et”
	•	“Minimal diff, refactor yok”
	•	“Write-path yok (sadece log)”
	•	“RBAC şart”
	•	“JSON schema + fallback şart”
	•	“Plan max 6 task”

PROMPT-2 (QA)

“SADECE son değişiklik için QA automation çalıştır. Yeni feature ekleme. Fail olursa sadece ilgili bug fix.”

PROMPT-3 (Deploy + Prod Smoke)

“Deploy/Republish sonrası prod smoke test checklistini çalıştır, sonuçları kısa release notu olarak yaz.”

İstersen bu 3 prompt’u senin DOSPRESSO formatına tek şablon halinde kilitleyip (her seferinde sadece feature adını değiştirerek) sana “kutsal prompt” yaparım.

⸻

Senin ekranına göre net cevap

Şu an ekranda “Republish this app” aşamasındasın.
Yapacağın şey: Republish’a bas → yukarıdaki prod smoke test adımlarını uygula.
Hepsi PASS ise, bir satırlık release notu yeter:

“Ops Copilot prod deploy edildi, PASS smoke (UI+API+RBAC+fallback).”
