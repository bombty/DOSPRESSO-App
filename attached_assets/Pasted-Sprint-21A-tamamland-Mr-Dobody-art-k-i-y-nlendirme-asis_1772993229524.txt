## TASK: Quality Control — 6-Perspective Audit (NO CODE CHANGES)

Do NOT modify any code. Only analyze and report issues.

### 1. CRITICAL ERRORS (Crash / Data Loss Risk)

```bash
grep -rn "Cannot access\|is not defined\|is not a function\|undefined is not\|null is not" client/src/ --include="*.tsx" --include="*.ts" 2>/dev/null | head -10
```

Check:
- Any circular dependencies in new files? (dobody-flow-context.tsx, dobody-flow-mode.tsx, dobody-mini-bar.tsx, dobody-flow.ts)
- Any new API endpoints returning errors?
- Does the flow-tasks API handle edge cases? (no user, no branch, no tasks)

### 2. SECURITY

```bash
grep -rn "router\.\(get\|post\|put\|patch\|delete\)" server/routes/dobody-flow.ts | head -20
```

Check:
- ALL endpoints in dobody-flow.ts have isAuthenticated middleware?
- Can a barista complete another user's task?
- Is user_id validated in flow-completions? (no IDOR)

### 3. TYPE SAFETY & DATA INTEGRITY

```bash
grep -rn "dobody_flow_tasks\|dobody_flow_completions" server/ --include="*.ts" | head -20
```

Check:
- Tables created correctly with proper types?
- Foreign key to users(id) correct type? (varchar vs integer — remember push_subscriptions had this bug)
- task_id in completions — is it properly formatted?

### 4. UI/UX CONSISTENCY

```bash
grep -rn "bg-white\|text-black" client/src/components/dobody-flow-mode.tsx client/src/components/dobody-mini-bar.tsx 2>/dev/null | head -10
```

Check:
- Flow card renders correctly in dark mode?
- Mini-bar doesn't overlap bottom nav?
- Mobile 375px — card fits, touch targets ≥ 44px?
- Greeting shows correct Turkish characters?

### 5. PERFORMANCE & MEMORY

```bash
grep -rn "setInterval\|setTimeout\|useEffect" client/src/components/dobody-flow-mode.tsx client/src/components/dobody-mini-bar.tsx client/src/contexts/dobody-flow-context.tsx 2>/dev/null | head -20
```

Check:
- Any polling intervals without cleanup?
- Context re-renders — does every child re-render when flow state changes?
- localStorage reads on every render? (should be only on mount)

### 6. INTEGRATION & REGRESSION

Check:
- Existing Mr. Dobody suggestions still visible below flow card?
- Landing pages still show all existing content?
- Login redirects still work?
- Sidebar filtering still works (Sprint 20B)?
- Bottom nav still works?
- Service Worker cache version bumped?

Run e2e tests:
```bash
npx playwright test --reporter=list 2>&1 | tail -20
```

### REPORT FORMAT

For each finding:
```
CATEGORY: [Critical/Security/Type/UI/Performance/Integration]
FILE: [file:line]

IMPACT: [what happens if not fixed]

```

If NO issues: "✅ Check complete — no issues found"
If issues found: Present fix plan, wait for approval, THEN fix.
