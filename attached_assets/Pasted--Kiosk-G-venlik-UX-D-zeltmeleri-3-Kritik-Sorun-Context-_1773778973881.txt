# Kiosk Güvenlik + UX Düzeltmeleri — 3 Kritik Sorun

## Context
DOSPRESSO franchise platform. Factory kiosk (client/src/pages/fabrika/kiosk.tsx) has 3 issues:
1. "Kiosk'tan Çık" button redirects to full management dashboard — security breach
2. After worker logout, device password screen shows again — should show staff selection
3. No production plan visible in kiosk — workers don't know what to produce

Do NOT touch: PIN auth logic, shift tracking, production entry, branch kiosk.

---

## FIX 1 — "Kiosk'tan Çık" Button Security

**File:** `client/src/pages/fabrika/kiosk.tsx`

### Current Problem
Find the exit/logout button that navigates to `/` or `/fabrika/dashboard`.
It currently takes kiosk users to the full management panel.

```bash
grep -n "Kiosk.*Çık\|exitKiosk\|navigate.*fabrika\|setLocation.*fabrika\|dashboard" client/src/pages/fabrika/kiosk.tsx | head -10
```

### Fix
The exit button should:
- NOT navigate to any management page
- Only return to the device password screen (first step)
- Require no admin confirmation (device password screen is already the gate)

```tsx
// Find the exit button and change:
// WRONG: navigate('/fabrika/dashboard') or setLocation('/fabrika/dashboard')
// RIGHT: resetKiosk() — goes back to device-password screen

// If there's a "Kiosk'tan Çık" or "Çıkış" button:
<button
  onClick={() => {
    // Full reset — back to device password
    resetKiosk();
    setStep('device-password'); // or whatever the first step is called
  }}
>
  Kiosk'tan Çık
</button>
```

Also check if `fabrika_operator` role users can accidentally navigate away.
The KioskGuard should prevent navigation to `/fabrika/dashboard` for kiosk sessions.

**Check KioskGuard:**
```bash
grep -n "KioskGuard\|kioskGuard" client/src/App.tsx | head -5
```

If KioskGuard is empty (just passes children through), it should at minimum
prevent authenticated `fabrika_operator`/`sube_kiosk` users from accessing
management pages. Add this protection:

```tsx
function KioskGuard({ children }: { children: React.ReactNode }) {
  const { user } = useAuth();
  const [location] = useLocation();
  
  // If a kiosk-role user somehow navigates away from kiosk, redirect back
  const isKioskRole = ['fabrika_operator', 'fabrika_personel', 
                        'fabrika_sorumlu', 'sube_kiosk'].includes(user?.role ?? '');
  const isOnKioskPage = location.startsWith('/fabrika/kiosk') || 
                         location.startsWith('/sube/kiosk') ||
                         location.startsWith('/hq/kiosk');
  
  if (isKioskRole && !isOnKioskPage) {
    // Kiosk users should only be on kiosk pages
    return <Redirect to={user?.role === 'sube_kiosk' 
      ? `/sube/kiosk/${user.branchId}` 
      : '/fabrika/kiosk'} />;
  }
  
  return <>{children}</>;
}
```

**Apply KioskGuard to ALL routes** — not just kiosk routes.
Wrap the main route tree so kiosk users can never escape.

## FIX 2 — Worker Logout Should NOT Show Device Password

**File:** `client/src/pages/fabrika/kiosk.tsx`

### Current Problem
When a worker ends their shift, `resetKiosk()` is called and takes the user
back to step 1 (device-password). But the device was already authenticated —
it shouldn't ask for device password again.

### Fix — Two-Level Reset

```typescript
// Split reset into two functions:

// 1. Worker logout (soft reset) — stay on device-authenticated session
const resetWorker = () => {
  setSelectedUser(null);
  setPinInput('');
  setSelectedStation(null);
  setCurrentSession(null);
  // Keep deviceAuthenticated = true
  setStep('select-user'); // go to staff selection, NOT device-password
};

// 2. Full kiosk exit (hard reset) — clear everything including device auth
const resetKiosk = () => {
  setSelectedUser(null);
  setPinInput('');
  setSelectedStation(null);
  setCurrentSession(null);
  setDeviceAuthenticated(false);
  setDevicePassword('');
  setStep('device-password'); // back to device password
};
```

Add a `deviceAuthenticated` state:
```typescript
const [deviceAuthenticated, setDeviceAuthenticated] = useState(false);
```

When device-auth succeeds:
```typescript
// In handleDeviceAuth success:
setDeviceAuthenticated(true);
setStep('select-user');
```

When worker ends shift or auto-logouts:
```typescript
// Use resetWorker() not resetKiosk()
resetWorker(); // stays on select-user, device still authenticated
```

Only use `resetKiosk()` for the explicit "Kiosk'tan Çık" button.

**Find all places that call resetKiosk or setStep('select-user') after worker logout:**
```bash
grep -n "resetKiosk\|setStep.*select\|auto-logout\|autoLogout\|end.*shift\|endShift" client/src/pages/fabrika/kiosk.tsx | head -20
```

Update each worker-logout location to use `resetWorker()` instead.

## FIX 3 — Show Production Plan in Kiosk

**File:** `client/src/pages/fabrika/kiosk.tsx`

### Add production plan to the "select-station" or "working" step

The endpoint already exists: `GET /api/factory/production-plans`
With today filter: `GET /api/factory/kiosk/today-plans`

Add a query for today's plans:
```typescript
const { data: todayPlans = [] } = useQuery({
  queryKey: ['/api/factory/kiosk/today-plans'],
  queryFn: () => fetch('/api/factory/kiosk/today-plans', {
    headers: { 'x-kiosk-token': kioskToken ?? '' }
  }).then(r => r.json()),
  refetchInterval: 60 * 1000, // refresh every 60 seconds
  enabled: deviceAuthenticated, // only fetch after device auth
});
```

### Show plans in select-station step

In the station selection screen, add a read-only plan summary below the stations:

```tsx
{/* Today's Production Plan */}
{todayPlans.length > 0 && (
  <div className="mt-6 w-full max-w-2xl">
    <div className="text-xs font-bold text-muted-foreground mb-2 uppercase tracking-wide">
      Bugünün Üretim Planı
    </div>
    <div className="grid grid-cols-1 gap-2">
      {todayPlans.map((plan: any) => (
        <div key={plan.id} className="flex items-center gap-3 px-3 py-2 rounded-lg bg-card border border-border">
          <span className="text-sm font-semibold text-foreground">{plan.productName}</span>
          <span className="text-xs text-muted-foreground">→</span>
          <span className="text-sm text-foreground">{plan.targetQuantity} {plan.unit}</span>
          <span className="ml-auto text-xs px-2 py-0.5 rounded-full bg-blue-100 text-blue-700 dark:bg-blue-900 dark:text-blue-300">
            {plan.stationName}
          </span>
        </div>
      ))}
    </div>
  </div>
)}

{todayPlans.length === 0 && deviceAuthenticated && (
  <div className="mt-4 text-xs text-muted-foreground">
    Bugün için üretim planı girilmemiş
  </div>
)}
```

### Also show plan in "working" step

When a worker is at their station working, show the plan for THEIR station:

```tsx
const stationPlan = todayPlans.filter(
  (p: any) => p.stationId === selectedStation?.id
);

{stationPlan.length > 0 && (
  <div className="text-xs text-muted-foreground">
    Hedef: {stationPlan[0].targetQuantity} {stationPlan[0].unit} — {stationPlan[0].productName}
  </div>
)}
```
