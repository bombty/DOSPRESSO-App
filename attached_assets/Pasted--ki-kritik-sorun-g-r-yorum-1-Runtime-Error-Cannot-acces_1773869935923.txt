### FIX PHASE — Based on investigation findings

#### Fix 1: Runtime Error
Based on Step 1 findings, fix the "Cannot access 'module' before initialization" error. Common causes:
- Circular import between files
- Variable named 'module' used before const/let declaration

#### Fix 2: HQ Role Academy Routing
The fundamental issue: Coach/Trainer need TWO things:
1. Their sidebar "Akademi Yönetimi" link → /akademi-hq (management view) ✅ (already done)
2. BUT they also need to ACCESS /akademi (student view) for PREVIEW purposes

The problem is likely:
- Coach sees "Akademi Yönetimi" in sidebar → goes to /akademi-hq ✅
- But INSIDE /akademi-hq, there might be links that go to /akademi (student view)
- Or coach navigates to /akademi directly from browser/bookmark
- When at /akademi, they see the student interface with no way to manage

**Solution approach — smart routing:**
In the student academy page (/akademi → akademi-v3/index.tsx), add a role check:
```typescript
// At the top of the student academy page:
const { user } = useAuth();
const isHQUser = user && ['admin', 'ceo', 'cgo', 'coach', 'trainer', 'kalite_kontrol', 'gida_muhendisi'].includes(user.role);

// Show a banner for HQ users:
{isHQUser && (
  <div className="bg-amber-500/10 border border-amber-500/30 rounded-lg p-3 mb-4 flex items-center justify-between">
    <div className="flex items-center gap-2">
      <Eye className="h-4 w-4 text-amber-500" />
      <span className="text-sm text-amber-600 dark:text-amber-400">
        Öğrenci ön izleme modundasınız
      </span>
    </div>
    <Button variant="outline" size="sm" onClick={() => navigate('/akademi-hq')}>
      Yönetim Paneline Dön
    </Button>
  </div>
)}
```

This way:
- HQ users CAN preview the student view (useful for content review)
- They see a clear banner that they're in preview mode
- One-click return to management panel
