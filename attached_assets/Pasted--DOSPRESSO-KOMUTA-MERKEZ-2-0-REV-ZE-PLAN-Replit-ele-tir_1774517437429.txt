## MİMARİ KARAR

**Seçenek B (Dinamik) — Replit'in de önerdiği:**

```
Tek MissionControlDynamic bileşeni
  ↓
GET /api/me/dashboard-data (birleştirilmiş endpoint — tek çağrı)
  ↓
Backend: rol + kullanıcı config → hangi widget'lar aktif?
  ↓
Her aktif widget için ilgili veriyi topla (backend'de birleştirilmiş)
  ↓
Frontend: widget listesini render (CollapsibleSection)
```

**Neden tek endpoint?**
- 10 widget × 270 kullanıcı = 2700 API çağrısı riski → tek endpoint ile 270 çağrı
- Backend'de veri toplama paralel yapılır (Promise.all)
- Frontend sadece sonucu render eder
- Cache: 2 dakika staleTime

**Mevcut MC bileşenleri ne olacak?**
- MissionControlCoach → KORUNUR (zaten iyi, Sprint 13'te yeniden yazıldı)
- MissionControlSupervisor → KORUNUR (özel bileşenler: PresenceBar, ShiftTimeline)
- MissionControlStajyer → KORUNUR (eğitim odaklı, iyi çalışıyor)
- MissionControlFabrika → KORUNUR (7 fabrika rolü, özel bileşenler)
- MissionControlMuhasebe → KORUNUR (finans odaklı)
- MissionControlYatirimci → ZENGİNLEŞTİRİLECEK (daha fazla widget)
- MissionControlHQ → DİNAMİK SİSTEME GEÇECEK (CEO/CGO/Admin + fallback roller)

## ADMİN WİDGET YÖNETİM SİSTEMİ

### DB Yapısı

```sql
-- Yeni tablo: Rol bazlı widget config
CREATE TABLE dashboard_role_widgets (
  id SERIAL PRIMARY KEY,
  role VARCHAR(50) NOT NULL,
  widget_key VARCHAR(100) NOT NULL,
  is_enabled BOOLEAN DEFAULT true,
  display_order INTEGER DEFAULT 0,
  default_open BOOLEAN DEFAULT true,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW(),
  UNIQUE(role, widget_key)
);

-- Widget tanımları (master list)
-- Mevcut dashboard_widget_items tablosu kullanılabilir veya yeni:
CREATE TABLE dashboard_widget_registry (
  id SERIAL PRIMARY KEY,
  widget_key VARCHAR(100) UNIQUE NOT NULL,
  title VARCHAR(200) NOT NULL,
  description TEXT,
  data_source VARCHAR(200),       -- API endpoint veya query key
  component_key VARCHAR(100),     -- frontend bileşen adı
  required_permissions TEXT[],     -- hangi API yetkisi gerekli (güvenlik)
  default_roles TEXT[],           -- varsayılan gösterilecek roller
  category VARCHAR(50),           -- 'operations', 'finance', 'hr', 'quality', 'crm'
  is_active BOOLEAN DEFAULT true,
  created_at TIMESTAMP DEFAULT NOW()
);
```

### Frontend — MissionControlDynamic

```typescript
// Tek bileşen, config'den gelen widget'ları render eder
function MissionControlDynamic() {
  const { data } = useQuery({
    queryKey: ['/api/me/dashboard-data'],
    staleTime: 2 * 60 * 1000,  // 2 dakika cache
  });

  return (
    <div className="space-y-4">
      {/* Selamlama + AI Briefing */}
      <Greeting />
      <AIBriefing />
      
      {/* KPI Strip */}
      <UnifiedKPI variant="compact" items={data.kpis} />
      
      {/* Widget'lar — config sırasıyla */}
      {data.widgets.map(widget => (
        <CollapsibleSection
          key={widget.key}
          title={widget.title}
          defaultOpen={widget.defaultOpen}
          badge={widget.badge}
        >
          <WidgetRenderer widgetKey={widget.key} data={widget.data} />
        </CollapsibleSection>
      ))}
      
      {/* Mr. Dobody */}
      <DobodySuggestions />
      
      {/* Quick Actions */}
      <QuickActions items={data.quickActions} />
    </div>
  );
}
```
