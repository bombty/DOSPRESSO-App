Replit ile daha güvenli/hızlı ilerlemek için optimizasyon (senin sorunun cevabı)

Altın kural

1 Agent run = 1 feature + 1 QA + 1 deploy (gerekirse).
Karıştırırsan agent “scope creep” yaşar ve refactor’a kayar.

Replit’te “Start building” / “Revise plan” nasıl kullanılmalı?
	•	Tek bir master prompt yapıştır → Agent plan üretsin.
	•	Plan doğruysa Start building.
	•	Plan yanlış/eksikse Revise plan ile aynı feature scope içinde düzelt.
	•	Build bitince ayrı bir “QA-only prompt” ile son kontrol yaptır (senin checklist’in gibi).
	•	Sonra deploy.

Aynı anda 1-2-3 farklı prompt’u art arda yapıştırma.
Çünkü agent bunları “tek görev” sanıp scope’u büyütür veya istemeden refactor eder.

En iyi pratik: 3 aşamalı prompt seti
	1.	BUILD PROMPT (feature implement)
	2.	QA PROMPT (sadece test, sadece bug fix)
	3.	RELEASE PROMPT (deploy + prod smoke checklist)

Bunları sırayla çalıştırırsın. Her aşamadan sonra:
	•	Plan iyi → Start building
	•	Plan değil → Revise plan
