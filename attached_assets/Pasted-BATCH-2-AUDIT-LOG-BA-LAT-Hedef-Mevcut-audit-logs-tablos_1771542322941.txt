BATCH-2 (AUDIT LOG) BAŞLAT.

Hedef: Mevcut audit_logs tablosu var ama hiçbir endpoint çağırmıyor. Audit log’u aktif et ve Admin panelde “Denetim Günlüğü” ekranı çıkar.

1) Audit event sözlüğü (Phase-1)
- auth.login_success, auth.login_failed, auth.logout
- user.created, user.updated, user.deactivated, user.deleted(soft), user.restored, user.role_changed, user.branch_changed
- role.created, role.updated, role.deleted, role.permission_changed
- branch.created, branch.updated, branch.deleted(soft), branch.restored
- task.created, task.updated, task.deleted(soft), task.status_changed
- checklist.completed, checklist.failed
- shift.created, shift.updated, shift.deleted(soft)
- equipment.created, equipment.updated, equipment.deleted(soft)
- equipment.fault_created, equipment.fault_resolved
- cash_report.created, cash_report.updated, cash_report.deleted(soft)
- inventory.count_submitted, inventory.adjustment
- procurement.created, procurement.approved, procurement.cancelled
- backup.completed, backup.failed, backup.manual_triggered
- settings.changed (AI ayarları dahil)

2) Audit kaydı zorunlu alanlar
- event_type
- actor_user_id, actor_role
- scope_branch_id (branchId; HQ/factory için null olabilir)
- target_resource, target_resource_id
- before/after (en azından kritik alanlar; mümkünse diff)
- request_id (her request için tek ID, middleware ile üret)
- ip_address, user_agent
- created_at

3) Implementasyon
- En kritik DELETE ve UPDATE endpoint’lerine createAuditLog çağrısı ekle (Batch-1’de listelenen riskli endpoint’ler öncelik).
- Soft delete kullanan yerlerde event_type “*.deleted” olsun, restore’da “*.restored”.
- Login/logout ve permission/role değişimleri mutlaka loglansın.

4) Admin UI
- Admin panelde “Denetim Günlüğü” sekmesi oluştur.
- Filtreler: tarih aralığı, şube, kullanıcı, event_type, resource
- Liste: Kim + Ne yaptı + Hangi şube + Ne değişti (kısa) + Zaman
- Detay drawer: before/after JSON + request_id + ip + user_agent
- CSV export ekle.

5) Test
- 1-2 kritik işlem yapınca audit_logs’da kayıt oluştuğunu kanıtla.
- UI’da filtreler çalışsın.
- Batch-2 bittiğinde dur ve Batch-3 (Soft delete/trash) için onay iste.
