## TASK: Sprint 22 — Remaining UX Fixes (Post-Audit)

The system-wide bug scan already fixed 11 crashes, 2 broken links, and 30+ ASCII Turkish strings. This sprint covers the remaining UX issues found during live CGO testing that the audit did NOT address.

### T001 — Personel List Must Show Names + Scores (~1 hour)

**Problem:** Şubeler → Işıklar → Personel tab shows only role ("barista") and start date. No names, no scores.

```bash
grep -rn "personel\|PersonelList\|StaffList\|employee.*card\|staff.*card" client/src/pages/ --include="*.tsx" | grep -i "sube\|branch\|detail\|yeni-sube" | head -10
```

**Fix:** Each staff card must show:
- Full name (firstName + lastName or fullName) as primary text
- Role in Turkish with proper capitalization (Barista, Supervisor, Müdür)
- Start date
- Composite score badge (last 30 days) — e.g., "78/100" with color coding (green >70, yellow 50-70, red <50)
- Aktif/Pasif status badge

If the API doesn't return names, find why:
```bash
grep -rn "personel\|staff\|employee" server/routes/ --include="*.ts" | grep -i "branch.*detail\|sube.*detail" | head -10
```

### T002 — Add Period Selector to Personnel Scores (~2 hours)

**Problem:** No way to filter personnel scores by time period.

**Add to branch detail personel tab:**
```
Dönem: [Son 1 Ay ▼]
Options: Son 1 Ay | Son 3 Ay | Son 6 Ay | Son 1 Yıl | Özel Tarih Aralığı
```

When period changes, re-fetch scores for that date range.

Check if the composite score API supports date params:
```bash
grep -rn "composite.*score\|compositeScore\|branch.*score\|personnel.*score" server/routes/ --include="*.ts" | head -10
```

If not, add `startDate` and `endDate` query params to the score API.

### T003 — Flow Mode Tasks Navigate to Wrong Page (~1 hour)

**Problem:** CGO Flow Mode shows "Şube Uyarıları — 38 aktif uyarı var" but navigates to /cgo-command-center instead of the actual problem page.

```bash
grep -rn "navigateTo\|navigate_to" server/routes/dobody-flow.ts | head -20
```

**Fix navigateTo for each task type:**
- "Şube Uyarıları" → `/branches-list` (so CGO can see which branches need attention)
- "Mr. Dobody Önerisi" about specific branch → `/sube/[branchId]` (direct to the problem branch)
- "Agent önerileri bekliyor" → `/agent-center` (already correct)
- Generic HQ tasks → relevant report page, not command center

### T004 — Performance Reports Empty for HQ Users (~1 hour)

**Problem:** Raporlar → Performans shows "Henüz performans verisi yok" with all metrics at 0, but CGO Komuta Merkezi shows real data (Ort. Skor: 92, Checklist: %100).

```bash
grep -rn "performans\|PerformansTab\|PerformanceReport\|performance.*tab" client/src/pages/ --include="*.tsx" | head -10
```

**Likely cause:** The Performans tab queries data with a branchId filter, but HQ users have no branchId → empty result. CGO Komuta Merkezi aggregates across all branches.

**Fix:** For HQ roles (ceo, cgo, admin, coach), the Performans tab should aggregate data across ALL branches, same as Komuta Merkezi does.

### T005 — Equipment List Compact + Warning Icons (~1 hour)

**Problem:** Equipment cards are too large. Serial numbers like "BLENDER-B5-1762764446563-TEST-P7RP-FINAL-tKpb-AUTOTEST" waste space. No indication of active faults.

```bash
grep -rn "equipment.*list\|ekipman.*list\|EquipmentCard\|EquipmentItem" client/src/ --include="*.tsx" | head -10
```

**Fix:**
- Truncate serial numbers: show first 20 chars + "..."
- Reduce card height to single line if possible
- Add ⚠️ icon with fault count if equipment has open faults
- Equipment name should be primary text (larger), serial secondary (smaller, muted color)

```
espresso          ESPRESSO-B5-17627...     Aktif
blender           BLENDER-B5-17627...  ⚠️1 Aktif  
ice               ICE-B5-17627...          Aktif
```
