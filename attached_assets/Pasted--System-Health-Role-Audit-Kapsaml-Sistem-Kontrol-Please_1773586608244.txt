### 1A: Role Home Routes

For each role, report:
- Role name
- Home path (where they land after login)
- Is the target page/component implemented? (PASS/FAIL)

Expected home routes:
| Role Group | Expected Home |
|---|---|
| admin, ceo, cgo | /hq-ozet or /merkez-dashboard |
| coach, trainer | coach dashboard |
| muhasebe, ik, satinalma | HQ department dashboard |
| mudur, supervisor | branch dashboard |
| barista, bar_buddy, stajyer | employee dashboard |

### 1B: Route Protection
Check App.tsx route definitions.

- Report unprotected routes (if any)

### 1C: NavRail Items vs Actual Routes
Check `client/src/lib/navigation-config.ts`.
For each role group's NavRail items, verify:
- Does the path in each NavRailItem actually exist as a route in App.tsx?

## 7. DATA INTEGRITY CHECK

### 7A: User Roles in Database
Run this query and report results:
SELECT role, COUNT(*) as count,
       SUM(CASE WHEN "deletedAt" IS NULL THEN 1 ELSE 0 END) as active
FROM users
GROUP BY role
ORDER BY count DESC;

### 7B: Branch Assignment
Run this query:
SELECT COUNT(*) as users_without_branch
FROM users
WHERE "branchId" IS NULL
AND "deletedAt" IS NULL
AND role NOT IN ('admin','ceo','cgo','coach','trainer','muhasebe','ik','satinalma');

Report how many non-HQ users have no branch assigned.

### 7C: Active Delegations
SELECT * FROM module_delegations WHERE is_active = true;
Report any currently active delegations.
