## BÖLÜM 1: MR. DOBODY AGENT SİSTEMİ

```bash
echo "════════════════════════════════════════════════"
echo "BÖLÜM 1: AGENT SİSTEMİ TAM HARİTA"
echo "════════════════════════════════════════════════"

echo "=== 1A: Agent skill dosyaları ==="
ls -la server/agent/skills/*.ts 2>/dev/null
echo "--- Her skill'in adı ve satır sayısı ---"
for f in server/agent/skills/*.ts; do
  name=$(basename "$f" .ts)
  lines=$(wc -l < "$f")
  # İlk yorum satırını al (skill açıklaması)
  desc=$(head -5 "$f" | grep -i "//\|description\|skill\|name" | head -1)
  echo "  $name ($lines satır) — $desc"
done

echo ""
echo "=== 1B: Her skill hangi rollere hitap ediyor? ==="
for f in server/agent/skills/*.ts; do
  name=$(basename "$f" .ts)
  roles=$(grep -i "role\|target.*role\|applicable\|hedef" "$f" | head -3)
  echo "  [$name] $roles"
done

echo ""
echo "=== 1C: Agent engine — skill nasıl çalıştırılıyor? ==="
wc -l server/services/agent-engine.ts 2>/dev/null
grep -n "function\|export\|class\|skill\|execute\|run\|trigger" server/services/agent-engine.ts 2>/dev/null | head -20

echo ""
echo "=== 1D: Agent scheduler — ne zaman çalışıyor? ==="
grep -n "agent\|scheduler\|cron\|interval\|daily\|weekly\|hourly" server/services/agent-scheduler.ts 2>/dev/null | head -20

echo ""
echo "=== 1E: Agent escalation sistemi ==="
wc -l server/services/agent-escalation.ts 2>/dev/null
grep -n "escalat\|level\|severity\|critical\|urgent\|remind\|alert" server/services/agent-escalation.ts 2>/dev/null | head -15

echo ""
echo "=== 1F: System completeness service (gap detection) ==="
wc -l server/services/system-completeness-service.ts 2>/dev/null
grep -n "gap\|check\|missing\|incomplete\|severity\|deepLink" server/services/system-completeness-service.ts 2>/dev/null | head -20

echo ""
echo "=== 1G: Guidance widget — Mr. Dobody frontend ==="
wc -l client/src/components/widgets/guidance-widget.tsx 2>/dev/null
grep -n "queryKey\|useQuery\|severity\|priority\|role" client/src/components/widgets/guidance-widget.tsx 2>/dev/null | head -10

echo ""
echo "=== 1H: Agent action center ==="
wc -l client/src/pages/agent-action-center.tsx 2>/dev/null
grep -n "action\|approve\|reject\|pending\|status" client/src/pages/agent-action-center.tsx 2>/dev/null | head -10

echo ""
echo "=== 1I: Agent aksiyonları — DB'de ne var? ==="
echo "SELECT action_type, status, COUNT(*) FROM agent_actions GROUP BY action_type, status ORDER BY count DESC LIMIT 20;" | psql 2>/dev/null
echo ""
echo "SELECT severity, COUNT(*) FROM agent_actions GROUP BY severity ORDER BY count DESC;" | psql 2>/dev/null

echo ""
echo "=== 1J: Agent skill loader — nasıl register ediliyor? ==="
grep -rn "registerSkill\|loadSkill\|skillRegistry\|SKILLS\|skillList" server/ --include="*.ts" | grep -v node_modules | head -15

echo ""
echo "=== 1K: .agents dizini (Replit agent skills) ==="
ls -la .agents/skills/ 2>/dev/null
for d in .agents/skills/*/; do
  name=$(basename "$d")
  lines=$(wc -l < "$d/SKILL.md" 2>/dev/null || echo "?")
  echo "  $name: $lines satır"
done
```

## RAPORLAMA

**Her bölüm için şu bilgileri düzenli tablo formatında sun:**

### Bölüm 1 Raporu — Agent Sistemi:
- Toplam skill sayısı ve isimleri
- Her skill'in hedef rolleri
- Scheduler çalışma zamanları (günlük/haftalık/saatlik)
- Escalation seviyeleri ve kuralları

### Bölüm 2 Raporu — Bildirim Zincirleri:
- Bildirim gönderen modüller ve hedef roller (tablo)
- En çok bildirim alan roller
- Çalışan onay akışları (kim → kimi → ne için)
- Kopuk bildirim zincirleri

### Bölüm 3 Raporu — Rol Hiyerarşisi:
- Tam rol hiyerarşisi ağacı (tree format)
- Her rolün kimi denetlediği
- Her rolün kime rapor verdiği
- Şube bazlı personel dağılımı

### Bölüm 4 Raporu — Agent Otonomisi:
- Mr. Dobody'nin otonom kararları listesi
- AI/GPT entegrasyon noktaları
- Otomatik rapor oluşturma durumu
- Agent'ın kullanıcıya önerme mekanizması

### Bölüm 5 Raporu — Fabrika Bilgi Akışı:
- Fabrikada bildirim gönderen/almayan noktalar (tablo)
- Skor verileri kime görünüyor, kime görünmüyor
- QC sonuçları kime ulaşıyor
- Üretim verileri hangi rollere açık

### Bölüm 6 Raporu — Şube→HQ Bilgi Akışı:
- Şube seviyesinde kalan veriler vs HQ'ya çıkan veriler (tablo)
- CEO/CGO'nun gördüğü hazır raporlar
- Coach'un gördüğü şube verileri
- Eksik raporlar (olması gereken ama olmayan)

### BONUS — Genel Bilgi Akış Haritası:
Tüm bulgulardan şu soruları cevapla:

1. **Alt → Üst bilgi akışı tam mı?**
   - Operatör üretim yapar → bu veri müdüre gidiyor mu?
   - Barista görev tamamlar → bu veri coach'a gidiyor mu?
   - Şube performansı → CEO/CGO dashboard'unda mı?

2. **Mr. Dobody ne kadar otonom?**
   - Kendi başına bildirim gönderiyor mu?
   - Kendi başına rapor oluşturuyor mu?
   - Kendi başına escalation yapıyor mu?
   - Yoksa sadece guidance widget'ta pasif öneriler mi sunuyor?

3. **Hangi raporlar otomatik, hangisi manuel?**
   - Günlük/haftalık/aylık otomatik rapor var mı?
   - Yoksa tüm raporlar kullanıcının sayfayı açmasına mı bağlı?

4. **Kopuk hiyerarşi noktaları:**
   - Veri üretilen ama kimsenin görmediği noktalar
   - Rapor olması gereken ama olmayan noktalar
   - Bildirim gitmesi gereken ama gitmeyen noktalar
