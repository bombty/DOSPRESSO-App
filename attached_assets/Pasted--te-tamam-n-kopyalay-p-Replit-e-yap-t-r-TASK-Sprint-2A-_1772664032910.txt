**Yapılacak:**

```sql

WHERE level = 4;

-- Seviye 5 (Supervisor): Liderlik + strateji
UPDATE career_path_levels SET required_module_ids = ARRAY[34, 35, 36, 37, 38, 39]
```

**ÖNEMLİ:** Yukarıdaki ID'ler audit raporundaki öneriler. Atamadan ÖNCE şunu çalıştır ve mevcut modüllerin title'larını kontrol et:


Modül içerikleri ve seviyelere uygunluğu mantıklıysa uygula. Uygun değilse en mantıklı eşleştirmeyi yap. Her seviyede EN AZ 3 modül olmalı.

### T004 — Composite Score Güncelleme Job'u Oluştur (~2 saat)

**Problem:** Composite Score formülü kodda var (training %25, practical %25, attendance %25, manager %25) ama düzenli güncelleme job'u YOK. 6 seed kullanıcı dışında kimsenin skoru hesaplanmıyor.

**Araştırma:**
```bash
grep -rn "compositeScore\|composite_score\|calculateScore\|computeScore\|updateScore" server/ --include="*.ts" | head -20
grep -rn "cron\|setInterval\|schedule\|job\|background" server/ --include="*.ts" | head -20
```

**Yapılacak:**

1. Mevcut composite score hesaplama fonksiyonunu bul
2. Background job oluştur: her gece 03:00 Istanbul saatinde (bildirim arşivleme 02:00'de, çakışmasın)
3. Job tüm aktif kullanıcılar için skoru yeniden hesaplar
4. Hesaplama mantığı (mevcut formül):
   - **Training (%25):** tamamlanan modül sayısı / atanmış toplam modül sayısı × 100
   - **Practical (%25):** checklist tamamlama oranı + görev zamanında bitirilme oranı
   - **Attendance (%25):** vardiya devamlılığı, devamsızlık oranı
   - **Manager (%25):** staff_evaluations tablosundaki son değerlendirme skoru (yoksa 50 default)
5. Hesaplanan skor ilgili tabloya yazılsın
6. Skor değişim log'u tutulsun (önceki skor → yeni skor, tarih)
7. Skor < 60 olan kullanıcılar için supervisor'a uyarı bildirimi gönder

**Eşik kontrolleri (mevcut):**
- ≥ 70: terfi için uygun
- 60-70: uyarı bölgesi
- < 60: tehlike (3 ay üst üste ise demotion riski)

**Önemli:** Training verisi yoksa o bileşeni geçici olarak 50 (nötr) kabul et.

**Doğrulama:**
```sql
SELECT u.username, u.role, cs.* FROM composite_scores cs 
JOIN users u ON cs.user_id = u.id 
WHERE u.is_active = true 
ORDER BY cs.total_score DESC LIMIT 20;
```

### T005 — Sınav Anti-Cheat Sistemi (~2-3 saat)

**Problem:** Mevcut quiz sistemi basit — sayfa kapanınca sınav devam edebilir, tab değiştirme kontrolü yok, süre kontrolü yetersiz.

**Araştırma:**
```bash
grep -rn "quiz\|Quiz\|sınav\|Sinav\|exam\|Exam" client/src/ --include="*.tsx" -l
grep -rn "quiz.*submit\|submitQuiz\|quiz.*answer\|quiz.*result" server/routes/ --include="*.ts" | head -10
```

**Yapılacak:**

**A) Tab Değiştirme / Sayfa Kapanma Tespiti:**

Quiz bileşenine ekle:

```typescript
// Tab değiştirme
document.addEventListener('visibilitychange', () => {
  if (document.hidden && examInProgress) {
    showWarning("Sınava geri dönün! 10 saniye içinde dönmezseniz sınavınız iptal edilecek.");
    tabSwitchTimer = setTimeout(() => {
      autoSubmitExam('tab_switch_timeout');
    }, 10000);
  } else {
    clearTimeout(tabSwitchTimer);
    hideWarning();
  }
});

// Sayfa kapatma
window.addEventListener('beforeunload', (e) => {
  if (examInProgress) {
    autoSubmitExam('page_close');
    e.preventDefault();
    e.returnValue = 'Sınavınız devam ediyor. Sayfayı kapatırsanız sınavınız sonlanacaktır.';
  }
});
```

**B) Multi-step Soru Gösterimi:**
- Sorular tek tek gösterilir (1 ekranda 1 soru)
- İleri/geri gitme YOK — geçtikten sonra dönemez
- Her soruda bireysel geri sayım (45-60 saniye)
- Toplam sınav süresi sağ üst köşede
- Progress bar: "Soru 3/10"
- Süre dolunca soru boş olarak işaretlenir, sonrakine geçer

**C) Sınav Kayıt Verisi:**
Her attempt için kaydet:
- Başlangıç/bitiş zamanı
- Her sorunun yanıtlanma süresi (saniye)
- Tab switch sayısı
- Tamamlanma şekli: normal / timeout / tab_switch / page_close
- Cihaz bilgisi (mobil/tablet/desktop)

**D) Anomali Flag'leri:**
- 10 soruyu 60 saniyede bitirdiyse → flag: "çok hızlı"
- 3+ tab switch → flag: "tab switching"
- page_close ile bittiyse → flag: "sayfa kapatıldı"
- Flag'ler yönetici panelinde görünür

### T007 — AI Soru Üretimi (~1-2 saat)

**Problem:** Soru havuzu yetersiz. Her modül için 15-20 soru gerekiyor. 50 modül × 15 = 750 soru manuel yazmak pratik değil.

**Yapılacak:**

1. Mevcut AI altyapısını kullan (multi-provider, RAG aktif)
2. Her modül için:
   - Modül içeriğini oku
   - AI'a gönder: "Bu içerikten 15 çoktan seçmeli soru üret. 4 şık, 1 doğru. Zorluk: [difficulty]. Türkçe. JSON format."
   - Soruları validate et (4 şık var mı, doğru cevap işaretli mi)
   - quiz_questions tablosuna kaydet
3. 50 modül için toplam ~750 soru üret
4. Final sınavı soruları: o seviyenin tüm modüllerinden rastgele seçim

**Doğrulama:**
```sql
SELECT tm.id, tm.title, COUNT(qq.id) as question_count 
FROM training_modules tm 
LEFT JOIN quiz_questions qq ON qq.module_id = tm.id 
GROUP BY tm.id, tm.title ORDER BY question_count ASC;
-- Her modülde en az 10 soru olmalı
```
