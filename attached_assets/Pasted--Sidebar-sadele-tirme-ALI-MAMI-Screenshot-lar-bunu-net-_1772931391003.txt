## TASK: URGENT FIX — Sidebar Simplification Not Working

The Sprint 20B sidebar changes are NOT visible. All roles still see the full grouped menu (40+ items). The SIDEBAR_ALLOWED_ITEMS filter is being bypassed by the menu group/section structure.

### THE PROBLEM

The menu service builds items grouped into sections (Operasyonlar, Ekipman & Bakım, etc.). SIDEBAR_ALLOWED_ITEMS filters individual item keys, but the GROUP HEADERS and their children still render if ANY child passes the scope check.

Example for barista — allowed items are: `['branch-dashboard', 'tasks-list', 'training-academy', 'notifications']`

But sidebar shows:
- Operasyonlar GROUP (because 'tasks-list' is inside this group) → and then ALL other items in the group also show (Şubeler, Checklistler, Kayıp Eşya, etc.)
- Ekipman & Bakım GROUP (should not show at all)
- Eğitim & Akademi GROUP (because 'training-academy' is inside)

### THE FIX

The filtering must happen at the ITEM level, not the group level. After building the menu, filter out items NOT in the role's allowed list, THEN remove empty groups.

Find where the menu is built and returned to the frontend:
```bash
grep -rn "buildMenu\|getMenu\|menuForRole\|filteredMenu" server/menu-service.ts | head -20
```

The fix logic should be:

```typescript
function getMenuForRole(role: string) {
  const allowedItems = SIDEBAR_ALLOWED_ITEMS[role];
  
  // If no allowlist defined, return full menu (admin fallback)
  if (!allowedItems) return fullMenu;
  
  // Filter: keep only items whose KEY is in the allowed list
  const filtered = fullMenu.sections.map(section => ({
    ...section,
    items: section.items.filter(item => allowedItems.includes(item.key))
  })).filter(section => section.items.length > 0); // Remove empty groups
  
  return { sections: filtered };
}
```

**CRITICAL:** The filter must:
1. Check each individual item's key against SIDEBAR_ALLOWED_ITEMS
2. Remove items NOT in the list (even if they pass scope check)
3. Remove group headers that have 0 remaining items
4. Keep `İletişim Merkezi` and `Profilim` as always-visible footer items (these are NOT in groups)
