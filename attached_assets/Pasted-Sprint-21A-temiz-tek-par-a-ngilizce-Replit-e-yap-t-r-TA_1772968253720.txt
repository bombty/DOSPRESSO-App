## TASK: Sprint 21A — Mr. Dobody Flow Mode (Guided Daily Workflow)

Transform Mr. Dobody from a suggestion bot into a step-by-step work flow guide. When users log in, Mr. Dobody greets them, shows their prioritized task list, and guides them through each task sequentially.

This is NOT a chatbot. This is a guided workflow overlay on existing landing pages.

### T001 — Database Tables for Flow Tasks (~30 min)

Create two new tables:

```sql
CREATE TABLE dobody_flow_tasks (
  id SERIAL PRIMARY KEY,
  title VARCHAR(255) NOT NULL,
  description TEXT,
  navigate_to VARCHAR(255),
  estimated_minutes INTEGER DEFAULT 5,
  priority VARCHAR(20) DEFAULT 'normal',
  target_roles TEXT[],
  target_branches INTEGER[],
  target_users INTEGER[],
  created_by INTEGER REFERENCES users(id),
  start_date DATE NOT NULL DEFAULT CURRENT_DATE,
  end_date DATE,
  is_active BOOLEAN DEFAULT true,
  source VARCHAR(20) DEFAULT 'manual',
  created_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE dobody_flow_completions (
  id SERIAL PRIMARY KEY,
  task_id VARCHAR(100) NOT NULL,
  user_id INTEGER REFERENCES users(id),
  completed_at TIMESTAMP DEFAULT NOW(),
  UNIQUE(task_id, user_id, completed_at::date)
);
```

Add these to schema.ts / drizzle schema following existing patterns.

**The endpoint merges TWO sources:**

**Source 1 — Manual tasks (from dobody_flow_tasks table):**
```sql
SELECT * FROM dobody_flow_tasks WHERE
  is_active = true
  AND start_date <= CURRENT_DATE
  AND (end_date IS NULL OR end_date >= CURRENT_DATE)
  AND (target_roles IS NULL OR user_role = ANY(target_roles))
  AND (target_branches IS NULL OR user_branch_id = ANY(target_branches))
  AND (target_users IS NULL OR user_id = ANY(target_users))
```
Exclude tasks already in dobody_flow_completions for today.

**Rendering — Flow Card (shown at TOP of landing page, above existing content):**

```
┌─────────────────────────────────────────────┐
│ 🤖 İyi Günler, Cihan!                       │
│ 🔥 8 gün streak! Bugün de devam edelim.     │
│                                              │
│ Bugün 4 işin var:                            │
│                                              │
│ 1. ⭐ Yeni Temizlik Protokolü (HQ)   ~5dk  │
│ 2. ☐ Açılış Checklist                ~5dk  │
│ 3. ☐ Cold Brew Eğitimi              ~5dk  │
│ 4. ☐ Stok Bildirimi                 ~2dk  │
│                                              │
│         [ İlk İşe Başla → ]                 │
│                                              │
│ ── veya ──                                   │
│ Kendi başıma devam etmek istiyorum →         │
└─────────────────────────────────────────────┘
```

⭐ = manual task from HQ (source='manual')
☐ = auto-detected task (source='auto')
Completed tasks show ✅ with strikethrough

**Behavior:**
1. On mount: fetch GET /api/dobody/flow-tasks
2. Show flow card with task list
3. "İlk İşe Başla" tapped → navigate to task[0].navigateTo, activate mini-bar
4. "Kendi başıma devam" tapped → hide card, save to localStorage `dobody_flow_dismissed_YYYY-MM-DD`, show normal page
5. If ALL tasks already completed today → show completion card instead
6. If 0 tasks → show "Bugün planlanmış görev yok. İyi çalışmalar! 🎉"

**Completion card:**
```
┌─────────────────────────────────────────────┐
│ 🎉 Bugünkü görevlerini tamamladın!          │
│                                              │
│ ⭐ 8 gün streak!  📊 Skor: 78/100          │
│                                              │
│ [ Mr. Dobody'ye Sor ]     [ Kapat ]         │
└─────────────────────────────────────────────┘
```

**Design requirements:**
- Subtle gradient background (dark: slate-800 to slate-900, light: brand colors)
- Mr. Dobody icon (small 🤖 or existing dobody avatar) on left of greeting
- Task items as cards with icon + title + estimated time

### T004 — Mini-Bar (Floating Task Progress Indicator) (~2 hours)

Create: `client/src/components/dobody-mini-bar.tsx`

When user navigates AWAY from the landing page to complete a task, show a floating bar above bottom nav.

**Create context provider:** `client/src/contexts/dobody-flow-context.tsx`

```typescript
interface DobodyFlowState {
  isFlowActive: boolean;
  tasks: FlowTask[];
  activeTaskIndex: number;
  setTaskComplete: (taskId: string) => void;
  dismissFlow: () => void;
  activateFlow: () => void;
}
```

**Wrap in main layout** (the root layout component that wraps all pages):
```tsx
<DobodyFlowProvider>
  <AppLayout>
    {children}
  </AppLayout>
  {/* Mini-bar renders here, above bottom nav */}
  <DobodyMiniBar />
</DobodyFlowProvider>
```

**Mini-bar visual (fixed position, bottom of screen, above bottom nav):**

Active task state:
```
┌──────────────────────────────────────────┐
│ 🤖 Görev 1/3: Açılış Checklist  [✓ Bitti]│
└──────────────────────────────────────────┘
```

Transition state (after marking complete, 2 second duration):
```
┌──────────────────────────────────────────┐
│ ✅ Harika! Sıradaki: Cold Brew Eğitimi    │
└──────────────────────────────────────────┘
```
After 2 seconds → auto-navigate to next task's page.

All complete state (5 second duration then auto-dismiss):
```
┌──────────────────────────────────────────┐
│ 🎉 Tüm görevler tamamlandı! Harika iş!   │
└──────────────────────────────────────────┘
```

**Mini-bar rules:**
- Only visible when flow mode is active AND user is NOT on their landing page
- Does NOT show on landing page (because flow card is visible there)
- Fixed bottom position, z-index above page content but below modals
- Height: ~48px, full width, semi-transparent background
- "✓ Bitti" button = calls POST /api/dobody/flow-tasks/:taskId/complete
- Tapping the task title text navigates back to landing page
- Swipe down to minimize (becomes small floating dot with task count badge)
