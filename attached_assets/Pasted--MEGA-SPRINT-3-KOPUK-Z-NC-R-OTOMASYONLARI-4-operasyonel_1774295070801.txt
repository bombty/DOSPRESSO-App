## ZİNCİR 1: VARDİYA → DEVAM → BORDRO

### Problem:
- Vardiya oluşturulunca çalışana bildirim gitmiyor
- PDKS gecikmeli giriş supervisor'a uyarı vermiyor (veya yetersiz)
- Ay sonu bordro hesaplama manuel tetikleniyor

### Çözüm — 3 otomasyon:

#### 1a: Vardiya atanınca → çalışana bildirim

Vardiya oluşturma/güncelleme endpoint'ini bul. Başarılı kayıt SONRASINA ekle:

```typescript
// Vardiya oluşturuldu → çalışana bildirim
await createNotification({
  userId: assignedUserId,
  type: 'shift_assigned',
  title: 'Yeni vardiya atandı',
  message: `${formatDate(shiftDate)} tarihinde ${startTime}-${endTime} vardiyası atandı`,
  deepLink: `/vardiyalar`,
  priority: 'medium'
});
```

#### 1b: PDKS gecikme → supervisor + müdür bildirim

Kiosk giriş endpoint'ini bul. Giriş zamanı > vardiya başlangıç + 15dk ise:

```typescript
// Gecikmeli giriş tespit edildi
if (minutesLate > 15) {
  // Supervisor'a bildirim
  await createNotification({
    userId: supervisorId,
    type: 'late_arrival',
    title: `Gecikmeli giriş: ${employeeName}`,
    message: `${employeeName} ${minutesLate} dakika geç giriş yaptı`,
    deepLink: `/pdks`,
    priority: minutesLate > 60 ? 'high' : 'medium'
  });
}
```

**⚠ Mevcut late-arrival-tracker skill'i bunu zaten yapıyor olabilir — ÖN ARAŞTIRMADA KONTROL ET. Yapıyorsa tekrar ekleme, sadece doğrula.**

#### 2b: Modül tamamlama sonrası kontrol

Eğitim modülü tamamlama endpoint'ini bul. Başarılı tamamlama SONRASINA ekle:

```typescript
// Modül tamamlandı → kariyer ilerlemesi kontrol et
const completedModules = await getCompletedModuleCount(userId);
const quizAvg = await getQuizAverage(userId);
const currentRole = user.role;
```

#### 3b: Tekrar arıza tespiti (Mr. Dobody)

```typescript
// Skill: equipment-lifecycle-tracker
// Trigger: arıza kaydedildiğinde
action: async (context) => {
  const { equipmentId, branchId } = context;
  
  // Son 30 günde aynı ekipmanın arıza sayısı
  const recentFaults = await db.select()
    .from(equipmentMaintenanceRecords)
    .where(and(
      eq(equipmentMaintenanceRecords.equipmentId, equipmentId),
      gte(equipmentMaintenanceRecords.createdAt, thirtyDaysAgo)
    ));
  
  if (recentFaults.length >= 3) {
    // CEO/CGO + fabrika müdüre uyarı
    await createAgentAction({
      userId: getCEOId(),
      skillId: 'equipment-lifecycle-tracker',
      type: 'alert',
      title: `Tekrar eden arıza: ${equipmentName}`,
      message: `${equipmentName} son 30 günde ${recentFaults.length}. kez arızalandı. Ekipman değişimi veya kapsamlı bakım öneriyorum.`,
      deepLink: `/ekipman/${equipmentId}`,
      priority: 'high'
    });
  }
}
```

#### 3c: CRM'e otomatik escalation

Arıza 48 saat içinde çözülmezse → CRM Franchise kanalına otomatik ticket:

```typescript
// Scheduler veya skill trigger
// 48 saat geçmiş çözülmemiş arızaları tara
const unresolvedFaults = await getUnresolvedFaults(48); // saat

for (const fault of unresolvedFaults) {
  // Zaten escalation ticket var mı?
  const existingTicket = await findTicket({
    ticketType: 'ariza_escalation',
    sourceId: fault.id
  });
  
  if (!existingTicket) {
    await createTicket({
      channel: 'franchise',
      ticketType: 'ariza_escalation',
      department: 'Teknik',
      title: `Arıza escalation: ${fault.equipmentName}`,
      description: `${fault.branchName} şubesinde ${fault.equipmentName} 48 saattir çözülemedi. Orijinal arıza: ${fault.description}`,
      branchId: fault.branchId,
      source: 'system',
      priority: 'high'
    });
  }
}
```

## ZİNCİR 4: STOK → ÜRETİM → SEVKİYAT

### Problem:
- Şube stoku kritik seviyeye düşünce fabrikaya otomatik talep gitmiyor
- Üretim tamamlanınca sevkiyat bildirimi yok

### Çözüm — Mr. Dobody skill: `supply-chain-monitor`

**⚠ ÖN ARAŞTIRMADA KONTROL ET:** Stok tabloları ve verileri mevcut mu? Eğer branch_stock_items veya inventory_items tablosu boş/yoksa, bu zincir şimdilik ATLANMALI — veri olmadan otomasyon çalışmaz.

#### 4a: Stok kritik seviye uyarısı

```typescript
// Skill: supply-chain-monitor
// Trigger: günlük scheduler (sabah 08:00)
action: async () => {
  // Kritik stok seviyesindeki şube ürünlerini bul
  const criticalItems = await db.select()
    .from(branchStockItems)
    .where(lte(branchStockItems.currentStock, branchStockItems.minimumStock));
  
  if (criticalItems.length > 0) {
    // Şube müdürüne + fabrika müdürüne bildirim
    for (const item of criticalItems) {
      await createAgentAction({
        userId: getBranchManagerId(item.branchId),
        skillId: 'supply-chain-monitor',
        type: 'warning',
        title: `Stok kritik: ${item.productName}`,
        message: `${item.branchName}'da ${item.productName} stoku ${item.currentStock} adet — minimum ${item.minimumStock}`,
        deepLink: `/stok`,
        priority: 'high'
      });
    }
    
    // Fabrika müdürüne özet
    await createAgentAction({
      userId: getFactoryManagerId(),
      skillId: 'supply-chain-monitor',
      type: 'summary',
      title: `${criticalItems.length} üründe stok kritik`,
      message: `${criticalItems.map(i => i.productName).join(', ')} — üretim planı güncellenmeli`,
      deepLink: `/fabrika`,
      priority: 'high'
    });
  }
}
```

#### 4b: Üretim tamamlanınca sevkiyat bildirimi
