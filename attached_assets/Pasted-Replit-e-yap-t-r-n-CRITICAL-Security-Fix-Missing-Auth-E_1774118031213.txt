## FIX 2: Add try/catch to Unprotected Routes (HIGH)

```bash
# Find routes without try/catch in the 5 affected files
for file in server/routes/employee-summary.ts server/routes/hr.ts server/routes/pdks.ts server/routes/branch-orders.ts server/routes/branch-inventory.ts; do
  echo "=== $file ==="
  # Count total routes
  total=$(grep -c "router\.\(get\|post\|patch\|put\|delete\)" "$file" 2>/dev/null)
  # Count routes with try/catch
  protected=$(grep -A3 "router\.\(get\|post\|patch\|put\|delete\)" "$file" 2>/dev/null | grep -c "try {")
  echo "  Total routes: $total, With try/catch: $protected, Missing: $((total - protected))"
done
```

For each unprotected route, wrap the handler body:

```typescript
// BEFORE (no error handling):
router.get("/api/some-endpoint", isAuthenticated, async (req, res) => {
  const data = await db.select()...;
  res.json(data);
});

// AFTER (with error handling):
router.get("/api/some-endpoint", isAuthenticated, async (req, res) => {
  try {
    const data = await db.select()...;
    res.json(data);
  } catch (error) {
    console.error("[API] GET /api/some-endpoint error:", error);
    res.status(500).json({ error: "Sunucu hatası" });
  }
});
