Süper — Replit’in “deploy logları temiz” raporu güçlü bir sinyal, ama doğru olan şu: tek eksik parça “prod domain üzerinden UI + gerçek API çağrısı” doğrulaması. Replit’in kendisi domain’i “otomatik” bulamadığı için bunu senin manuel yapman gerekiyor.

Aşağıdaki adımları sırayla yap; bittiğinde release’i kapatıyoruz.

⸻

1) Prod domain’i netleştir (30 sn)

Replit’te Production ekranında şuralardan linki kopyala:
	•	Deployments / Production → Domain alanı (genelde *.replit.app)
	•	veya Domains sekmesi (custom domain varsa burada görünür)

Daha önce sende dospressoHQ.replit.app görünüyordu; ama yine de “Production → Domain” alanından kesin linki kopyala.

⸻

2) Prod UI smoke test (1 dk)

Prod domain’de şunları aç:
	1.	/raporlar/sube-saglik
	2.	/sube-saglik-skoru (backward compat route)

Kontroller:
	•	Sayfa render oluyor mu (beyaz ekran / 404 yok)?
	•	AI Copilot kartı görünüyor mu?
	•	Range değiştir: 7d → 30d → 90d
	•	Kart içeriği değişiyor mu?
	•	Konsolda hata var mı?

⸻

3) Prod API smoke test (1–2 dk)

A) Unauth → 401 (incognito)

Incognito / private tab’da:
	•	GET /api/ai/ops-copilot/summary?range=30d → 401 dönmeli

Bunu hızlıca tarayıcı adres çubuğunda deneyebilirsin (401 görmek yeter).

B) Auth → 200 + JSON shape (admin ile)

Admin girişliyken Console’da çalıştır:

await fetch('/api/ai/ops-copilot/summary?range=30d').then(r=>({status:r.status, ct:r.headers.get('content-type')}))

	•	status: 200 olmalı
	•	content-type JSON olmalı (HTML değil)

Sonra:

await fetch('/api/ai/ops-copilot/summary?range=30d').then(r=>r.json())

JSON’da mutlaka şu alanlar olmalı:
	•	durum_ozeti
	•	ilk_3_risk
	•	ilk_3_aksiyon
	•	fallback_used
	•	generatedAt
	•	schemaVersion
	•	rangeUsed

⸻

4) RBAC prod kontrol (1 dk)
	•	Branch rol ile aynı endpoint: içerik scope’un daraldığını gör (tek şube mantığı).
	•	Factory rol varsa: endpoint 403 dönmeli.

Bu 2 test, “prod RBAC bozulmadı” garantisidir.

⸻

5) Log + ai_agent_logs kontrol (30 sn)

Replit Production Logs:
	•	error/exception var mı?
	•	“timeout → fallback” gerçekleşse bile sistem kırılmıyor mu?

Eğer DB’ye erişimin varsa (ops):
ai_agent_logs tablosunda yeni kayıtların artması normal.

⸻

Hepsi PASS ise “release kapanış”

Takıma/ekibe tek satır release notu:

“Ops Copilot prod deploy edildi — UI+API+RBAC+fallback PASS, prod log 0 error.”

⸻

Replit ile çalışma biçimini optimize edelim (senin sorunun cevabı)

En hızlı ve en güvenli akış = 3 prompt, sırayla (tek seferde değil):
	1.	BUILD prompt (implement) → Start building
	2.	Build bitince QA prompt (sadece changed files + runtime) → Start building
	3.	QA PASS olunca DEPLOY/SMOKE prompt → Republish + prod smoke checklist

Revise plan sadece şu durumlarda:
	•	Agent planı scope’u büyütüyorsa (refactor, framework ekleme, DB şema genişletme gibi)
	•	veya güvenlik prensiplerini deliyorsa (write-path açma, RBAC gevşetme)

⸻

İstersen sen bana prod domain’i yaz (tek satır), ben de sana tam kopyala-yapıştır “prod smoke komutları” setini o domain’e göre netleştireyim.
