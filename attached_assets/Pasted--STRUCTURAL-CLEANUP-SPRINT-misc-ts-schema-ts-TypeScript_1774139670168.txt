### Step 5: Handle cross-references

Some tables reference others (foreign keys). When splitting:
- Import the referenced table from its new location
- Example: `factory.ts` references `users` → `import { users } from './users';`
- BUT this creates circular dependencies if users.ts also references factory

**Solution for circular deps:**
- Keep ALL pgTable definitions that have FK references to tables in OTHER files
- Only extract tables that reference tables WITHIN the same file
- OR use Drizzle's `relations()` separately (relations can stay in schema.ts)

**If circular dependency issues arise:**
- Keep the problematic table in schema.ts
- Don't force it — some tables are too interconnected
