2) A) SİSTEM MİMARİSİ ANALİZİ (KANITLI)

2.1 Backend Bütünlük Kontrolü

Çıktı Tablosu: API Catalog
Her endpoint için:
	•	Path, Method, Controller/Handler dosyası
	•	Auth: public / role-based / branch-scoped / kiosk-scoped
	•	Request schema + validation katmanı (ör. zod/joi/class-validator)
	•	Response schema + error formatı (tek standart mı?)
	•	Rate limit / throttling / abuse koruması
	•	Audit log: kim, ne zaman, hangi branch üzerinde işlem yaptı?

Uyumsuzluk Analizi
	•	FE’de kullanılan endpoint listesi çıkar → BE envanteriyle eşleştir
	•	Eksik endpoint / orphan endpoint listesi üret

2.2 Database Bütünlük Kontrolü

Çıktı Tablosu: DB Schema Map
	•	Tablo adı, PK, FK, index, cascade kuralları
	•	Multi-tenant izolasyonu: her tenant/branch tablosunda branch_id zorunlu mu?
	•	Soft delete var mı? (deleted_at) → orphan kontrol yaklaşımı
	•	Migration drift: migration dosyaları vs gerçek DB uyumu

Orphan Kayıt Tespiti
	•	En az şu varlıklar için orphan kontrol: checklist_runs, equipment, tickets, announcements, media_assets, quiz_attempts

2.3 Frontend-Backend Uyum Kontrolü

Çıktı Tablosu: FE↔BE Contract Matrix
	•	FE component/sayfa → çağırdığı endpointler
	•	Beklenen type → dönen type (mismatch)
	•	Cache invalidation (SWR/RTK Query/React Query) stratejisi
	•	Loading/Error/Empty state kontrol listesi

5) D) GÜVENLİK & YETKİ (OWASP + MULTI-TENANT)

5.1 Authentication
	•	Session timeout, refresh token, kiosk token ömrü ve scope
	•	Brute force koruması, MFA opsiyonu (varsa), device binding

5.2 Authorization (Zorunlu Testler)
	•	IDOR: başka şubenin resource ID’si ile erişim dene
	•	Admin-only endpointlere normal user ile erişim dene
	•	Object-level authorization kontrolü (her “GET/UPDATE/DELETE” için)

5.3 Input & Upload Security
	•	XSS, SQLi, SSRF (özellikle URL alanları)
	•	File upload: mime sniffing, AV tarama, boyut limiti, signed URL, path traversal
	•	PDF raporlama: HTML injection / template injection riski

5.4 Logging & Secrets
	•	PII loglanıyor mu?
	•	Secrets env’de mi? repo’da leak var mı?
	•	Audit trail yeterli mi?

⸻

6) E) PERFORMANS & OPTİMİZASYON (ÖLÇÜLEBİLİR)

Hedefler:
	•	Sayfa TTI < 3sn (ortalama mobil)
	•	API p95 < 500ms (kritik endpointlerde)
	•	DB slow query listesi + index önerisi

Üret:
	•	En yavaş 10 API endpoint (varsa ölçümle, yoksa tahmini + “UNVERIFIED”)
	•	En pahalı 10 SQL paterni (JOIN, aggregation)
	•	FE bundle analizi, re-render kaynakları, image lazy load, caching
