GOAL
Excel Import “Quality Pack” ekle: kolon eşleştirme (mapping) + import önizleme + validasyon + duplicate/çakışma raporu. Mevcut import akışını bozma; sadece import başlamadan önce yeni bir “Review” adımı ekle.

SCOPE GUARD (çok kritik)
- Mevcut çalışan import akışını (Excel İçe Aktar -> Simülasyon -> Onayla ve Uygula) KIRMA.
- Yeni UI/logic sadece ADDITIVE olacak: Import öncesi “Review & Confirm” step.
- Backend değişikliği ancak zorunluysa (örn: preview/validate endpoint) yapılabilir; mümkünse existing parse/validation logic reuse et.
- Prod/Preview davranışı aynı kalsın.

CURRENT CONTEXT
- HR import backend endpointleri mevcut:
  - list batches, batch detail, error report, rollback vb.
- Frontend IK sayfasında Import modal + geçmiş importlar + batch detail sheet var.

FEATURES (A paketi)
1) Kolon Eşleştirme (Mapping UI)
   - Excel dosyası seçildikten sonra otomatik eşleştirme öner:
     - “Kullanıcı Adı” -> username
     - “Ad” -> firstName
     - “Soyad” -> lastName
     - “E-posta” -> email
     - “Telefon” -> phone
     - “Rol” -> role
     - “Şube” -> branchName / branchId (sistemde ne varsa)
     - “İşe Giriş” -> startDate
     - “Durum” -> status (active/passive)
   - Kullanıcı mapping’i dropdown ile değiştirebilsin.
   - “Zorunlu alanlar” boş kalırsa Continue kapalı + açıklama göster.

2) Önizleme (Review) + Validasyon (Import başlamadan)
   - “Review” ekranında:
     - Toplam satır sayısı
     - Geçerli satır sayısı
     - Hatalı satır sayısı
     - Duplicate sayısı (username/email/phone bazında)
   - İlk 20 satırı tablo şeklinde göster:
     - satır no, mapping sonrası alanlar, validation sonucu (OK / Warning / Error)
   - Validation kuralları:
     - username zorunlu, regex: [a-zA-Z0-9._-]{3,}
     - email format kontrol (varsa)
     - phone normalize: sadece rakam, TR için +90 / 0 başlangıcı normalize et (soft rule)
     - startDate parse: dd.MM.yyyy / yyyy-MM-dd destekle, parse edilemiyorsa error
     - role eşleştirme: bilinmeyen rol -> warning veya error (policy belirle)
     - branch eşleştirme: bilinmeyen şube -> warning veya error (policy belirle)
   - UI’da Error’ları kırmızı, Warning’leri sarı badge ile göster.

3) Duplicate / Çakışma Raporu
   - Dosya içinde duplicate:
     - username/email/phone tekrar ediyorsa satırları işaretle.
   - Sistemle çakışma (mevcut kullanıcılar):
     - username/email/phone DB’de varsa:
       - Import Mode = Upsert ise: “Update olacak” şeklinde label
       - Import Mode = Create-only ise: “Conflict” error
   - Review ekranında filtreler:
     - All / Only Errors / Only Warnings / Only Conflicts / Only Duplicates

4) Import Onayı (Confirm)
   - “Onayla ve Uygula” butonu sadece:
     - Error sayısı 0 ise aktif
     - (Opsiyon) Warning varsa yine aktif ama “Warnings var” uyarısı ver
   - “Hatalı satırları atla” seçeneği (şu anki davranışla uyumlu) Review ekranında da görünsün.

IMPLEMENTATION NOTES (tercih edilen yaklaşım)

  - Import modalı 3 step yap:
    1) Upload

    3) Review & Confirm
- Backend:
  - Eğer mümkünse: dosyayı parse edip “dry-run validation” yapan yeni endpoint:
    POST /api/hr/employees/import/preview
    Body: file + mapping + mode + options(skipInvalid)
    Response:
      - rowsPreview[0..20] (mapped + issues)
      - counts: total, valid, errors, warnings, duplicatesInFile, conflictsInDb
      - issues list: {rowNumber, field, code, message, severity}
  - Import execute endpoint mevcutsa reuse et; yoksa current flow’u bozmadan import’u başlatacak şekilde entegre et.

  - Preview için ilk 500 satır üzerinde full validation; daha büyükte sampling + count; ama UI’ya açıkça yaz.
