## TASK: Sprint 25 — Security Hardening (Pre-Pilot Critical)

This is the final sprint before pilot launch. Fix all security gaps identified in the IT audit.

### T001 — Helmet.js Security Headers (~1 hour)

**Problem:** IT audit reports Helmet/CSP may not be active. Verify and fix.

```bash
# Check if helmet is installed
grep -rn "helmet" package.json | head -3

# Check if helmet is used in server
grep -rn "helmet\|Helmet" server/ --include="*.ts" | head -10

# Check actual response headers
curl -I http://localhost:5000/ 2>/dev/null | grep -i "x-frame\|content-security\|x-content-type\|strict-transport\|x-xss\|permissions-policy\|referrer-policy"
```

**If Helmet is NOT active, add it:**

```typescript
import helmet from 'helmet';

// Add BEFORE all routes
app.use(helmet({
  contentSecurityPolicy: {
    directives: {
      defaultSrc: ["'self'"],
      scriptSrc: ["'self'", "'unsafe-inline'", "'unsafe-eval'", "https://cdnjs.cloudflare.com"],
      styleSrc: ["'self'", "'unsafe-inline'", "https://fonts.googleapis.com"],
      fontSrc: ["'self'", "https://fonts.gstatic.com"],
      imgSrc: ["'self'", "data:", "blob:", "https://*.amazonaws.com", "https://*.replit.dev"],
      connectSrc: ["'self'", "https://api.openai.com", "wss://*"],
      frameSrc: ["'none'"],
      objectSrc: ["'none'"],
      baseUri: ["'self'"]
    }
  },
  crossOriginEmbedderPolicy: false, // Required for external images
  crossOriginResourcePolicy: { policy: "cross-origin" } // Required for S3 images
}));
```

**If Helmet IS active but CSP is wrong, fix the directives.**

**Verify after fix:**
```bash
curl -I http://localhost:5000/ 2>/dev/null | grep -i "x-frame\|content-security\|strict-transport\|x-content-type"
```

Expected:
- `X-Frame-Options: DENY` or `SAMEORIGIN`
- `Content-Security-Policy: ...`
- `X-Content-Type-Options: nosniff`
- `Strict-Transport-Security: ...`

**IMPORTANT:** After adding Helmet, test that the app still loads correctly. CSP can break inline scripts, external fonts, or API calls if directives are too strict. If the frontend breaks, loosen the specific directive that's blocking.

### T002 — Rate Limiting (~1 hour)

**Problem:** Rate limiting package installed but may not be active. Brute-force risk on login and sensitive endpoints.

```bash
# Check if express-rate-limit is installed
grep -rn "rate-limit\|rateLimit\|rate_limit" package.json server/ --include="*.ts" | head -10

# Check if it's applied
grep -rn "rateLimiter\|limiter\|windowMs\|max:" server/ --include="*.ts" | head -10
```

**Add rate limiting to these critical endpoints:**

```typescript
import rateLimit from 'express-rate-limit';

// Strict limit for login (prevent brute-force)
const loginLimiter = rateLimit({
  windowMs: 15 * 60 * 1000, // 15 minutes
  max: 10, // 10 attempts per window
  message: { error: 'Çok fazla giriş denemesi. 15 dakika sonra tekrar deneyin.' },
  standardHeaders: true,
  legacyHeaders: false,
  keyGenerator: (req) => req.ip || req.connection.remoteAddress || 'unknown'
});

// Medium limit for API endpoints
const apiLimiter = rateLimit({
  windowMs: 1 * 60 * 1000, // 1 minute
  max: 100, // 100 requests per minute
  message: { error: 'İstek limiti aşıldı. Lütfen bekleyin.' }
});

// Strict limit for password reset
const passwordResetLimiter = rateLimit({
  windowMs: 60 * 60 * 1000, // 1 hour
  max: 3,
  message: { error: 'Şifre sıfırlama limiti aşıldı.' }
});

// Apply
app.use('/api/login', loginLimiter);
app.use('/api/auth/login', loginLimiter); // Check which login path is used
app.use('/api/forgot-password', passwordResetLimiter);
app.use('/api/admin', apiLimiter);
```

Find the correct login endpoint path:
```bash
grep -rn "login\|Login" server/routes/ --include="*.ts" | grep "router\.\(post\|get\)" | head -5
```

**Also add rate limiting to kiosk PIN login:**
```bash
grep -rn "pin\|kiosk.*auth\|kiosk.*login" server/routes/ --include="*.ts" | head -5
```

### T003 — Kiosk Endpoint Security (~2 hours)

**Problem:** 14 kiosk endpoints lack isAuthenticated middleware. They use PIN-based auth but not all endpoints verify the PIN/session.

```bash
# Find all kiosk-related endpoints
grep -rn "kiosk\|factory.*quick\|factory.*pin" server/routes/ --include="*.ts" | grep "router\.\(get\|post\|put\|patch\|delete\)" | head -20
```

**Current kiosk auth flow:**
1. User enters PIN on kiosk → POST to PIN verify endpoint → returns kiosk JWT token
2. Subsequent kiosk requests should include this JWT token
3. But some endpoints don't verify the token

**Fix:** Create a `isKioskAuthenticated` middleware that verifies the kiosk JWT:

```bash
# Check existing kiosk auth
grep -rn "isKioskAuthenticated\|kioskAuth\|kiosk.*token\|kiosk.*jwt" server/ --include="*.ts" | head -10
```

If it exists, apply it to ALL kiosk endpoints that don't have it.
If it doesn't exist, create it:

```typescript
function isKioskAuthenticated(req, res, next) {
  const token = req.headers['x-kiosk-token'] || req.query.kioskToken;
  if (!token) return res.status(401).json({ error: 'Kiosk doğrulaması gerekli' });
  
  try {
    const decoded = jwt.verify(token, process.env.KIOSK_SECRET || 'kiosk-secret');
    req.kioskUser = decoded;
    next();
  } catch {
    return res.status(401).json({ error: 'Geçersiz kiosk oturumu' });
  }
}
```

Apply to ALL factory kiosk endpoints:
- quick-start, quick-complete, quick-end
- break start/end
- station switch
- fault report from kiosk
- production output

**DO NOT apply to:**
- The PIN login endpoint itself (that's the entry point)
- Public health check endpoints

