Claude (IT danışman) raporu okudu, bir şüphe ortaya çıktı:
Bu "bug" aslında B.1 SQL'inde TIMEZONE hatası OLABİLİR.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
HİPOTEZ
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

B.1 query:
  LEFT JOIN pdks_records pr ON pr.user_id = sa.user_id
    AND pr.record_date = s.shift_date   ← sorun burada

Dikkat: shift_attendance.check_in_time = UTC timestamp,
pdks_records.record_date = Türkiye saatine göre DATE.

Basri Şen 29 Mart 21:13 UTC check-in:
  - Bu TR saatinde 30 Mart 00:13
  - pdks_records.record_date = '2026-03-30' yazılmış OLABİLİR
  - shift.shift_date = '2026-03-29'
  - JOIN eşleşmez → "missing" FALSE POSITIVE

Yani pilot bug YOK, B.1 query'min kendisi yanlış ölçüyor olabilir.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
DOĞRULAMA SQL — 4 ŞÜPHELİ KAYIT İÇİN
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

SELECT
  u.first_name || ' ' || u.last_name AS user_name,
  s.shift_date AS shift_day,
  sa.check_in_time AT TIME ZONE 'UTC' AT TIME ZONE 'Europe/Istanbul' AS check_in_tr,

  -- Aynı gün pdks?
  (SELECT COUNT(*) FROM pdks_records pr
   WHERE pr.user_id = sa.user_id
     AND pr.record_date = s.shift_date
     AND pr.record_type = 'giris') AS pdks_same_day,

  -- Ertesi gün (UTC→TR geçiş)?
  (SELECT COUNT(*) FROM pdks_records pr
   WHERE pr.user_id = sa.user_id
     AND pr.record_date = s.shift_date + INTERVAL '1 day'
     AND pr.record_type = 'giris') AS pdks_next_day,

  -- Önceki gün?
  (SELECT COUNT(*) FROM pdks_records pr
   WHERE pr.user_id = sa.user_id
     AND pr.record_date = s.shift_date - INTERVAL '1 day'
     AND pr.record_type = 'giris') AS pdks_prev_day,

  -- HİÇBİR GÜN PDKS YOK MU? (±1 gün window)
  (SELECT COUNT(*) FROM pdks_records pr
   WHERE pr.user_id = sa.user_id
     AND pr.record_date BETWEEN s.shift_date - INTERVAL '1 day'
                            AND s.shift_date + INTERVAL '1 day'
     AND pr.record_type = 'giris') AS pdks_in_window

FROM shift_attendance sa
INNER JOIN shifts s ON sa.shift_id = s.id
LEFT JOIN users u ON sa.user_id = u.id
WHERE sa.check_in_time IS NOT NULL
  AND s.shift_date >= NOW() - INTERVAL '30 days'
  AND NOT EXISTS (
    SELECT 1 FROM pdks_records pr2
    WHERE pr2.user_id = sa.user_id
      AND pr2.record_date = s.shift_date
      AND pr2.record_type = 'giris'
  )
ORDER BY s.shift_date;

YORUMLAMA
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

pdks_in_window = 1 → TIMEZONE FALSE POSITIVE (pipeline sağlam)
                     → B.1 query'min kendisi düzeltilecek
                     → Pilot GO, değişiklik yok

pdks_in_window = 0 → GERÇEK BUG (pipeline kopuk)
                     → Pazartesi sabah fix zorunlu
                     → 3 kayıt backfill gerekli

Özellikle Basri Şen 29.03 21:13 UTC kaydı için pdks_next_day
değerine dikkat et — bu en büyük timezone adayı.
