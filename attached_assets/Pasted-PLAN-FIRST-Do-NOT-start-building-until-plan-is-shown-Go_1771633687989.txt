Goal:
Admin should be able to create/edit custom Titles (job titles) and Roles, assign permissions, and apply them to users. This must work for HQ + Factory + Branch.

Current context:
We already have users, roles/permissions UI, and admin area. Problem: admin needs to create new titles/roles from UI (not hardcoded), and permissions must control what modules/screens/actions appear. Admin user must never be deletable or demoted.

Requirements:
1) Data model
- If we already have tables for roles/permissions, extend them; otherwise create:
  - roles (id, name, scope: HQ/Factory/Branch/All, isSystem, isDeletable)
  - titles (id, name, scope, isSystem, isDeletable)
  - permissions (id, key, description, module)
  - role_permissions (roleId, permissionId)
  - user_roles (userId, roleId) OR a single roleId on users if that’s current approach
  - user_title (userId, titleId) OR titleId on users
- Add guards: Admin role isSystem=true and isDeletable=false; cannot be removed from admin user.

2) UI/Admin
- Admin > Yönetim & Ayarlar > “Rol ve Yetki Yönetimi”:
  - Create Role (name, scope)
  - Edit Role: toggle permissions by module (view/create/edit/approve/export/import)
  - Create Title (job title) with scope
  - Assign Title to user
- Add “Yeni Rol Ekle” and “Yeni Title Ekle” buttons where appropriate.
- Ensure permission keys drive sidebar/menu visibility consistently.

3) Backend API
- POST/GET/PATCH roles, titles
- POST/DELETE role_permissions
- PATCH user role/title assignment
- Ensure auth middleware blocks unauthorized actions.

4) Safety
- Migration must be reversible.
- Existing roles/titles must be migrated/seeded without breaking current users.
- Admin cannot be deleted, demoted, or modified by import.
