### T004 — Lot/Parti İzlenebilirlik Sistemi (~2-3 saat)

**Gıda güvenliği için ZORUNLU — müşteri sorun bildirdiğinde tüm zincir geriye izlenebilmeli.**

**A) production_lots tablosu:**
```sql
CREATE TABLE IF NOT EXISTS production_lots (
  id SERIAL PRIMARY KEY,
  lot_number VARCHAR(50) UNIQUE NOT NULL,
  product_id INTEGER REFERENCES factory_products(id),
  production_record_id INTEGER REFERENCES production_records(id),
  quantity DECIMAL(10,2) NOT NULL,
  unit VARCHAR(20) NOT NULL,
  production_date TIMESTAMP NOT NULL DEFAULT NOW(),
  expiry_date TIMESTAMP,
  status VARCHAR(20) DEFAULT 'produced',
  quality_status VARCHAR(20) DEFAULT 'pending',
  operator_user_id VARCHAR(255) REFERENCES users(id),
  station_id INTEGER REFERENCES factory_stations(id),
  storage_location VARCHAR(100),
  parent_lot_id INTEGER REFERENCES production_lots(id),
  notes TEXT,
  created_at TIMESTAMP DEFAULT NOW()
);
CREATE INDEX idx_lot_number ON production_lots(lot_number);
CREATE INDEX idx_lot_product ON production_lots(product_id);
CREATE INDEX idx_lot_expiry ON production_lots(expiry_date);
CREATE INDEX idx_lot_status ON production_lots(status);
```

**B) lot_ingredients tablosu:**
```sql
CREATE TABLE IF NOT EXISTS lot_ingredients (
  id SERIAL PRIMARY KEY,
  lot_id INTEGER REFERENCES production_lots(id) NOT NULL,
  raw_material_id INTEGER,
  inventory_id INTEGER,
  supplier_lot_number VARCHAR(100),
  quantity_used DECIMAL(10,2),
  unit VARCHAR(20),
  created_at TIMESTAMP DEFAULT NOW()
);
```

**C) Lot numarası:** LOT-YYYY-MM-DD-NNN (her gün 001'den başlar)

### T005 — Yarı Mamül → Mamül Zinciri & Anti-Suistimal (~3-4 saat)

**A) production_records yeni kolonlar:**
```sql
ALTER TABLE production_records ADD COLUMN IF NOT EXISTS parent_production_id INTEGER REFERENCES production_records(id);
ALTER TABLE production_records ADD COLUMN IF NOT EXISTS semi_finished_product_id INTEGER;
ALTER TABLE production_records ADD COLUMN IF NOT EXISTS production_stage VARCHAR(20) DEFAULT 'single';
ALTER TABLE production_records ADD COLUMN IF NOT EXISTS lot_id INTEGER REFERENCES production_lots(id);
ALTER TABLE production_records ADD COLUMN IF NOT EXISTS waste_quantity DECIMAL(10,2) DEFAULT 0;
ALTER TABLE production_records ADD COLUMN IF NOT EXISTS waste_reason TEXT;
```

**B) İki aşamalı üretim (/api/production/complete genişlet):**

```
Aşama 1 — Yarı Mamül:
{ recipe_id, produced_quantity: 1000, production_stage: 'semi_finished', station_id, waste_quantity: 50 }
→ Hammaddeler düşer → 1000 yarı mamül stoka → LOT oluşur

Aşama 2 — Mamül:
{ recipe_id, produced_quantity: 900, production_stage: 'finishing', parent_production_id, semi_finished_product_id, station_id, waste_quantity: 100 }
→ ANTİ-SUİSTİMAL: 900 <= yarı mamül stok(1000) ✅
→ Yarı mamül stoktan 900 düşer → Süsleme hammaddeleri düşer → 900 mamül stoka → LOT oluşur (parent bağlı)
```

**C) Anti-suistimal kuralları:**
1. Mamül miktarı > yarı mamül stok → ENGELLE + Türkçe hata: "Yarı mamül stokta sadece X adet var, Y adet üretilemez"
2. Fire oranı %5+ → supervisor bildirimi
3. Fire oranı %10+ → fabrika müdürü bildirimi
4. Kiosk'tan girilen miktar da aynı kontrole tabi

**D) Kiosk → production_records entegrasyonu:**

Kiosk'ta operatör "Tepsi Tamamla" basınca arka planda:
1. factory_production_outputs kaydı (mevcut — kiosk takibi)
2. **YENİ:** production_records kaydı (maliyet takibi)
3. **YENİ:** Hammadde otomatik düşme
4. **YENİ:** LOT otomatik oluşturma
5. **YENİ:** Kalite kontrole düşme

**Operatör bunların hiçbirini görmez — sadece miktar ve fire girer.**
