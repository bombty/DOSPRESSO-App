Fix branch kiosk password hashing issue. This is critical — 19 branches cannot login to kiosk.

## ROOT CAUSE
Two problems:
1. 19 branches have plain text "0000" in branches.kioskPassword — /api/login rejects them
2. branch_kiosk_settings uses plain text comparison, branches table uses bcrypt — inconsistent

## FIX 1 — Hash all existing plain text passwords in DB

Run this migration on startup (check if already hashed first):
```typescript
import bcrypt from 'bcrypt';

export async function migrateKioskPasswords() {
  try {
    // Fix branches.kioskPassword — hash all plain text passwords
    const result = await db.execute(sql`
      SELECT id, kiosk_password FROM branches 
      WHERE kiosk_password IS NOT NULL 
      AND deleted_at IS NULL
    `);
    
    for (const branch of result.rows) {
      const pwd = branch.kiosk_password as string;
      // Skip if already bcrypt hashed
      if (pwd.startsWith('$2b$') || pwd.startsWith('$2a$')) continue;
      
      const hashed = await bcrypt.hash(pwd, 10);
      await db.execute(sql`
        UPDATE branches SET kiosk_password = ${hashed} 
        WHERE id = ${branch.id}
      `);
    }
    
    // Fix branch_kiosk_settings.kiosk_password too
    const settings = await db.execute(sql`
      SELECT id, kiosk_password FROM branch_kiosk_settings 
      WHERE kiosk_password IS NOT NULL
    `);
    
    for (const setting of settings.rows) {
      const pwd = setting.kiosk_password as string;
      if (pwd.startsWith('$2b$') || pwd.startsWith('$2a$')) continue;
      
      const hashed = await bcrypt.hash(pwd, 10);
      await db.execute(sql`
        UPDATE branch_kiosk_settings SET kiosk_password = ${hashed}
        WHERE id = ${setting.id}
      `);
    }
    
    console.log('[Kiosk] Password migration complete');
  } catch (err) {
    console.error('[Kiosk] Password migration failed:', err);
  }
}
```

Register in server/index.ts startup (run once, after DB ready).
