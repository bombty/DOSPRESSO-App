## FIX 1: Lock Viewport — No Scroll

The kiosk runs on a TABLET in a factory. It must NEVER scroll. The entire UI must fit in the viewport.

### Approach:
```typescript
// The outermost wrapper of the kiosk page must be:
<div className="h-screen w-screen overflow-hidden flex flex-col bg-background">
  
  {/* Fixed header — always visible */}
  <div className="flex-shrink-0 h-12 border-b border-border flex items-center justify-between px-4">
    <span className="text-sm font-medium">Üretim Takip Sistemi</span>
    <div className="flex items-center gap-2">
      {/* fullscreen button */}
      {/* Kiosk'tan Çık button */}
    </div>
  </div>
  
  {/* Content area — fills remaining height, internal scroll if needed */}
  <div className="flex-1 overflow-y-auto">
    {/* All step content goes here */}
  </div>
  
</div>
```

Key changes:
- Outer div: `h-screen overflow-hidden` — locks to viewport
- Header: `flex-shrink-0` — never compressed, always visible
- Content: `flex-1 overflow-y-auto` — scrolls internally if content is tall, but header stays fixed

### 2C: Data query for Mr. Dobody alerts
```bash
# Check what agent guidance endpoint returns
grep -n "guidance\|pending.*action" server/routes/agent.ts | head -10
```

Add a query to fetch factory-relevant agent alerts:
```typescript
// Mr. Dobody alerts for factory
const dobodyQuery = useQuery({
  queryKey: ['/api/agent/guidance'],
  queryFn: async () => {
    try {
      const res = await kioskFetch('/api/agent/guidance');
      if (!res.ok) return [];
      const data = await res.json();
      // Filter for factory-relevant alerts only
      const factoryAlerts = (data.gaps || data.alerts || data || []).filter((item: any) => 
        item.category === 'factory' || 
        item.module === 'fabrika' ||
        item.tags?.includes('factory') ||
        item.tags?.includes('üretim')
      );
      return factoryAlerts;
    } catch {
      return [];
    }
  },
  enabled: step === 'select-user' || step === 'worker-home',
  refetchInterval: 300000, // 5 minutes
  retry: false,
});
const dobodyAlerts = dobodyQuery.data || [];
```

### 2D: If guidance endpoint doesn't work for kiosk auth

The guidance endpoint might require web session (not kiosk token). If so, check:
```bash
grep -B3 "guidance" server/routes/agent.ts | grep "isAuthenticated\|isKiosk" | head -3
```

If it uses `isAuthenticated` (which already has kiosk fallback), it should work.
If it uses a different auth, add kiosk support OR create a simpler endpoint:
```typescript
// GET /api/factory/kiosk/alerts — factory-specific alerts for kiosk display
router.get('/api/factory/kiosk/alerts', isKioskAuthenticated, async (req, res) => {
  try {
    // Get recent factory-related agent actions
    const alerts = await db.select({
      id: agentPendingActions.id,
      title: agentPendingActions.title,
      description: agentPendingActions.description,
      severity: agentPendingActions.severity,
      createdAt: agentPendingActions.createdAt,
    })
    .from(agentPendingActions)
    .where(and(
      eq(agentPendingActions.category, 'production'),
      eq(agentPendingActions.status, 'pending'),
      gte(agentPendingActions.createdAt, sql`NOW() - INTERVAL '24 hours'`)
    ))
    .orderBy(desc(agentPendingActions.createdAt))
    .limit(5);
    
    res.json(alerts);
  } catch (error) {
    res.json([]); // Silent fail — kiosk shouldn't crash for alerts
  }
});
```

Use this endpoint instead if the main guidance endpoint doesn't support kiosk auth.
