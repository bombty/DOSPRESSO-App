## ADIM 1: canViewSalary Bug Fix (15 dk)

**Sorun:** hr.ts satır ~4560 civarı — canViewSalary() fonksiyonu CEO, CGO, Muhasebe_IK rollerini reddediyor. Sadece admin ve muhasebe true dönüyor.

**Bul ve düzelt:**

```bash
echo "=== canViewSalary mevcut durumu ==="
grep -n "canViewSalary" server/routes/hr.ts | head -10
echo ""
echo "--- Fonksiyon içeriği ---"
grep -A20 "const canViewSalary" server/routes/hr.ts
```

**Düzeltme kuralı:**
- CEO → tüm maaşları görebilmeli (return true)
- CGO → tüm maaşları görebilmeli (return true)  
- Muhasebe_IK → tüm maaşları görebilmeli (return true)
- Coach → SADECE şube personeli maaşını görebilmeli (HQ personeli göremez)
- Trainer → HİÇBİR maaş göremez
- Yatırımcı → SADECE kendi şubesi maaşını görebilmeli
- Admin + Muhasebe → her şeyi görebilir (mevcut — dokunma)

**Önerilen implementasyon:**
```typescript
const canViewSalary = (viewerRole: string, targetUserId: string, viewerUserId: string, viewerBranchId: number | null, targetBranchId: number | null): boolean => {
    // Tam erişim: admin, muhasebe, ceo, cgo, muhasebe_ik
    if (['admin', 'muhasebe', 'ceo', 'cgo', 'muhasebe_ik'].includes(viewerRole)) return true;
    
    // Coach: sadece şube personeli (HQ personeli hariç)
    if (viewerRole === 'coach') {
        return targetBranchId !== null; // HQ (null branch) personeli göremez
    }
    
    // Yatırımcı: sadece kendi şubesi
    if (viewerRole === 'yatirimci_branch') {
        return viewerBranchId === targetBranchId;
    }
    
    // Kendi maaşını herkes görebilir
    if (viewerUserId === targetUserId) return true;
    
    // Diğer herkes: göremez
    return false;
};
```

**Test:**
```bash
echo "=== canViewSalary TEST ==="
# CEO bordro sayfasını açabiliyor mu?
curl -s -b /tmp/ceo_cookie http://localhost:5000/api/salary/employees 2>/dev/null | head -5
echo ""
echo "Düzeltme sonrası CEO, CGO, Muhasebe_IK true dönmeli"
```

## ADIM 3: HQ_ROLES Konsolidasyonu (30 dk)

**Sorun:** Aynı HQ_ROLES sabiti 5 farklı dosyada 5 farklı şekilde tanımlı. Trainer bazı listelerde yok.

### 3A: Merkezi tanım oluştur

```typescript
/**
 * HQ rolleri — şubeye bağlı olmayan, merkez ofis rolleri.
 * TÜM dosyalar bu tek tanımı import etmeli.
 */
export const HQ_ROLES = [
  'admin', 'ceo', 'cgo', 'coach', 'trainer',
  'muhasebe_ik', 'satinalma', 'marketing',
  'kalite_kontrol', 'gida_muhendisi', 'fabrika_mudur',
  'yatirimci_hq'

export type HQRole = typeof HQ_ROLES[number];

/** HQ rolü mü kontrolü */
export const isHQRole = (role: string): boolean => HQ_ROLES.includes(role as HQRole);
```

## ADIM 4: Trainer Eksik Modüller (30 dk)

**Sorun:** Trainer, Coach'un erişebildiği modüllerin %67'sine erişemiyor. academy.ts HQ_ROLES'ta trainer yoktu (Adım 3 ile düzeldi). Ama role_module_permissions'da da eksikler olabilir.

```bash
echo "=== Coach vs Trainer modül karşılaştırması ==="
echo "
SELECT 'coach_only' as status, module, actions 
FROM role_module_permissions 
WHERE role = 'coach' 
AND module NOT IN (SELECT module FROM role_module_permissions WHERE role = 'trainer')
ORDER BY module;
" | psql

echo ""
echo "--- Trainer'ın mevcut modül sayısı ---"
echo "SELECT COUNT(*) FROM role_module_permissions WHERE role = 'trainer';" | psql

echo ""
echo "--- Coach'un mevcut modül sayısı ---"
echo "SELECT COUNT(*) FROM role_module_permissions WHERE role = 'coach';" | psql
```

**Düzeltme kuralı:**
Trainer'ın erişmesi gereken modüller (Coach ile aynı olmalı, şunlar HARİÇ):
- `accounting` → Trainer maaş GÖREMEMELİ
- `admin` → Trainer admin paneline GİREMEMELİ
- Finans/bordro → Trainer erişmemeli

Trainer'ın KESİNLİKLE erişmesi gerekenler:
- `akademi` + tüm alt modülleri (trainer'ın asıl işi!)
- `gorevler`, `checklist` (şube operasyonu izleme)
- `raporlar` (performans takibi)
- `denetim` (kalite denetimi)
- `ekipman` (ekipman izleme)
- `iletisim_merkezi` (iletişim)
- `dobody`, `dobody.bildirim` (Mr. Dobody)

```sql
-- Trainer'a eksik modülleri ekle (Coach'ta olup Trainer'da olmayanlar)
-- Finans/bordro/admin hariç
INSERT INTO role_module_permissions (role, module, actions)
SELECT 'trainer', module, actions
FROM role_module_permissions
WHERE role = 'coach'
AND module NOT IN (SELECT module FROM role_module_permissions WHERE role = 'trainer')
AND module NOT IN ('accounting', 'admin', 'bordro', 'finans')
ON CONFLICT (role, module) DO NOTHING;

-- Doğrula
SELECT COUNT(*) as trainer_modules FROM role_module_permissions WHERE role = 'trainer';
SELECT COUNT(*) as coach_modules FROM role_module_permissions WHERE role = 'coach';
```
