## ADIM 1: YENİ SKİLL — branch-task-tracker.ts (1.5 saat)

**Amaç:** Şube görevlerini takip eden, hatırlatan, gecikme uyarısı veren skill.

**Hedef roller:** supervisor, mudur, coach, trainer
**Schedule:** daily (sabah) + hourly (gün içi kontrol)

### 1A: Skill dosyası oluştur

Mevcut skill'lerin yapısını referans al (production-director.ts veya team-tracker.ts). Aynı pattern'ı takip et.

**Skill'in yapması gerekenler:**

```
GÜNLÜK (sabah — skill-daily tetiklemesiyle):
1. Bugün tamamlanması gereken görevleri say
2. Dünden kalan (overdue) görevleri say
3. Supervisor/müdür için özet bildirim oluştur:
   "Bugün 5 görev var (2 dünden kalan). 3'ü açık havuzda."

SAATLIK (skill-hourly tetiklemesiyle, varsa):
4. Açık havuzda 3+ saat bekleyen görev varsa uyar:
   "2 görev 3+ saattir sahipsiz"
5. Bugünkü görevlerden tamamlanmayanları kontrol et

HAFTALIK (Pazartesi — skill-weekly tetiklemesiyle):
6. Geçen haftanın tamamlama oranını hesapla
7. Coach + müdür için karşılaştırma:
   "Geçen hafta %78 tamamlama (önceki hafta %85 — düşüş)"
```

**Veri kaynakları:**
```sql
-- Bugünkü görevler
SELECT * FROM branch_task_instances 
WHERE DATE(due_date) = CURRENT_DATE AND branch_id = ?;

-- Overdue görevler
SELECT * FROM branch_task_instances 
WHERE status = 'pending' AND due_date < NOW() AND branch_id = ?;

-- Açık havuzda bekleyenler
SELECT * FROM branch_task_instances 
WHERE status = 'pending' AND claimed_by_user_id IS NULL AND branch_id = ?;

-- Haftalık tamamlama oranı
SELECT 
  COUNT(CASE WHEN status = 'completed' THEN 1 END) as tamamlanan,
  COUNT(*) as toplam
FROM branch_task_instances
WHERE due_date BETWEEN ? AND ? AND branch_id = ?;
```

**⚠ DİKKAT:** branch_task_instances tablosunun kolon isimlerini önce kontrol et:
```bash
echo "SELECT column_name FROM information_schema.columns WHERE table_name = 'branch_task_instances' ORDER BY ordinal_position;" | psql
```
Eğer kolon isimleri farklıysa (due_date yerine scheduled_date, vb.) → kodu uyarla.

### 1B: Skill'i registry'ye kaydet

Mevcut skill-registry.ts'nin pattern'ına göre yeni skill'i ekle. **Önce mevcut registry'yi oku, sonra ekle.**

### 1C: Agent engine'e entegre et

`analyzeForBranchMgmt` fonksiyonuna branch-task-tracker'ı ekle (supervisor, mudur için).
`analyzeForHQOps` fonksiyonuna da ekle (coach, trainer için).

**⚠ DANIŞMA NOKTASI:** Eğer agent engine'deki analyze fonksiyonları hardcoded skill listesi kullanıyorsa ve yeni skill eklemek yapıyı değiştirecekse → DUR, mevcut yapıyı göster, onay al.

## ADIM 2: YENİ SKİLL — factory-daily-summary.ts (1 saat)

**Amaç:** Fabrika günlük üretim/fire/QC özetini müdüre ve üst yönetime raporlayan skill.

**Hedef roller:** fabrika_mudur, ceo, cgo
**Schedule:** daily (akşam 18:00 veya skill-daily tetiklemesiyle)

**Skill'in yapması gerekenler:**

```
GÜNLÜK:
1. Bugünkü toplam üretim miktarı (tüm istasyonlar)
2. Bugünkü toplam fire miktarı + fire oranı
3. QC bekleyen LOT sayısı
4. QC onaylanan/reddedilen sayısı
5. Fabrika müdüre özet bildirim:
   "Bugün: 450 adet üretim, %2.3 fire, 3 LOT QC bekliyor, 2 QC onaylandı"
6. CEO/CGO'ya sadece önemli sapmalarda:
   "Fabrika: Fire oranı %5.2 (hedef: <%3)" → sadece eşik aşımında

HAFTALIK (Pazartesi):
7. Geçen haftanın üretim trendi
8. İstasyon bazlı verimlilik karşılaştırması
9. Çalışan skor özeti (ortalama + en düşük)
```

**Veri kaynakları:**
```sql
-- Bugünkü üretim
SELECT SUM(produced_quantity) as toplam_uretim, SUM(waste_quantity) as toplam_fire
FROM factory_production_outputs 
WHERE DATE(created_at) = CURRENT_DATE;

-- QC bekleyen
SELECT COUNT(*) FROM production_lots WHERE status = 'kalite_bekliyor';

-- QC sonuçları (bugün)
SELECT decision, COUNT(*) FROM factory_quality_checks 
WHERE DATE(checked_at) = CURRENT_DATE GROUP BY decision;

-- Haftalık skor ortalaması
SELECT AVG(total_score), MIN(total_score), MAX(total_score) 
FROM factory_worker_scores 
WHERE period_type = 'weekly' AND period_date >= CURRENT_DATE - INTERVAL '7 days';
```

**⚠ DANIŞMA NOKTASI:** Eğer mevcut production-director.ts veya waste-analyzer.ts skill'leri zaten benzer veri çekiyorsa → DUR, overlap'i göster. Birleştirmek mi ayrı tutmak mı daha iyi? Onay al.

**Bildirim spam önleme:** CEO/CGO'ya sadece eşik aşımında bildirim (fire >%3, QC red >%10, üretim hedefin <%80). Normal günlerde gönderme.

**Skill'in yapması gerekenler (veri varsa):**

```
GÜNLÜK:
1. Bugün geç kalan personeli tespit et
   (actual_start > planned_start + 30 dakika → geç)
2. ≤30 dk geç → personele bildirim: "Bugün 15 dk geç kaldınız"
3. >1 saat geç → supervisor'a anlık bildirim

AYLIK (her ayın 1'i):
4. Geçen ay kaç kez geç kalındığını hesapla (kişi bazında)
5. 2+ kez geç kalan → supervisor uyarı
6. 4+ kez geç kalan → supervisor + yatırımcı + CGO + trainer
7. Şube geneli oran yüksekse (>%20 personel düzenli geç) → CGO

ÖNEMLİ KURAL:
- Bireysel günlük gecikmeler HQ'ya BİLDİRİLMEZ
- Sadece şube geneli sistemik sorun → CGO
- Aylık 4+ tekrar → escalation (bireysel ama ciddi)
```

```bash
echo "=== Öksüz sayfa tam listesi ==="
find client/src/pages -name "*.tsx" -not -path "*/components/*" | while read f; do
  base=$(basename "$f" .tsx)
  if ! grep -q "$base" client/src/App.tsx 2>/dev/null; then
```
