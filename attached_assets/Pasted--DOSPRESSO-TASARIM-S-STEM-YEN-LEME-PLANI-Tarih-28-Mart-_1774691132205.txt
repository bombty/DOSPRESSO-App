# ÖNCEKİ HATALARIN ANALİZİ VE ÇÖZÜMLER

| Hata | Neden | Bu Sefer Ne Yapacağız |
|------|-------|----------------------|
| Mockup ile kod uyuşmadı | Tailwind class'ları inline stil'leri override etti | Merkezi CSS token dosyası + !important olmadan cascade kontrolü |
| Font'lar çok küçük kaldı | 7-8px değerler ekranda neredeyse görünmüyor | Minimum 11px kuralı, tüm boyutlar tek dosyadan |
| Max-width 800px → ekran boş | Sabit değer kullandım | Responsive grid: minmax(280px, 1fr) |
| Parça parça yapıldı | Sadece HomeScreen'e odaklandım | Katmanlı mimari: token → component → page |
| Her push "bu sefer oldu" | Sonucu göremiyorum | Her katmanda test komutu + Replit doğrulama |
| Alt sayfalar eski kaldı | Global layout değişmedi | App.tsx layout sistemi yeniden tasarlanacak |

## Katman 2: Shared Layout Bileşenleri (YENİ)

### 2a. PageShell — Tüm sayfaları sarar
```
client/src/components/layout/PageShell.tsx

Props:
  - title: string (sayfa başlığı)
  - subtitle?: string
  - showBackButton?: boolean (→ Ana Sayfa)
  - showDateFilter?: boolean
  - rightActions?: ReactNode
  - children: ReactNode
  - fullWidth?: boolean (default true)

Yapısı:
  <div maxWidth=1200px, padding=20px 24px>
    <header: ← Ana Sayfa | Sayfa Başlığı | Filtre butonu>
    <main: children>
  </div>
```

### 2b. WidgetCard — Tüm widget'lar için
```
client/src/components/layout/WidgetCard.tsx

Props:
  - title: string
  - icon?: LucideIcon
  - badge?: { label: string, color: string }
  - span?: 1 | 2 | 3 (grid column span)
  - onClick?: () => void
  - children: ReactNode

Stil: var(--ds-card-*) token'larından beslenir
```

### 2c. KPIBar — Üst KPI şeridi
```
client/src/components/layout/KPIBar.tsx

Props:
  - items: Array<{ value: string|number, label: string, color: string }>

Yapısı: flex row, her item card-style
```

### 2d. WidgetGrid — Widget yerleşim container
```
client/src/components/layout/WidgetGrid.tsx

Props:
  - columns?: 2 | 3 (default 3)
  - children: ReactNode

Stil: CSS Grid, responsive, gap var(--ds-gap-md)
```

### 2e. ModuleTabBar — Alt sayfa navigasyonu (sidebar yerine)
```
client/src/components/layout/ModuleTabBar.tsx

Props:
  - tabs: Array<{ id: string, label: string, icon: LucideIcon }>
  - activeTab: string
  - onChange: (id: string) => void

Desktop: yatay tab bar (sayfa üstünde)
Mobil: aynı yatay tab bar (scroll edilebilir)
```

# RİSK ANALİZİ

| Risk | Olasılık | Etki | Önlem |
|------|----------|------|-------|
| Sidebar kaldırınca navigasyon kaybolur | Yüksek | Yüksek | Tab bar önceden hazır olmalı, sidebar kaldırma son adım |
| Token dosyası import edilmez | Düşük | Yüksek | index.css'e ilk satırda @import |
| Alt sayfa bileşenleri sidebar'a bağımlı | Orta | Orta | grep ile tüm sidebar referanslarını bul |
| MC dashboard verileri bozulur | Düşük | Yüksek | Eski MC bileşenlerine DOKUNMA, yeni ControlDashboard paralel oluştur |
| 280 sayfa hepsini değiştirmek imkansız | Kesin | Orta | Sadece aktif kullanılan ~15 sayfayı hedefle, geri kalanı kademeli |
