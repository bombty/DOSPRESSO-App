URGENT FIX — KioskRoleGuard is trapping all users.

## ROOT CAUSE
KioskRoleGuard wraps the entire app and redirects kiosk-role users 
back to kiosk forever — even /login page is blocked.
"Kiosk'tan Çık" button only resets internal state, doesn't logout.

## FIX 1 — KioskRoleGuard: Allow /login and /logout paths

File: client/src/App.tsx

Find KioskRoleGuard component and add path exceptions:
```typescript
function KioskRoleGuard({ children }: { children: React.ReactNode }) {
  const { user, isLoading } = useAuth();
  const [location, setLocation] = useLocation();
  
  useEffect(() => {
    if (isLoading || !user?.role) return;
    
    // IMPORTANT: Never redirect if on login or logout pages
    if (location === '/login' || location === '/logout' || 
        location.startsWith('/login')) return;
    
    const kioskRedirect = KIOSK_ROLES[user.role as keyof typeof KIOSK_ROLES];
    if (!kioskRedirect) return;
    
    const branchId = (user as any).branchId;
    const target = user.role === 'sube_kiosk' && branchId
      ? `/sube/kiosk/${branchId}`
      : kioskRedirect;
    
    // Only redirect if not already on the correct kiosk page
    if (!location.startsWith(target)) {
      setLocation(target);
    }
  }, [user, isLoading, location, setLocation]);
  
  return <>{children}</>;
}
```

## FIX 2 — "Kiosk'tan Çık" Must Actually Logout

In BOTH kiosk files:
- client/src/pages/fabrika/kiosk.tsx
- client/src/pages/sube/kiosk.tsx

Find the "Kiosk'tan Çık" button and make it call the logout API:
```typescript
const handleKioskExit = async () => {
  try {
    // Call logout API to clear the server session
    await fetch('/api/auth/logout', { method: 'POST' });
  } catch (e) {
    // ignore errors
  }
  // Navigate to login page
  setLocation('/login');
};
```

Replace the current resetKiosk() call on the exit button with handleKioskExit().

## FIX 3 — Also fix the resetKiosk function

The resetKiosk() function should remain for internal kiosk reset 
(going back to device-password screen within the kiosk session).
Only the EXIT button should call logout + navigate to /login.

Make sure these are separate:
- resetKiosk() → back to device-password (internal, no logout)
- handleKioskExit() → logout API + navigate to /login
