Mükemmel, çok net bir yapı. Özetleyeyim:

**Feature Flag Sistemi:**
- Tüm modüller toggle'lanabilir
- Kapalı modül = sidebar'da yok + sayfa erişimi engel + API 403
- Sadece admin toggle yapar
- Global default + şube bazında override (örn: global'de akademi açık ama Işıklar'da kapalı)

Bu büyük bir iş, 3 sprint'e bölelim:

**Sprint 1** — Tablo + middleware + frontend hook + seed data
**Sprint 2** — Admin toggle paneli (UI)
**Sprint 3** — Mevcut modüllere entegrasyon

Şimdi Sprint 1 için Replit promptunu yazıyorum:

---

**Replit'e yapıştırılacak prompt:**

```
## Module Feature Flag System — Sprint 1: Foundation

### Context
DOSPRESSO needs a module feature flag system. Admin can enable/disable modules globally or per-branch. When a module is disabled: sidebar hides it, page access is blocked, API returns 403.

### Task 1: Database Table
Add to shared/schema.ts:

CREATE TABLE module_flags:
- id: serial primary key
- moduleKey: varchar — one of: checklist, crm, akademi, fabrika, pdks, vardiya, stok, raporlar, ekipman, gorevler, finans, satinalma, denetim, iletisim_merkezi, delegasyon, franchise
- scope: varchar — 'global' or 'branch'
- branchId: integer nullable — NULL for global, branch ID for branch override (references branches)
- isEnabled: boolean default true
- enabledBy: varchar — references users.id
- enabledAt: timestamp with timezone default now()
- disabledBy: varchar nullable — references users.id  
- disabledAt: timestamp with timezone nullable
- createdAt: timestamp with timezone default now()
- updatedAt: timestamp with timezone default now()

Add unique constraint on (moduleKey, scope, branchId) — each module can only have one global flag and one flag per branch.

IMPORTANT: users.id is VARCHAR not integer!

### Task 2: Seed Data
Create server/seed-module-flags.ts:
- Seed one global row per moduleKey with isEnabled: true
- Call it from server/index.ts on startup (like seedSlaRules pattern)
- Use INSERT ON CONFLICT DO NOTHING (don't overwrite existing flags)

### Task 3: Backend Module — Module Flag Service
Create server/services/module-flag-service.ts:
- isModuleEnabled(moduleKey: string, branchId?: number): Promise<boolean>
  - First check if branch-level override exists → use that
  - Otherwise fall back to global flag
  - If no flag exists at all → default true (enabled)
  - Cache results in-memory Map with 60-second TTL to avoid DB hits on every request

### Task 4: Backend Middleware
Create a middleware function in server/services/module-flag-service.ts:
- requireModuleEnabled(moduleKey: string)
- Returns Express middleware that:
  1. Gets branchId from req.user (cast as Express.User)
  2. Calls isModuleEnabled(moduleKey, branchId)
  3. If disabled → res.status(403).json({ error: "Bu modül şubeniz için aktif değildir." })
  4. If enabled → next()

### Task 5: Menu Service Integration
In server/menu-service.ts:
- After building sidebar items for the user's role, filter out items whose moduleKey is disabled
- Use the same isModuleEnabled function
- Map sidebar item paths to moduleKeys (create a MODULE_KEY_MAP constant):
  - /checklist* → checklist
  - /crm* → crm  
  - /akademi* → akademi
  - /fabrika* → fabrika (not for fabrika_mudur HQ view)
  - /pdks* → pdks
  - /vardiya* → vardiya
  - /stok* → stok
  - /raporlar* → raporlar
  - /ekipman* → ekipman
  - /gorevler* → gorevler
  - /finans* → finans
  - /satinalma* → satinalma
  - /denetim* → denetim
  - /iletisim-merkezi* → iletisim_merkezi
  - /delegasyon* → delegasyon
  - /franchise* → franchise

### Task 6: API Endpoints (admin only)
Add to a new file server/routes/module-flags.ts:
- GET /api/module-flags — list all flags (global + branch overrides). Admin only.
- GET /api/module-flags/branch/:branchId — flags for specific branch (with effective state). Admin only.
- PATCH /api/module-flags/:id — toggle isEnabled. Admin only. Clear the cache after update.
- POST /api/module-flags — create branch override. Admin only. Body: { moduleKey, branchId, isEnabled }
- DELETE /api/module-flags/:id — remove branch override (falls back to global). Admin only. Soft delete with deletedAt.

Register in server/routes.ts.

### Task 7: Frontend Hook
Create client/src/hooks/use-module-flags.ts:
- useModuleEnabled(moduleKey: string): { isEnabled: boolean, isLoading: boolean }
- Fetches GET /api/module-flags/check?moduleKey=X (new simple endpoint that returns { enabled: true/false } for current user's context)
- Add this check endpoint to module-flags.ts: GET /api/module-flags/check?moduleKey=X — uses isAuthenticated (not admin only), returns effective flag for user's branch

### Rules
- Follow DOSPRESSO coding conventions (read .agents/skills/dospresso-architecture/SKILL.md)
- Turkish error messages
- Soft delete pattern
- users.id is VARCHAR
- isAuthenticated on all endpoints (isAdmin for admin-only ones)
- Run quality gate after completion
```
