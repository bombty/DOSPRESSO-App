Yapılacak hazırlıklar:

1) PIN seed planı
- Şube/ofis personeli için branch_staff_pins tablosuna PIN kaydı öner.
- Fabrika personeli için factory_staff_pins tablosuna PIN kaydı öner.
- PIN başlangıç değeri pilot için 0000.
- bcrypt hash placeholder kullan: :BCRYPT_0000_PIN
- Kiosk, admin, system, yatırımcı, soft-deleted kullanıcılar PIN listesine dahil edilmesin.
- Sadece aktif insan personel dahil edilsin.

2) Işıklar kiosk mode
- Işıklar branch_id=5 şu an QR mode görünüyorsa PIN mode’a alınması için SQL preview hazırla.
- Ama mevcut QR ayarlarını silme; sadece mode update öner.

3) HQ kiosk user
- HQ branch_id=23 için kiosk user yok.
- SQL preview’da hqkiosk veya hq_kiosk isimli kiosk/device user oluşturma öner.
- role olarak sube_kiosk mı, hq_kiosk mı sistemde mevcut role göre doğrula.
- Eğer hq_kiosk rolü yoksa sube_kiosk kullan, ama branch_id=23 olacak.
- Admin/system/yatırımcı/personel gibi değil, cihaz hesabı olarak işaretle.
- Kiosk için branch_kiosk_settings bağlantısını kontrol et.

4) Fabrika kiosk filtre riski
- Fabrika kiosk user role=fabrika_operator görünüyor.
- Bu kullanıcı PIN/personel listesine dahil oluyor mu, SQL/kod üzerinden kontrol et.
- Eğer görünme riski varsa raporla.
- Bu preview’da role değiştirme yapma; sadece risk ve öneri ver.
- Gerekirse ayrı frontend/backend filter task açacağız.

5) must_change_password / PIN değişim
- users.must_change_password veya pin_must_change benzeri alan var mı kontrol et.
- Pilot için web şifre değişimi ve kiosk PIN değişimi ayrı mı?
- Eğer alan varsa yeni import edilen kullanıcılar için true yapmayı öner.
- Eğer yoksa sadece not düş.

6) Vardiya
- Bugün için vardiya yok ama plansız giriş destekleniyor.
- Bu fazda shift seed yapma.
- Sadece minimum vardiya seed için ayrı öneri ver.
