Replit ile çalışma hızını/güvenliğini nasıl optimize edelim?

Altın kural: “1 prompt = 1 değişiklik paketi”

Senin sorundaki kafa karışıklığının ana sebebi şu: Replit agent “plan → build” döngüsünde birden fazla hedef verilince bazen doğru dosyaya ekliyor ama UI navigasyonunu kaçırabiliyor.

En hızlı ve risksiz akış:
	1.	Sen tek bir “iş paketi promptu” yapıştırırsın.
	2.	Agent plan çıkarır.
	3.	Planı okursun: “doğru sayfayı mı hedefliyor?”
	4.	Start building
	5.	QA + Republish
	6.	Sonra ikinci paket.

Revise plan ne zaman?

Sadece şu durumda:
	•	Agent planı yanlış dosyaya götürüyorsa,
	•	“refactor yapacağım” gibi riskli şeye kayıyorsa,
	•	RBAC/write-path gibi güvenlik çizgilerini aşıyorsa.

Onun dışında her prompt sonrası Revise plan değil → Start building.
