## Gördüğüm Sorunlar

**1. Öneriler kime/neye ait belli değil**
7 adet "Performans uyarisi" — ama hangisi hangi personel, hangi şube? Hepsi aynı görünüyor. CGO "bunlardan hangisini onaylayayım?" diyemez.

**2. "Detay" tıklayınca "görev bulunamadı"**
Detay linki muhtemelen yanlış bir route'a yönlendiriyor — agent suggestion ile görev sistemi arasında bağlantı kopuk.

**3. Onay/Red bağlamı yok**
✓ tıklayınca "onaylandı" ama NE onaylandı? ✗ tıklayınca "red nedeni" istiyor ama NEYİ reddediyorum? Kullanıcı kör aksiyonlar alıyor.

**4. Tüm öneriler aynı başlık**
7/7 = "Performans uyarisi" — başlıkta personel adı veya şube adı olmalı.

**5. ASCII Türkçe**
- "Performans uyarisi" → "Performans uyarısı"
- "Iptal" → "İptal"

**6. Flow Mode CGO için boş**
"Bugün atanmış görev bulunmuyor" — ama Agent Merkezi'nde 7 bekleyen öneri var. Bu çelişki.

**7. Generic AI metin**
Tüm öneriler "Ekip üyelerinizle birebir görüşmeler yaparak..." gibi genel motivasyon metni. Hangi personel? Skoru kaç? Ne yapılmalı? Somut değil.

## TASK: Fix Agent Center / Mr. Dobody Recommendation Quality Issues

Multiple critical UX problems found in the Agent Center (Agent Merkezi) and Mr. Dobody recommendations system.

### T001 — Suggestions Must Include Person/Branch Name in Title (~2 hours)

**Problem:** All 7 agent suggestions show identical title "Performans uyarisi" with generic motivational text. No indication of WHO or WHICH BRANCH the recommendation is about. CGO cannot distinguish between them.

**Find the suggestion generation code:**
```bash
grep -rn "Performans uyarisi\|performans.*uyari\|performance.*warning" server/ --include="*.ts" | head -20
grep -rn "generateSuggestion\|createSuggestion\|agentSuggestion\|agent.*insight" server/ --include="*.ts" | head -20
```

**Fix:** Every agent suggestion MUST include:
- **Title:** Include the person's name AND branch. Example: "Performans Uyarısı: Cihan Kolakan (Işıklar)"
- **Description:** Include specific data. Example: "Composite skoru 32/100 — 3 aydır tehlike bölgesinde. Son 30 günde 2 checklist kaçırdı."
- NOT generic motivational text like "birebir görüşmeler yaparak motivasyonu artırın"

**The suggestion should contain:**
```typescript
{
  title: "Performans Uyarısı: Cihan Kolakan (Işıklar)",
  description: "Composite skor: 32/100. 3 aydır düşük. Devamsızlık: 4 gün. Checklist: %60.",
  actionLabel: "Profili Gör",
  navigateTo: "/personel-detay/[userId]",
  targetUserId: userId,
  targetBranchId: branchId,
  priority: "high",
  type: "performance_warning"
}
```

**Find where agent skills generate these suggestions:**
```bash
grep -rn "team.tracker\|team_tracker\|performance_coach\|daily_coach" server/agent/ --include="*.ts" -l
ls server/agent/skills/
```

Read each skill file and ensure suggestions include:
- Person name (firstName + lastName or fullName)
- Branch name
- Specific metrics (score, days, percentages)
- Direct link to the relevant person/page

### T002 — Fix "Detay" Link (Currently Shows "Görev Bulunamadı") (~1 hour)

**Problem:** Clicking "Detay" on a suggestion navigates to a page that shows "görev bulunamadı" (task not found). The link target is wrong.

**Find where Detay link is rendered:**
```bash
grep -rn "Detay\|detay.*link\|detay.*navigate\|suggestion.*detail" client/src/ --include="*.tsx" | grep -i "agent\|dobody\|suggest" | head -15
```

**Fix:** The "Detay" link should navigate to the relevant page based on suggestion type:
- Performance warning → `/personel-detay/[targetUserId]` (person's profile)
- Stock warning → `/stok-yonetimi` or branch stock page
- Checklist warning → `/checklists` or branch checklist page
- Training gap → `/egitim` or training report
- Customer feedback → `/misafir-memnuniyeti`

Use the `navigateTo` field from the suggestion data. If navigateTo is empty or invalid, show the suggestion detail in a dialog instead of navigating.

### T006 — Deduplicate Similar Suggestions (~30 min)

**Problem:** 7 suggestions all have the same title and very similar text. Even if they're about different people, they look identical.

**Fix:** In the agent skill that generates performance warnings:
1. Group by type — don't create 7 separate "Performans uyarısı" cards
2. Instead, create ONE summary card: "5 personelde performans düşüklüğü var" with a list inside
3. OR ensure each card has a unique title with the person's name (T001 fix handles this)

After T001, if titles are now unique (e.g., "Performans Uyarısı: Cihan Kolakan"), duplicates become distinguishable. But if the SAME person generates multiple identical suggestions, add dedup:

```bash
grep -rn "dedup\|duplicate\|unique.*suggest" server/agent/ --include="*.ts" | head -10
```
