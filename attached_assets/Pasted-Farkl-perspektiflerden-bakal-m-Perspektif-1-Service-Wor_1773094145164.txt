## TASK: EMERGENCY — Browser Still Shows dispatcher.useState Crash

User cleared browser cache but error persists. The fix works on server side but client still gets old/broken JavaScript.

### APPROACH 1: Force Service Worker to die

Add this to the VERY TOP of client/src/main.tsx (before any React code):

```typescript
// Emergency: unregister all service workers on load
if ('serviceWorker' in navigator) {
  navigator.serviceWorker.getRegistrations().then(registrations => {
    registrations.forEach(reg => reg.unregister());
  });
}
```

### APPROACH 2: Kill Vite cache completely

```bash
# Stop the server
pkill -f vite || true
pkill -f node || true

# Delete ALL caches
rm -rf node_modules/.vite
rm -rf client/dist
rm -rf .cache
rm -rf /tmp/vite-*

# Verify no nested Radix packages
find node_modules/@radix-ui -mindepth 2 -name "node_modules" -type d 2>/dev/null
# Must return nothing

# If nested packages found, delete them
find node_modules/@radix-ui -mindepth 2 -name "node_modules" -type d -exec rm -rf {} + 2>/dev/null

# Do a fresh Vite build to verify it compiles
npx vite build 2>&1 | tail -10
# Must say "built in X seconds" with no errors

# Restart
npm run dev
```

### APPROACH 4: Nuclear — Remove the collapsible component entirely

If the collapsible component was only added for sidebar and isn't critical:

```bash
# Check what uses collapsible
```

If it's only used in 1-2 places, replace it with a simple div + useState toggle:


